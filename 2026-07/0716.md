## 02_numpy - 04_numpy_indexing_slicing.py (인덱싱/슬라이싱)

- 1차원 인덱싱: `arr[0]`, `arr[-1]`처럼 파이썬 리스트와 동일하게 접근
- 2차원 인덱싱: `arr[행, 열]` 형태로 콤마 하나로 위치 지정 (`arr[1,2]`, `arr[-1,-1]`)
- 1차원 슬라이싱: `arr[:5]`(앞 5개), `arr[-3:]`(뒤 3개), `arr[::2]`(2칸씩), `arr[::-1]`(역순) → start부터 end 미만까지 step만큼 추출
- 2차원 슬라이싱: `:`는 해당 방향 전체를 의미 → `arr[0,:]`(첫 행), `arr[:,0]`(첫 열), `arr[:2,:]`(앞 두 행), `arr[:,1:]`(마지막 두 열)
- **Fancy Indexing**: 원하는 위치들을 리스트로 묶어 한 번에 조회 (`numbers[[1,3,5]]`, `students[[0,2]]`)
- **Boolean Indexing**: 조건식을 인덱스 자리에 넣어 조건에 맞는 요소만 추출 (`scores[scores >= 80]`, `numbers[numbers % 2 == 0]`)

## 02_numpy - 05_numpy_operations.py (연산 & Broadcasting)

- 요소별(element-wise) 연산: 같은 shape의 배열끼리 `+ - * /`는 같은 위치 요소끼리 계산
- 스칼라 연산: 배열에 숫자 하나를 연산하면 모든 요소에 적용
- 수학 함수: `np.abs()`(절댓값), `np.round()`(반올림), `np.ceil()`(올림), `np.floor()`(내림)
- **Broadcasting**: shape이 다른 배열끼리도 조건이 맞으면 자동으로 확장해서 연산
  - `(2,3) + (3,)` → 가능 (끝 차원 일치)
  - `(2,3) + (2,1)` → 가능 (한쪽 차원이 1)
  - `(2,3) + (2,)` → 불가능 (끝 차원 불일치)
  - 규칙: 뒤쪽 차원부터 비교해서 같거나 둘 중 하나가 1이면 계산 가능
  - `(2,1)` shape의 가중치 배열은 행마다 다른 가중치를 곱할 때 활용

## 02_numpy - 06_numpy_reshape.py (형태 변경)

- `arr.reshape(행, 열)`: 요소 개수를 유지한 채 형태 변경 (전체 개수가 맞아야 함)
- `reshape(-1, n)` / `reshape(n, -1)`: -1은 "나머지는 Numpy가 알아서 계산"이라는 의미
- `flatten()`: 다차원 → 1차원, **새 배열을 생성**하므로 복사본 수정 시 원본 유지
- `ravel()`: 다차원 → 1차원, 가능하면 **원본 메모리를 공유**하므로 복사본 수정 시 원본도 바뀜
- `transpose()` / `.T`: 행과 열을 뒤바꿈 (전치)

## 02_numpy - 07_numpy_aggregation.py (집계 함수)

- `sum()`, `mean()`: 1차원은 전체 합/평균, 2차원은 전체 요소 기준 합/평균
- `max()`, `min()`: 최댓값/최솟값
- `argmax()`, `argmin()`: 값이 아니라 **최댓값/최솟값의 위치(index)** 반환
- **axis**: `axis=0`은 세로(열별, 과목별) 방향, `axis=1`은 가로(행별, 학생별) 방향으로 집계
- `std()`: 표준편차, 값이 클수록 데이터가 평균에서 멀리 흩어져 있음을 의미

## 02_numpy - 08_numpy_vector.py (벡터)

- Numpy에서 1차원 배열이 곧 벡터, `.shape`/`.ndim`으로 형태·차원 확인
- **Norm(벡터의 크기)**: `np.linalg.norm(v)` → 원점에서 벡터 끝까지의 거리
- **내적(dot product)**: 같은 위치 요소끼리 곱한 뒤 모두 더한 값
  - `np.dot(a, b)` 또는 `a @ b`로 계산 (`@`는 행렬곱 연산자)

## 02_numpy - 09_numpy_simiarity.py (코사인 유사도)

- `cos(A,B) = (A·B) / (|A| * |B|)` → 분자는 두 벡터의 내적, 분모는 두 벡터 크기의 곱
- **Dot Product의 한계**: 내적만 쓰면 벡터의 크기(길이)에 영향을 받아 방향이 같아도 값이 계속 커짐 (`[1,1]·[10,10]`=20, `[1,1]·[100,100]`=200)
- **코사인 유사도**는 크기(scale)에 영향받지 않고 오직 **방향의 유사성**만 -1~1 사이 값으로 표현
  - 방향이 같으면 1에 가까움, 수직이면 0, 반대 방향이면 -1
  - `[1,1]` vs `[10,10]`, `[1,1]` vs `[100,100]` 모두 동일한 유사도(1.0)로 계산됨

## 02_numpy/project - movie_recommender.py (코사인 유사도 기반 영화 추천)

- 흐름: 영화 데이터 준비 → 사용자 취향 입력 → 유사도 계산 → 가장 높은 점수 찾기 → 추천 결과 출력
- 영화별 특징(액션/스릴러/코미디/감동)을 0/1 원-핫 형태의 2차원 배열로 표현
- 사용자 입력도 동일한 형태의 벡터로 받아, 취향을 하나도 선택하지 않으면(`user.any()`가 False) 예외 처리 후 종료
- 각 영화 벡터와 사용자 벡터의 코사인 유사도를 반복 계산해 `scores` 배열 생성
- `np.argmax(scores)`로 가장 유사도가 높은 영화의 인덱스를 찾아 추천

## 03_gpt - 01_token_embedding_attention.py (GPT 핵심 개념 미니 실습)

- **토큰화(tokenize)**: 가장 단순한 형태로 `text.split()`을 이용해 문장을 토큰 리스트로 분리
- **임베딩(embedding)**: 단어를 의미를 담은 숫자 벡터로 표현, 의미가 비슷한 단어일수록 코사인 유사도가 높게 나옴 (사과-바나나 유사도 > 사과-자동차)
- **Attention 예제**: Query·Key·Value 개념을 직접 구현
  - `query`: 지금 찾고자 하는 정보
  - `keys`: 각 토큰이 어떤 특징을 갖는지 표현
  - `scores = keys @ query`로 관련도 점수 계산 → `softmax`로 가중치 변환 → `weights @ values`로 각 value를 가중치만큼 섞은 결과(context) 도출
- **softmax**: `scores - max(scores)`로 값을 안정화(overflow 방지)한 뒤 지수화하고 합으로 나눠 확률분포로 변환
- **다음 토큰 예측 흉내**: 후보 토큰들의 점수에 softmax를 적용해 확률로 바꾸고, `argmax`로 가장 가능성 높은 다음 토큰 선택 → GPT가 다음 단어를 예측하는 원리를 축약해서 보여주는 예제
