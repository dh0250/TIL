## 사용자 문의 API 구현 (PR #119)

### 완성한 기능

- `POST /api/v1/inquiries` 문의 등록 API (#109)
- `GET /api/v1/inquiries` 본인 문의 목록 조회 API (#110)
- `GET /api/v1/inquiries/{inquiryId}` 문의 상세 조회 API (#111)
- `PATCH /api/v1/inquiries/{inquiryId}` 문의 수정 API (#112)
- `DELETE /api/v1/inquiries/{inquiryId}` 문의 삭제 API - 소프트 삭제 (#113)

### 주요 구현 내용

- `InquiryRepository` - `findByMemberIdAndIsDeletedFalse()` JPA 자동 쿼리 생성
- `InquiryAnswerRepository` - `findByInquiryIdAndIsDeletedFalse()` 삭제된 답변 제외
- `InquiryRepository` - `findByIdAndIsDeletedFalse()` 삭제된 문의 상세 조회 차단
- `resolveMemberId(email)` - JWT email → memberId 변환
- `validateUser()` - 본인 문의 소유권 검증 (403 AccessDeniedException)
- `validateNotAnswered()` - 답변 완료 문의 수정/삭제 차단 (400)
- `InquiryAlreadyAnsweredException` 추가 및 `GlobalExceptionHandler` 등록

### DTO 구조


### 배운 것

**JPA 메서드명 자동 쿼리 생성**

- `findByMemberIdAndIsDeletedFalse` → `WHERE member_id=? AND is_deleted=false`
- `@Param` 불필요 (직접 `@Query` 작성할 때만 필요)
- 단순 조건 → 메서드명으로, 복잡한 조건 (null 처리 등) → `@Query` 직접 작성

**소프트 삭제 확인 방법**

- DB에서 `is_deleted=false` 필터 걸려있으면 삭제된 행 안 보임
- 전체 조회: `SELECT * FROM inquiries ORDER BY inquiry_id DESC`

**SQL 로그 확인 방법**

```yaml
spring:
  jpa:
    show-sql: true
    properties:
      hibernate:
        format_sql: true
logging:
  level:
    org.hibernate.SQL: DEBUG
    org.hibernate.orm.jdbc.bind: TRACE
```
