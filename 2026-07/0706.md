### 팀 테스트 코드 패턴

```java
@SpringBootTest
@Transactional   // 테스트 후 DB 자동 롤백
class AdminXxxServiceTest {

    @Autowired
    private AdminXxxService adminXxxService;

    @Autowired
    private XxxRepository xxxRepository;

    @Test
    @DisplayName("한글로 테스트 이름 작성")
    void methodName_scenario() {
        // given - 테스트 데이터 준비
        Member member = Member.createUser("test@test.com", "pw", "닉네임", HouseholdType.ONE_PERSON);
        memberRepository.save(member);

        // when - 실제 메서드 호출
        Result result = service.method(...);

        // then - 결과 검증
        assertThat(result).isNotNull();
    }
}
```

### JdbcTemplate vs Repository.save() 비교

| 방식 | 장점 | 단점 |
| --- | --- | --- |
| `JdbcTemplate` | JPA 캐시 우회, 빠름 | SQL 직접 작성, 유지보수 어려움 |
| `Repository.save()` | 실제 코드와 동일한 방식 | JPA 1차 캐시 주의 필요 |

팀 프로젝트에서는 `Repository.save()` 방식이 더 자연스럽고 유지보수하기 쉬움

### 배운 것

**given / when / then 패턴**

- `given`: 테스트에 필요한 데이터 DB에 저장
- `when`: 테스트할 Service 메서드 호출
- `then`: `assertThat()`으로 결과 검증

**예외 케이스 테스트**

```java
Throwable thrown = catchThrowable(() -> service.method(wrongId));
assertThat(thrown).isInstanceOf(XxxNotFoundException.class);
```

**assertThat 주요 메서드**

```java
assertThat(result).isNotNull();
assertThat(result).isEqualTo(expected);
assertThat(list).hasSize(1);
assertThat(list).extracting(Response::field).containsOnly(value);
assertThat(thrown).isInstanceOf(Exception.class);
```
