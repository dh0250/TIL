# 냉파마스터 백엔드 프로젝트 요약 🧊

## 서비스 개요

냉파마스터는 냉장고 속 식재료를 관리하고, 소비기한 알림과 보유 재료 기반 레시피 추천을 통해 식재료 낭비를 줄이는 서비스. 백엔드는 회원 인증, 냉장고/장보기, 레시피, 댓글, 문의, 알림, 냉파 점수, 통계, 관리자 기능을 담당.

## 기술 스택

- **Language**: Java 21
- **Framework**: Spring Boot 4.1.0
- **Security**: Spring Security, JWT
- **Database**: PostgreSQL, H2
- **Persistence**: Spring Data JPA, MyBatis
- **Migration**: Flyway
- **Mail**: Java Mail Sender, SMTP
- **Build/Test**: Gradle, JUnit 5, Spring Boot Test, Mockito, MockMvc

## 주요 기능

- **인증/회원**: 회원가입·로그인, access/refresh token 발급·재발급, 이메일 인증(5회 오입력 시 재발급), 탈퇴 회원 이메일 재사용 차단, 일반/관리자 권한 분리
- **냉장고/장보기/사전 재료**: 재료 CRUD, 전체/부분 사용 처리, 소비기한 임박·만료 조회, 장보기 목록 관리 및 구매 시 쿠팡 이동, 장보기→냉장고 이동, 사전 재료 검색·카테고리 조회
- **레시피/댓글**: 레시피 CRUD, 보유/부족 재료 계산, 보유 재료 비율·임박 재료·선호 음식·제외 재료 기반 추천, 키워드 검색, 좋아요, 댓글 CRUD(작성자/관리자 권한 제어)
- **문의**: 문의 등록·조회, 답변 전 수정/삭제, 답변 완료 후 수정/삭제 제한, 관리자 답변 등록 시 알림 생성
- **알림/이메일**: 소비기한 임박/만료 알림, 문의 답변 알림, 이메일 인증 이력 정리 스케줄러
- **냉파 점수/통계**: 레시피 등록 시 점수 증가, 만료 발생 시 차감, 무만료 4일 유지 시 보상, 카테고리별 만료량·자주 버려지는 재료 TOP5·최근 만료 기록 조회
- **관리자**: 회원 조회/탈퇴/복구/권한 변경, 문의·레시피·사전 재료 관리, 평균 냉파 점수·주간 만료 추이·TOP5 등 운영 통계

## 인증 흐름

이메일 인증 코드 요청 → 기존/탈퇴 회원 확인 → 코드 발송 → 코드 확인 → 회원가입 → 로그인 시 access/refresh token 발급 → 보호 API는 JWT 기반으로 사용자/관리자 권한 판단.

## 스케줄링

| 스케줄러 | 실행 시점 | 역할 |
| --- | --- | --- |
| `DailyScoreScheduler` | 매일 00:00 | 만료 재료 확인, 알림 생성, 만료 이력 저장, 냉파 점수 반영 |
| `EmailVerificationCleanupScheduler` | 매일 01:00 | 만료/오래된 이메일 인증 이력 정리 |

관리자 수동 실행 API: `POST /api/v1/admin/scores/run-scheduler`

## 프로젝트 구조

```text
src/main/java/com/naengpa/naengpamasterbackend
├── admin          # 관리자 회원/문의/레시피/재료/통계 관리
├── comment        # 레시피 댓글
├── fridge         # 냉장고 재료
├── global         # 인증, 보안, 예외, 공통 응답
├── inquiry        # 사용자 문의
├── member         # 회원, 선호 음식, 제외 재료
├── notification   # 알림
├── product        # 사전 재료, 재료 카테고리
├── recipe         # 레시피, 추천, 좋아요
├── score          # 냉파 점수, 점수 이력, 스케줄러
├── shopping       # 장보기
└── statistics     # 사용자 만료 통계
```

## 품질 관리 포인트

- 사용자/관리자 API 분리 + JWT 권한 검증으로 접근 범위 제어
- 공통 응답 형식과 전역 예외 처리로 프론트엔드 일관성 확보
- 요청 DTO 검증으로 서비스 로직 진입 전 오류 차단
- 주요 도메인에서 본인 데이터 접근 여부 검증
- 삭제 필요 데이터는 soft delete로 이력 추적 가능성 유지
- refresh token, 이메일 인증, 알림 읽음 상태 등은 별도 테이블로 관리
- 냉파 점수·만료 이력·알림·통계를 스케줄러와 도메인 서비스 분리로 연결
- 레시피 추천은 보유/부족 재료, 선호/제외 재료, 임박 재료를 함께 계산
- 관리자 통계 API는 평균 점수, 만료량, TOP5, 주간 추이를 분리된 응답 DTO로 제공
