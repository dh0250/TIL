# 실제 OpenAI Function Calling API (02_openai_function_calling_api.py)

01_function_calling_demo.py에서 흉내만 냈던 tool 선택 과정을 실제 OpenAI API로 구현. 흐름은 4단계: ① 질문 + tool 목록을 GPT에 전달 → ② GPT가 호출할 tool과 인자를 선택 → ③ Python이 실제 함수를 실행 → ④ 실행 결과를 다시 GPT에 전달해 최종 자연어 답변을 받음.

```python
TOOLS = [
    {
        "type": "function",
        "function": {
            "name": "search_lesson",
            "description": "수업 자료를 검색한다.",
            "parameters": {
                "type": "object",
                "properties": {"keyword": {"type": "string"}},
                "required": ["keyword"]
            }
        }
    }
]

response = client.chat.completions.create(
    model="gpt-4.1-mini",
    messages=messages,
    tools=TOOLS
)

assistant = response.choices[0].message
tool_call = assistant.tool_calls[0]
args = json.loads(tool_call.function.arguments)

result = search_lesson(**args)

messages.append(assistant)
messages.append({
    "role": "tool",
    "tool_call_id": tool_call.id,
    "content": json.dumps(result, ensure_ascii=False)
})

response = client.chat.completions.create(model="gpt-4.1-mini", messages=messages)
```

- `tools` 파라미터로 함수 스키마(JSON Schema)를 전달하면 모델이 직접 실행하지 않고 "이 함수를 이 인자로 호출해라"라는 결정만 반환한다.
- tool 실행 결과는 `role: "tool"` 메시지로, 반드시 해당 `tool_call.id`와 함께 대화 기록에 추가해야 모델이 문맥을 이어서 최종 답변을 만든다.

# LangChain PromptTemplate / ChatPromptTemplate (01_prompt_templagte_basic.py)

LangChain으로 프롬프트를 템플릿화하는 두 가지 방식.

- **PromptTemplate**: 하나의 문자열 안에 `{변수}`를 넣어 완성된 프롬프트 문자열(`invoke` 결과)을 만든다.
- **ChatPromptTemplate**: `(role, content)` 튜플 리스트로 system/user 메시지를 구성하고, `to_messages()`로 실제 메시지 객체 목록을 얻는다.

```python
template = PromptTemplate.from_template(
    """
    너는 AI기초를 가르치는 강사다.
    {audience}에게 {topic}을 쉽게 설명해라.
    """
)
prompt_value = template.invoke({"audience": "Python과 Numpy를 막 배운 학생", "topic": "Embedding"})

chat_prompt = ChatPromptTemplate.from_messages([
    ("system", "너는 초보자를 가르치는 AI강사다. 어려운 용어는 쉽게 풀어쓴다."),
    ("user", "질문:{question}\n답변 형식: {answer_format}")
])
prompt_value = chat_prompt.invoke({
    "question": "Attention이 왜 필요한가요?",
    "answer_format": "1. 한줄 정의\n2. 쉬운 예시\n3. Numpy와 연결\n4. 요약"
})
for message in prompt_value.to_messages():
    print(f"[{message.type}] {message.content}")
```

두 방식 모두 `invoke()`로 실행하며, 이후 배울 LCEL 체인의 첫 구성 요소가 된다.

# LCEL로 체인 구성하기 (02_runnable_chain_basic.py)

LangChain Expression Language(LCEL)의 핵심은 `|` 연산자로 여러 컴포넌트를 파이프라인처럼 연결하는 것. `PromptTemplate | ChatOpenAI | StrOutputParser` 순서로 이어 붙이면 "프롬프트 완성 → 모델 호출 → 응답에서 문자열만 추출"이 하나의 체인으로 묶인다.

```python
template = PromptTemplate.from_template(
    """
    너는 AI 기초 강사다.
    주제: {topic}
    대상: {audience}
    ...
    """
)

model = ChatOpenAI(model="gpt-4.1-mini", temperature=0.7, max_tokens=300)

chain = template | model | StrOutputParser()

result = chain.invoke({
    "topic": "Attention",
    "audience": "Python과 Numpy를 막 배운 학생"
})
```

- `StrOutputParser()`가 없으면 `chain.invoke()`의 결과는 메시지 객체(`AIMessage`)이며, 붙이면 바로 `content` 문자열만 반환된다.
- API 키 누락 등은 `os.getenv("OPENAI_API_KEY")`로 사전 검증 후 `ValueError`를 발생시켜 명확하게 실패시킨다.

# JsonOutputParser로 구조화된 출력 받기 (03_output_parser_practice.py)

Pydantic 모델로 원하는 출력 스키마를 정의하면, `JsonOutputParser`가 그 스키마에 맞는 포맷 지시문(`format_instructions`)을 자동 생성해 프롬프트에 주입해준다. 03_gpt 단계에서 `response_format`으로 JSON을 강제했다면, LangChain에서는 파서가 스키마 설명 + 파싱까지 함께 처리한다.

```python
class SentimentAnalysis(BaseModel):
    label: str = Field(description="감정 분석 결과('긍정', '부정', '중립' 중 하나)")
    reason: str = Field(description="그렇게 판단한 상세 이유")

parser = JsonOutputParser(pydantic_object=SentimentAnalysis)

template = PromptTemplate(
    template="""
    아래 문장을 분석하여 지정된 JSON형식으로 출력해라
    {format_instructions}
    문장:
    {sentence}
    """,
    input_variables=["sentence"],
    partial_variables={"format_instructions": parser.get_format_instructions()}
)

model = ChatOpenAI(model="gpt-4.1-mini", temperature=0.0, max_tokens=200)
chain = template | model | parser

result = chain.invoke({"sentence": "수업이 어렵지만 예제가 있어서 따라갈 만하다."})
print(result["label"], result["reason"])
```

- `partial_variables`로 템플릿 생성 시점에 `format_instructions`를 미리 채워 넣어, 매 `invoke()`마다 반복 입력할 필요가 없게 한다.
- 체인의 마지막이 `parser`이므로 `chain.invoke()` 결과가 곧바로 dict(Pydantic 스키마 형태)로 나온다.

# 퀴즈 생성기: PromptTemplate + LCEL + JsonOutputParser 종합 (04_quiz_generation.py)

앞서 배운 요소(PromptTemplate, LCEL 체인, Pydantic 기반 JsonOutputParser)를 모두 합쳐 주제를 입력하면 객관식 퀴즈 1개를 JSON으로 생성하는 프로그램.

```python
class Quiz(BaseModel):
    question: str = Field(description="퀴즈 문제")
    choices: list[str] = Field(description="보기 4개")
    answer: str = Field(description="정답")
    explanation: str = Field(description="정답 설명")

parser = JsonOutputParser(pydantic_object=Quiz)

prompt = PromptTemplate(
    template="""
    당신은 AI 강사입니다.
    주제: {topic}
    객관식 퀴즈를 1개 만들어 주세요.
    {format_instructions}
    """,
    input_variables=["topic"],
    partial_variables={"format_instructions": parser.get_format_instructions()}
)

model = ChatOpenAI(model="gpt-4o-mini", temperature=0.7)
chain = prompt | model | parser

result = chain.invoke({"topic": "Embedding"})
```

- `list[str]` 같은 Pydantic 필드 타입도 `format_instructions`에 그대로 반영되어, 모델이 `choices`를 배열 형태로 정확히 채워 넣는다.
- 이 패턴(스키마 정의 → 파서 생성 → 프롬프트에 지시문 삽입 → LCEL 체인 실행)은 감정 분석뿐 아니라 퀴즈 생성처럼 구조가 정해진 어떤 생성 작업에도 재사용 가능하다는 점이 핵심.
