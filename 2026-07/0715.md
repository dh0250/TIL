## 06_api - 02_datetime.ipynb (날짜/시각)

- `datetime.today()` / `datetime.now()`: 현재 날짜+시각 반환 (`datetime.datetime` 타입)
- 속성 접근: `now.year`, `.month`, `.day`, `.hour`, `.minute`, `.second`
- `now.date()` / `now.time()`: 날짜부분 / 시각부분만 분리해서 반환
- `now.weekday()`: 월요일(0)~일요일(6) 인덱스 반환 → 요일 리스트와 매핑해서 요일 구하기
- **형식표현(strftime/strptime)**
  - `%Y`(연도4자리)/`%y`(연도2자리), `%m`(월), `%d`(일), `%H`(24시간)/`%I`(12시간), `%M`(분), `%S`(초), `%a`(요일 약어)
  - `now.strftime(fmt)`: 날짜/시간 → 문자열
  - `datetime.strptime(text, fmt)`: 문자열 → 날짜/시간 객체
- `timedelta(days=, hours=...)`: 날짜 연산(더하기/빼기)에 사용 → 미래/과거 날짜 계산
- D-day 계산: `date.today()`와 목표 `date` 객체를 빼면 `timedelta` 반환, `.days`로 남은 일수 추출

## 06_api - 01_builtins.ipynb (내장 모듈/함수)

- `dir(__builtins__)`: import 없이 바로 쓸 수 있는 내장 함수/예외 목록 확인
- `map(func, iterable)`: 각 요소에 함수를 적용한 결과를 map 객체로 반환 (리스트내포로도 대체 가능). 여러 iterable을 동시에 넘겨 인자로 사용 가능
- `filter(func, iterable)`: 함수 결과가 True인 요소만 남김 (리스트내포의 `if`조건과 동일한 역할)
- `functools.reduce(func, iterable, 초기값)`: iterable을 순회하며 누적연산 후 단일 값 반환
  - `func(누적값, 현재값)` 형태로 호출됨, 초기값 생략 시 첫 요소가 초기값
  - 합계, 최대값 구하기 등에 활용

## 07_oop - 01_oop.ipynb (객체지향 기초)

- 클래스: 객체의 설계도, 관례상 PascalCase로 명명
- `__init__(self, ...)`: 생성자, 객체 생성 시 속성 초기화 담당. 모든 인스턴스 메소드는 첫 매개변수로 `self` 필요
- 클래스 속성 vs 인스턴스 속성: 클래스 속성은 모든 인스턴스가 공유, 인스턴스 속성은 객체별로 별도 저장
- `@classmethod`: 첫 인자로 클래스(`cls`) 전달받음, 클래스 속성 활용
- `@staticmethod`: 클래스/인스턴스 인자 없이 독립적으로 동작하는 메소드
- **객체지향 4대 원칙**: 캡슐화, 상속, 다형성, 추상화

## 07_oop - 02_encapsulation.ipynb (캡슐화)

- 캡슐화 없이 속성을 직접 노출하면 외부에서 잘못된 값(음수 곱셈 등)으로 데이터가 훼손될 수 있음
- `self.__속성명`: private 속성 선언 → 클래스 외부에서 직접 접근 불가
- `deposit()`, `withdraw()`처럼 public 메소드 내부에서 유효성 검사 후 private 속성을 수정하도록 강제
- `@property`: private 속성을 읽기 전용으로 노출하는 getter
- `@속성명.setter`: 해당 속성에 값을 대입할 때 실행되는 setter (검증 로직 삽입 가능)

## 08_module - 01_scope.ipynb (변수 유효범위)

- global scope(전역변수): 어디서나 접근 가능, 다른 코드셀에서도 유지
- local scope(지역변수): 함수/블럭 내부에서만 유효. 같은 이름이면 지역변수가 전역변수보다 우선 사용됨
- **call by value**: 함수 인자로 불변(immutable) 타입(숫자 등)을 넘기면 함수 내부에서 값을 바꿔도 원본에 영향 없음
- **call by reference**: 리스트 등 가변(mutable) 타입을 넘기면 함수 내부에서의 변경이 원본에도 반영됨

## 08_module - 02_module.ipynb (모듈)

- 모듈 = `.py` 파일 단위로 변수/함수/클래스를 묶어 재사용
- `dir(모듈명)`: 모듈이 제공하는 이름들 조회, `dir()`: 현재 모듈에서 사용 가능한 이름들
- `__name__`: 직접 실행 시 `'__main__'`, import된 모듈은 자기 파일명
- import 방식 비교
  - `import 패키지.모듈` → `패키지.모듈.변수`로 접근 (번거로움)
  - `from 패키지 import 모듈` (추천) → `모듈.변수`
  - `from 패키지 import 모듈 as m` → 별칭 사용
  - `from 모듈 import 변수, 함수` → 이름 충돌 위험 있어 주의, `import *`는 비추천

## 09_io - 01_os.ipynb (os 모듈)

운영체제와 상호작용(디렉터리/파일/환경변수/경로 처리)하는 표준 라이브러리

- 디렉터리: `os.getcwd()`, `os.chdir()`, `os.listdir()`, `os.mkdir()`(1개), `os.makedirs()`(중첩), `os.rmdir()`, `os.removedirs()`
- 파일: `os.remove()`, `os.rename()`, `os.stat()`

- 실습: `os.path.exists()`로 존재 확인 후 `os.mkdir()`로 디렉토리 생성, `os.path.join()`으로 안전하게 파일 경로 조합

## 09_io - 02_io.ipynb (파일 입출력)

- `open(경로, 모드, encoding=)`: 파일을 열기 위한 기본 함수
- 모드: `'r'`(읽기, 없으면 에러) / `'w'`(쓰기, 덮어쓰기, 없으면 생성) / `'a'`(이어쓰기) / `'x'`(배타적 생성) / `'b'`(바이너리) / `'t'`(텍스트, 기본)
- 읽기: `read()`(전체), `readline()`(한 줄), `readlines()`(전체를 리스트로)
- 쓰기: `write()`(문자열 1개), `writelines()`(문자열 리스트)
- `f.close()`를 꼭 호출해야 메모리 누수 방지 → **`with open(...) as f:`** 블럭 사용 시 자동으로 닫아줌 (권장)
- 파일 순회: `for line in f:` 또는 `while line := f.readline():` (바다코끼리 연산자로 대입+조건 동시 처리)
- `'a'` 모드로 열면 기존 내용 뒤에 이어서 씀

## 10_exception - 01_exception.ipynb (예외처리)

- 파이썬은 Exception과 Error를 엄격히 구분하지 않음: `TypeError`, `ValueError`, `MemoryError` 등 이름에 Error가 붙어도 모두 `Exception`의 하위 클래스
- 예외 계층: `BaseException` → `SystemExit`/`KeyboardInterrupt`/`GeneratorExit`/`Exception`(대부분의 예외가 여기 속함)
- `try...except`: 예외 발생 시 정상 흐름으로 복구. `except Exception as e:`로 예외 객체 접근 가능
- 여러 예외 타입을 구체적인 것부터 순서대로 `except`로 분기 처리 가능 (`ZeroDivisionError`, `IndexError` → 마지막에 `Exception`으로 나머지 처리)
- `try / except / else / finally` 구조
  - `else`: try블럭이 예외 없이 끝났을 때만 실행
  - `finally`: 예외 발생 여부와 상관없이 항상 실행
- `traceback.print_exc()`: 예외 발생 시 상세 스택트레이스 출력(디버깅용)
- `except: pass`로 예외를 무시하면 디버깅이 어려워지므로 확실한 경우에만 사용
- `raise 예외타입('메시지')`: 명시적으로 예외를 발생시킴 (예: 미구현 기능에 `NotImplementedError`)

## 02_numpy - 01_numpy_intro.py (Numpy 개요)

- `import numpy as np`, `np.__version__`로 버전 확인
- 파이썬 리스트의 `+`는 리스트 연결이지만, Numpy 배열의 `+`는 같은 위치 요소끼리 더하는 **요소별(element-wise) 연산**
- 2차원 배열(행=학생, 열=과목)로 성적표 표현 가능
- `arr.mean()`, `.max()`, `.min()`: 전체 통계
- `axis` 매개변수로 방향 지정: `axis=0`(열방향, 과목별 평균), `axis=1`(행방향, 학생별 평균)

## 02_numpy - 02_numpy_array_create.py (배열 생성)

- `np.array(list, dtype=)`: 리스트 → Numpy 배열. 1차원은 한 줄, 2차원은 행의 길이가 모두 같아야 함
- `np.zeros(shape)` / `np.ones(shape)`: 0/1로 채운 배열 생성, shape는 튜플로 (행, 열) 지정. 기본 dtype은 float
- `np.full(shape, value)`: 지정한 값으로 채운 배열 생성
- `np.arange(start, stop, step)`: 간격(step) 기준으로 생성 (파이썬 `range`와 유사)
- `np.linspace(start, stop, num)`: 개수(num) 기준으로 균등 분할하여 생성
- `arange`는 "간격" 기준, `linspace`는 "개수" 기준이라는 차이가 핵심

## 02_numpy - 03_numpy_array_attributes.py (배열 속성)

- `arr.shape`: 배열의 모양을 튜플로 반환. 1차원 `(개수,)`, 2차원 `(행, 열)`, 3차원 `(깊이, 행, 열)`
- `arr.ndim`: 배열의 차원 수
- `arr.size`: 전체 요소 개수 (shape 값들의 곱)
- `arr.dtype`: 배열 요소의 자료형. Numpy 배열은 하나의 자료형만 저장하므로, 숫자와 문자열이 섞이면 자동으로 문자열 등으로 통일 변환됨
