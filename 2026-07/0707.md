# 오늘 한 일

## 냉파마스터 DDL 시드 데이터 정합성 수정 (`fix/BE-sql-create`)

### 작업 내용

- 로컬에서 DB를 새로 생성할 때 사용하는 `db/냉파마스터_ddl.sql`의 `products` 시드 데이터가 실제 운영 DB와 어긋나는 문제 발견
- 기존에는 `INSERT INTO products (product_id, product_category_id, name, default_expiry_days, is_active)` 구문과, 이후 컬럼 구성이 다른(`created_at`, `updated_at` 포함) 두 번째 `INSERT` 구문으로 나뉘어 있어 데이터 정합성이 깨져 있었음
- 운영 DB의 `products` 테이블을 실제 스냅샷 기준으로 재추출하여 하나의 `INSERT` 구문으로 통합
- `product_id`를 명시적으로 지정하던 방식에서 제거하고, 시퀀스(auto-increment)로 생성되도록 변경
- `created_at` / `updated_at` 값을 운영 데이터 실제 값 그대로 반영
- 변경 규모: `db/냉파마스터_ddl.sql` 1,799줄 추가 / 4,607줄 삭제
- PR #208 (`fix/BE-sql-create`) 로 리뷰 후 `dev` 브랜치 병합 완료

### 배운 것

**시드 데이터와 PK 관리**

- 마이그레이션/시드 파일에서 PK를 하드코딩하면, 운영 데이터가 바뀌었을 때 재생성 시 실제 데이터와 어긋나기 쉬움
- `product_id`처럼 auto-increment 컬럼은 값을 직접 지정하지 않고 시퀀스에 맡기는 편이 로컬 DB 재생성 시 운영 환경과의 정합성을 유지하기 더 안전함
- DDL/시드 파일은 스키마 변경(컬럼 추가 등) 시점마다 전체를 다시 덤프해서 동기화하는 것이 부분 수정보다 안전함
