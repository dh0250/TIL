### 배운 것

**소프트 삭제 + 재등록 시 DB 구조**

```
inquiry_id | content    | is_deleted
1          | 첫번째 답변 | true   ← 삭제된 거
1          | 두번째 답변 | false  ← 현재 유효한 답변
```

- `findByInquiryId()` → 삭제된 답변 포함 조회 (문제)
- `findByInquiryIdAndIsDeletedFalse()` → 유효한 답변만 조회 (해결)

**JPA 메서드명에 정렬 조건 추가**

```java
// WHERE is_deleted = false ORDER BY created_at DESC
findByIsDeletedFalseOrderByCreatedAtDesc(Pageable pageable)
```

## 관리자 통계 API 구현 완료 (PR)

### 완성한 기능

- `GET /api/v1/admin/statistics/score-average` 냉파 점수 평균 조회 (#75)
- `GET /api/v1/admin/statistics/expired-count` 만료 건수 조회 - 이번주/지난주 비교 + 증감률 (#76)
- `GET /api/v1/admin/statistics/category?period={7|30}` 카테고리별 만료량 조회 (#77)
- `GET /api/v1/admin/statistics/top-ingredients` 자주 버려지는 재료 TOP5 + 순위 변동 (#78)
- `GET /api/v1/admin/statistics/weekly-trend` 주간 만료 추이 (최근 6주) (#79)

### 주요 구현 내용

- `AdminScoreRepository` - `AVG(s.score)` JPQL 집계 쿼리
- `AdminStatisticsRepository` - `ExpiredProduct` 기반 집계 쿼리
- 만료 건수: 이번주/지난주 `BETWEEN` 쿼리 + 증감률 계산
- TOP5: 이번주/지난주 각각 조회 후 Map으로 순위 비교 → `rankChange` 계산
- 카테고리별: `period` 필수 파라미터 (7 또는 30), 그 외 400 에러
- 주간 추이: for문 6회 `countByCreatedAtBetween()` 호출 후 label 붙여서 반환

### 배운 것

**JPQL vs Native Query**

- JPQL: Entity 클래스명/필드명 기준 (`FROM Score s`)
- Native Query: 실제 테이블명/컬럼명 기준 + `nativeQuery = true`
- JPQL은 `LIMIT` 미지원 → `PageRequest.of(0, 5)`로 대체

**JPQL GROUP BY 결과 Object[]로 받기**

```java
List<Object[]> results = repository.findGroupByResult();
results.stream().map(row -> new Response(
    (String) row[0],  // 첫번째 컬럼
    (Long) row[1]     // 두번째 컬럼
)).toList();
```

**rankChange 계산 로직**

```java
Map<String, Integer> lastWeekRankMap = new HashMap<>();
for (int i = 0; i < lastWeek.size(); i++) {

Integer rankChange = lastRank != null ? lastRank - currentRank : null;

```
