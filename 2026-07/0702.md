## 관리자 통계 API 구현

### 완성한 기능

- `GET /api/v1/admin/statistics/score-average` 냉파 점수 평균 조회 (#75)
- `GET /api/v1/admin/statistics/expired-count` 만료 건수 조회 - 이번주/지난주 비교 (#76)
- `GET /api/v1/admin/statistics/category` 카테고리별 만료량 조회 (#77)
- `GET /api/v1/admin/statistics/top-ingredients` 자주 버려지는 재료 TOP5 조회 (#78)
- `GET /api/v1/admin/statistics/weekly-trend` 주간 만료 추이 조회 (#79)

### 주요 구현 내용

- `AdminScoreRepository` - `AVG(s.score)` JPQL 집계 쿼리
- `AdminStatisticsRepository` - `ExpiredProduct` 기반 집계 쿼리
- 만료 건수: 이번주/지난주 `BETWEEN` 쿼리로 각각 조회 후 증감률 계산
- 카테고리별: `startDate IS NULL OR createdAt >= startDate` 패턴으로 7일/30일/전체 통합 처리
- TOP5: JPQL은 `LIMIT` 미지원 → `Pageable`로 대체 (`PageRequest.of(0, 5)`)
- 주간 추이: for문으로 6주 각각 조회 후 label 붙여서 반환

### DTO 구조

```
AdminScoreAverageResponse   - averageScore, memberCount
AdminExpiredCountResponse   - thisWeekCount, lastWeekCount, weekChangePct
AdminCategoryStatResponse   - categoryName, expiredCount
AdminTopExpiredIngredientResponse - productName, discardedCount
AdminWeeklyTrendResponse    - List<AdminWeeklyData>
AdminWeeklyData             - label, count
```

### 배운 것

**JPQL vs Native Query**

- JPQL: Entity 클래스명/필드명 기준 (`FROM Score s`)
- Native Query: 실제 테이블명/컬럼명 기준 (`FROM scores`) + `nativeQuery = true`
- JPQL은 `LIMIT` 미지원 → `Pageable`로 대체

**JPQL GROUP BY 결과 받기**

```java
// Object[]로 받아서 직접 캐스팅
List<Object[]> results = repository.findGroupByResult();
results.stream().map(row -> new Response(
    (String) row[0],  // 첫번째 컬럼
    (Long) row[1]     // 두번째 컬럼
)).toList();
```

**전주 대비 증감률 계산**

```java
Double weekChangePct = lastWeekCount == 0 ? null
    : Math.round(((double)(thisWeekCount - lastWeekCount) / lastWeekCount) * 100 * 10) / 10.0;
```

**날짜 계산**

```java
// 이번주 월요일 ~ 일요일
LocalDate startOfWeek = LocalDate.now().with(DayOfWeek.MONDAY);
LocalDate endOfWeek = LocalDate.now().with(DayOfWeek.SUNDAY);

// 지난주
LocalDate startOfLastWeek = startOfWeek.minusWeeks(1);
LocalDate endOfLastWeek = endOfWeek.minusWeeks(1);

// 최근 30일
LocalDate thirtyDaysAgo = LocalDate.now().minusDays(30);
```
