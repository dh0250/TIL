# 📝UseMemo

## 🔧 전체 코드

```html
<script type="text/babel">

        const { useState, useEffect, useMemo } = React;

        const hardCalculator = (num) => {
            console.log('어려운 계산 시작');
            for (let i =0; i < 1999999990; i++) {
                return num + 10000;
            }
        }

        const easyCalculator = (num) => {
            console.log('쉬운 계산 시작');
            return num + 1;
        }

        function App() {
            const [hardNumber, setHardNumber] = useState(1);
            const [easyNumber, setEasyNumber] = useState(1);

            const hardSum = useMemo(() => {
                return hardCalculator(hardNumber);
            }, [hardNumber]); // hardNumber가 바뀔 때만 이 함수를 다시 실행

            // const hardSum = hardCalculator(hardNumber);
            const easySum = easyCalculator(easyNumber);

            return (
                <>
                    <h3>어려운 계산기</h3>
                    <input 
                        type="number"
                        value={hardNumber}
                        onChange={e => setHardNumber(parseInt(e.target.value))}
                    />
                    <span> + 10000 = {hardSum}</span>
                    <hr />
                    <h3>쉬운 계산기</h3>
                    <input 
                        type="number"
                        value={easyNumber}
                        onChange={e => setEasyNumber(parseInt(e.target.value))}
                    />
                    <span> + 1 = {easySum}</span>
                </>
        )}

        ReactDOM.createRoot(document.getElementById('root')).render(<App />);
        
    </script>
```

## 🧠 코드 흐름 설명

### 1. hardCalculator (무거운 계산)

- 반복문이 포함된 느린 함수
- 실행될 때마다 콘솔에 "어려운 계산 시작" 출력
- 렌더링될 때마다 실행되면 성능이 떨어짐

### 2. useMemo 사용 이유

```
const hardSum = useMemo(() => {
    return hardCalculator(hardNumber);
}, [hardNumber]);
```

- hardNumber가 바뀔 때만 실행됨
- 이전 결과를 기억해서 재사용
- easyNumber가 바뀌어도 hardCalculator는 다시 실행되지 않음

### 3. easyCalculator (가벼운 계산)

```
const easySum = easyCalculator(easyNumber);
```

- 단순 +1 계산
- 빠르기 때문에 최적화 필요 없음

### 4. useMemo 이해 (렌더링 동작)

### ✔ hardNumber 변경

- hardCalculator 실행됨
- easyCalculator 실행됨

### ✔ easyNumber 변경

- hardCalculator 실행 ❌ (막힘)
- easyCalculator 실행됨

## ⚠️ 주의할 점

### 1. useMemo는 반드시 변수에 담기

값을 저장하지 않으면 의미 없음

### 2. 의존성 배열

```
[hardNumber]
```

이 값이 바뀔 때만 실행됨

### 3. 모든 곳에 쓰면 안 됨

- 무거운 계산에만 사용
- 남용하면 오히려 성능 나빠짐

## 📌 useMemo + useEffect (객체 참조 문제 해결)

### 🔧 전체 코드

```html
<script type="text/babel">

        const { useState, useEffect, useMemo } = React;

        function App() {
            const [isKorea, setIsKorea] = useState(true);
            const [number, setNumber]= useState(0);

            console.log('렌더링...');

            const location = useMemo(() => {
                return { country: isKorea ? '한국' : '외국'}
            }, [isKorea]); // iskorea 값이 바뀔 때만 새로운 객체를 만들어 줘

            // const location = {
            //     country: isKorea ? '한국' : '외국'
            // };

            useEffect(() => {
                console.log('useEffect 호출됨...location이 변경되었습니다.');
            }, [location]);

            return (
                <>
                    <h2>지금 당신이 있는 위치는?</h2>
                    <p>국가: {location.country}</p>
                    <button onClick={() => setIsKorea(!isKorea)}>국가 토글 하기</button>
                    <hr />
                    <h2>좋아하는 숫자를 입력해 주세요</h2>
                    <input 
                        type="number"
                        value={number}
                        onChange={e => setNumber(e.target.value)} />
                </>
        )}

        ReactDOM.createRoot(document.getElementById('root')).render(<App />);
        
    </script>
```

## 🧠 핵심 개념

### 1. 문제 상황 (useMemo 없을 때)

```
const location = {
    contry: isKorea ? '한국' : '외국'
};
```

- 겉보기엔 같은 값이어도 렌더링될 때마다 **새로운 객체 생성됨**

```
useEffect(() => {
    console.log('useEffect 호출됨');
}, [location]);
```

- React는 "값"이 아니라 **참조(주소)**를 비교함
- number만 바꿔도 location 객체는 매번 새로 생성→ useEffect 계속 실행 (불필요)

### 2. useMemo로 해결

```
const location = useMemo(() => {
    return { contry: isKorea ? '한국' : '외국' }
}, [isKorea]);
```

- isKorea가 바뀔 때만 객체 새로 생성
- number 바뀌면 → 기존 객체 재사용

## 🔄 실행 흐름

### ✔ number 변경

- 렌더링 발생 → location 그대로 유지 → useEffect 실행 ❌

### ✔ isKorea 변경

- location 새 객체 생성 → useEffect 실행 ✅

### 2. input 값 타입

```
onChange={e => setNumber(e.target.value)}
```

- 문자열로 들어옴

✔️ 숫자로 쓰려면:

```
onChange={e => setNumber(parseInt(e.target.value))}
```

💡 useMemo는 "값"이 아니라 **객체/배열의 '참조 유지'를 위해서도 사용**

## 🚀 useMemo 사용

- useEffect dependency에 객체/배열 들어갈 때
- 불필요한 effect 실행 막고 싶을 때
- props로 객체 넘길 때 (리렌더 방지)

## 📌 useCallback (함수 재생성 방지)

### 🔧 전체 코드

```html
<div id="root"></div>
    
    <script type="text/babel">

        const { useState, useEffect, useCallback } = React;

        function App() {

            const [number, setNumber] = useState(0);
            const[toggle, setToggle] = useState(false);

            const printNumber = useCallback(() => {
                console.log('current number: ', number);
            }, [number]); // number값이 바뀔 때만 이 함수를 새로 만들어라
            // 함수 안에서 사용하는 state나 props가 있다면, 반드시 의존성 배열에 넣어줘야 한다.

            // const printNumber = () => {
            //     console.log('current number: ', number);
            // }

            useEffect(() => {
                console.log('printNumber 함수가 변경되었습니다.');
            }, [printNumber])

            return (
                <>
                    <input 
                        type="number"
                        value={number}
                        onChange={e => setNumber(parseInt(e.target.value))} 
                    />
                    <button onClick={printNumber}>숫자 출력</button>
                    <hr />
                    <button onClick={() => setToggle(!toggle)}>토글 버튼</button>
                    <p>토글 상태: {String(toggle)}</p> {/*불린값은 문자열로 바꿔야 JSX 문법으로 화면에 표현된다*/}
                </>
            )
        }

        ReactDOM.createRoot(document.getElementById('root')).render(<App />);
        
    </script>
```

## 🧠 핵심 개념

### 1. 기본 문제 (useCallback 없을 때)

```
const printNumber = () => {
    console.log(number);
}
```

- 컴포넌트가 렌더링될 때마다 **함수가 계속 새로 생성됨**

```
useEffect(() => {
    console.log('함수 변경됨');
}, [printNumber]);
```

- 함수는 객체라서 매번 주소가 바뀜
- number 안 바뀌고 toggle만 바꿔도 useEffect 계속 실행됨 ❌

### 2. useCallback으로 해결

```
const printNumber = useCallback(() => {
    console.log('current number: ', number);
}, [number]);
```

- number 바뀔 때만 함수 새로 생성
- toggle 바뀌면 → 기존 함수 재사용

## 🔄 실행 흐름

### ✔ number 변경

- printNumber 새로 생성됨 → useEffect 실행됨 ✅

### ✔ toggle 변경

- printNumber 그대로 유지 → useEffect 실행 안 됨 ❌

## ⚠️ 중요 포인트

### 1. 의존성 배열 필수

```
[number]
```

- 함수 안에서 사용하는 값은 반드시 넣어야 함
- 넣지 않으면 이전 값(클로저 문제) 사용됨

### 2. useCallback의 역할

- 함수 자체를 저장하는 게 아니라 **함수의 "참조(주소)"를 유지**

### 3. 사용해야 할 때

- useEffect dependency에 함수 들어갈 때
- 자식 컴포넌트에 함수 props로 넘길 때 (리렌더 방지)

❌ 필요 없는 경우

- 단순한 함수
- 성능 문제 없는 경우

## 💡 useMemo vs useCallback 차이

| 구분 | useMemo | useCallback |
| --- | --- | --- |
| 저장 대상 | 값 | 함수 |
| 목적 | 계산 결과 캐싱 | 함수 재생성 방지 |

#### 🚨 주의할 점

- useCallback 남용하면 오히려 코드 복잡해짐
- 성능 최적화는 "필요할 때만"

💡 useCallback → 함수를 기억해서 불필요한 재생성을 막음

## 📌 useCallback + 자식 컴포넌트 렌더링 최적화

### 🔧 전체 코드

```html
<div id="root"></div>
    
    <script type="text/babel">

        const { useState, useEffect, useCallback } = React;

        function Square({genSquareStyle}) {
            console.log('자식 컴포넌트(Square) 렌더링');

            // 부모가 준 스타일을 저장할 state
            const [style, setStyle] = useState({});

            useEffect(() => {
                console.log('자식 useEffect 실행(스타일 변경됨)');
                setStyle(genSquareStyle());
            }, [genSquareStyle]);

            return <div style={style}></div>;
        }

        function App() {

            const [size, setSize] = useState(200);
            const [isDark, setIsDark] = useState(false);

            const genSquareStyle = useCallback(() => {
                return {
                    backgroundColor: 'orangered',
                    width: `${size}px`,
                    height: `${size}px`
                };
            }, [size]);

            // size state를 이용해 스타일 객체를 만드는 함수 정의
            // const genSquareStyle = () => {
            //     return {
            //         backgroundColor: 'orangered',
            //         width: `${size}px`,
            //         height: `${size}px`
            //     };
            // };

            return (
                <>
                    <div style={{backgroundColor: isDark? 'black' : 'white'}}>
                        <input 
                            type="range" min="100" max="300"
                            onChange={e => setSize(parseInt(e.target.value))}
                        />
                        <button onClick={() => setIsDark(!isDark)}>테마 변경</button> 
                        <Square genSquareStyle={genSquareStyle}/>   
                    </div>
                </>
            )
        }

        ReactDOM.createRoot(document.getElementById('root')).render(<App />);
    </script>
```

## 📌 useState vs 변수 vs useRef 비교

### 🔧 전체 코드

```html
<div id="root"></div>
    
    <script type="text/babel">

        const { useState, useRef, useEffect } = React;

        function Counter() {
            console.log('Counter 렌더링');

            // 1. useState (값이 바뀌면 컴포넌트를 '리렌더링')
            const [count, setCount] = useState(0);

            // 2. 일반 지역 변수
            let variable = 0;

            // 3. useRef : .current 프로퍼티에 값 저장
            const countRef = useRef(0);

            const increaseCount = () => {
                setCount(count + 1);
            };

            const increaseVariable = () => {
                variable += 1;
                console.log('지역 변수 값: ', variable);
            };

            const increaseCountRef = () => {
                countRef.current += 1;
                console.log('Ref 값: ', countRef.current);
            }

            return (
                <>
                    <h1>State: {count}</h1>
                    <h2>Variable: {variable}</h2>
                    <h3>Ref: {countRef.current}</h3>
                    <button onClick={increaseCount}>State 증가</button>
                    <button onClick={increaseVariable}>variable 증가</button>
                    <button onClick={increaseCountRef}>Ref 증가</button>
                </>
            )
        }

        function App() {
            return (
                <>
                    <Counter />
                </>
            )
        }

        ReactDOM.createRoot(document.getElementById('root')).render(<App />);
        
    </script>
```

## 🧠 핵심 개념

### 1. useState

```
const [count, setCount] = useState(0);
```

- 값이 바뀌면 → 컴포넌트 **리렌더링 발생**
- 화면에 바로 반영됨
- UI 상태 관리용
- 가장 일반적으로 사용

### 2. 일반 변수 (variable)

```
let variable = 0;
```

- 렌더링될 때마다 초기화됨 (0으로 리셋)
- 값 변경해도 화면 반영 안 됨
- 함수 컴포넌트는 렌더링될 때마다 다시 실행됨
- variable도 매번 새로 생성됨

### 3. useRef

```
const countRef = useRef(0);
```

- 값 유지됨 (렌더링 사이에서도 살아있음)
- 값 변경해도 리렌더링 안 일어남
- .current 로 접근
- 값 저장용 (렌더링 영향 없음)

## 🔄 동작 비교

### ✔ State 증가 버튼

- 값 증가
- 리렌더링 발생
- 화면 업데이트됨

### ✔ Variable 증가 버튼

- 값 증가 (콘솔에만 보임)
- 리렌더링 없음
- 화면에는 계속 0

### ✔ Ref 증가 버튼

- 값 증가
- 리렌더링 없음
- state

## 📊비교

| 구분 | 값 유지 | 리렌더링 | 화면 반영 |
| --- | --- | --- | --- |
| useState | O | O | O |
| 변수 | X | X | X |
| useRef | O | X | X |

## 📌 useRef로 input 포커스 제어하기

### 🔧 전체 코드

```html
<div id="root"></div>
    
    <script type="text/babel">

        const { useState, useEffect, useRef } = React;

        function App() {

            const [form, setForm] = useState({
                username: '',
                password: ''
            });

            // useRef로 서랍을 만든다.
            const usernameRef = useRef(); // .current 프로퍼티가진 객체 반환

            useEffect(() => {
                console.log('마운트 되었습니다.');
                console.log(usernameRef.current);
                usernameRef.current.focus();
            }, [])

            const onChangeHandler = (e) => {
                setForm({
                    ...form,
                    [e.target.name]: e.target.value
                })
            }

            const onClickHandler = () => {
                alert(`username: ${form.username}, password: ${form.password}`);
                setForm({ username: '', password: ''});
                // 로그인 버튼 클릭 후에도 다시 username 포커스
                usernameRef.current.focus();
            }

            return (
                <>
                    <input 
                    type="text"
                    name="username"
                    onChange={onChangeHandler}
                    value={form.username}
                    ref={usernameRef}
                />
                <br />
                <input 
                    type="password"
                    name="password"
                    onChange={onChangeHandler}
                    value={form.password}
                />
                <br />
                <button onClick={onClickHandler}>로그인</button>
                </>
            )
        }

        ReactDOM.createRoot(document.getElementById('root')).render(<App />);
        
    </script>
```

## 🧠 핵심 개념

### 1. useRef의 역할

```
const usernameRef = useRef();
```

- DOM 요소를 직접 참조할 수 있는 객체 생성
- usernameRef.current → 실제 input DOM
- 렌더링이 바뀌어도 값 유지됨

### 2. ref 연결

```
<input ref={usernameRef} />
```

- 해당 input 요소가 usernameRef.current에 저장됨

### 3. 포커스 주기

```
usernameRef.current.focus();
```

- 실제 DOM API 사용
- input에 커서 자동 이동

## 🔄 실행 흐름

### ✔ 첫 렌더링 (마운트)

```
useEffect(() => {
    usernameRef.current.focus();
}, []);
```

- 컴포넌트 처음 생성됨
- username input에 자동 포커스

### ✔ 로그인 버튼 클릭

```
setForm({ username: '', password: '' });
usernameRef.current.focus();
```

- 입력값 초기화
- 다시 username으로 커서 이동

## ⚠️ 중요한 포인트

### 1. ref는 렌더링과 무관

- 값 바뀌어도 리렌더링 ❌
- 단순 DOM 접근용

### 2. useEffect에서 사용하는 이유

### 3. 객체 업데이트 패턴

```
setForm({
    ...form,
    [e.target.name]: e.target.value
});
```

- 기존 값 유지하면서 일부만 변경

💡useRef = "DOM을 직접 다루기 위한 통로"

## 📌 Custom Hook (useDelayTimer)

### 🔧 전체 코드

```html
<div id="root"></div>
    
    <script type="text/babel">

        const { useState, useEffect } = React;

        // 커스텀 훅 정의
        // 이름이 'use'로 시작하는 일반 자바스크립트 함수
        // 내부에서 useState와 useEffect를 마음껏 사용할 수 있다.
        function useDelayTimer(delay) {
            // 시간이 지났는지 여부를 저장하는 상태
            const [isReady, setIsReady] = useState(false);

            useEffect(() => {
                console.log(`${delay}ms 타이머 시작!`);

                const timer = setTimeout(() => {
                    setIsReady(true);
                    console.log('타이머 종료! 준비 완료');
                }, delay);

                // 언마운트 될 때 타이머 정리(메모리 누수 방지)
                return () => clearTimeout(timer);
            }, [delay]);

            return isReady;
        }

        function App() {

            const isReady = useDelayTimer(3000);
            
            return (
                <>
                    <h1>Custom Hook</h1>    
                    <hr />
                    { isReady ? 
                        <h2 style={{color: 'blue'}}>준비 완료! 안녕하세요~</h2>
                        : <h2 style={{color: 'red'}}>잠시만 기다려주세요...</h2>
                    }
                </>
            )
        }

        ReactDOM.createRoot(document.getElementById('root')).render(<App />);
        
    </script>
```

### 1. 커스텀 훅이란?

- React 훅 로직을 **재사용하기 위한 함수**
- 이름은 반드시 use 로 시작
- 내부에서 useState, useEffect 사용 가능

### 2. useDelayTimer 동작

```
function useDelayTimer(delay) {
```

- delay(ms)만큼 기다린 뒤 상태를 true로 변경하는 훅

### 3. 상태 관리

```
const [isReady, setIsReady] = useState(false);
```

- 처음에는 false . 시간이 지나면 true로 변경

### 4. 타이머 실행

```
const timer = setTimeout(() => {
    setIsReady(true);
}, delay);
```

- delay(ms) 후 실행됨

### 5. 클린업 (중요)

```
return () => clearTimeout(timer);
```

- 컴포넌트가 사라질 때 타이머 제거
- 메모리 누수 방지
- React에서 매우 중요한 패턴

## 📌 Promise (비동기 처리 개선)

### 🔧 전체 코드

```html
<script>

        /*
        - Promise
        비동기 작업이 미래의 언젠가 끝날 텐데, 
        작업이 끝나면 성공(resolve)했는지 실패(reject)했는지 꼭 알려줄게!
        라고 약속하는 '약속 증서'객체를 먼저 반환하는 개념이다.
        */

        function increase(number) {
            const promise = new Promise((resolve, reject) => {
                setTimeout(() => {
                    const result = number + 10;

                    if (result > 50) {
                        // 실패 -> reject 함수 호출하며 에러 전달
                        const e = new Error('숫자가 너무 큽니다!');
                        return reject(e);
                    }

                    // 성공 -> 계산된 결과와 함께 resolve 처리
                    resolve(result);
                }, 1000);
            });
            return promise;
        }

        increase(0)
            .then(number => { // increase가 성공하면 그 결과(number)를 받아서 실행
                console.log(number);
                return increase(number);
            })
            .then(number => {
                console.log(number);
                return increase(number);
            })
            .then(number => {
                console.log(number);
                return increase(number);
            })
            .then(number => {
                console.log(number);
                return increase(number);
            })
            .then(number => {
                console.log(number);
                return increase(number);
            })
            .catch(error => { // .then() 체인 중간에 reject 하면 여기서 잡힘
                console.error(error);
            })
            .finally(() => { // 성공하든 실패하든 상관없이 무조건 마지막에 실행
                console.log('모든 작업이 끝났습니다.');
            })
    </script>
```

## 🧠 핵심 개념

### 1. Promise란?

- "비동기 작업의 결과를 약속하는 객체"
- 성공 → resolve
- 실패 → reject

### 2. Promise 생성

```
new Promise((resolve, reject) => {
```

- 비동기 작업을 감싸는 객체

### 3. 성공 처리 (resolve)

```
resolve(result);
```

- then으로 결과 전달됨

### 4. 실패 처리 (reject)

```
reject(new Error('숫자가 너무 큽니다!'));
```

- catch로 전달됨

## 🔄 실행 흐름

### increase(0)부터 시작

1초마다 실행:

- 10
- 20
- 30
- 40
- 50
- 60 → ❌ 에러 발생 (50 초과)

## ⛓ then 체이닝

```
.then(number => {
    return increase(number);
})
```

- 이전 결과를 다음 작업으로 넘김
- 콜백 지옥 없이 "순차 실행" 가능

## ❌ 에러 발생 시

```
if (result > 50)
```

- 60이 되는 순간 reject 실행

## 🛑 catch

```
.catch(error => {
    console.error(error);
})
```

- 중간에 에러 나면 여기서 처리
- 이후 then 실행 안 됨

## 🔚 finally

```
.finally(() => {
    console.log('모든 작업이 끝났습니다.');
})
```

- 성공 / 실패 상관없이 무조건 실행

## ⚠️ 중요한 포인트

### 1. return 중요

```
return increase(number);
```

- 이걸 해야 다음 then으로 값 전달됨

### 2. Promise 체인의 장점

✔️ 가독성 ↑

✔️ 순차 실행 가능

✔️ 에러 처리 쉬움

## 💡 콜백 vs Promise

| 구분 | 콜백 | Promise |
| --- | --- | --- |
| 가독성 | 나쁨 | 좋음 |
| 구조 | 중첩 | 체이닝 |
| 에러 처리 | 어려움 | 쉬움 |

💡 Promise = "비동기 작업을 깔끔하게 연결하는 방법"

## 📌 async / await (비동기 코드의 동기화)

### 🔧 전체 코드

```html
<script>

        /*
        - Promise
        비동기 작업이 미래의 언젠가 끝날 텐데, 
        작업이 끝나면 성공(resolve)했는지 실패(reject)했는지 꼭 알려줄게!
        라고 약속하는 '약속 증서'객체를 먼저 반환하는 개념이다.
        */

        function increase(number) {
            const promise = new Promise((resolve, reject) => {
                setTimeout(() => {
                    const result = number + 10;

                    if (result > 50) {
                        // 실패 -> reject 함수 호출하며 에러 전달
                        const e = new Error('숫자가 너무 큽니다!');
                        return reject(e);
                    }

                    // 성공 -> 계산된 결과와 함께 resolve 처리
                    resolve(result);
                }, 1000);
            });
            return promise;
        }

        async function run() {
            console.log('작업 시작...');

            // await: increase(0) Promise가 끝날 때까지 기다렸다가,
            //        resolve된 결과값을 result 변수에 할당
            
            try {
                let result = await increase(0);
                console.log(result);

                result = await increase(result);
                console.log(result);

                result = await increase(result);
                console.log(result);

                result = await increase(result);
                console.log(result);

                result = await increase(result);
                console.log(result);

                result = await increase(result);
                console.log(result);

            } catch (error) { // try 블록 안에서 reject 되면 catch 블록 실행
                console.error(error);
            } 
            console.log('모든 작업이 끝났습니다.');

        }

        run();

    </script>
```
