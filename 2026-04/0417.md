# 📝iteration

### rendering-list

- 과일 목록 (FruitList)
    - 배열 데이터를 map()으로 순회하여 리스트 생성함
    - 각 요소에는 반드시 key 필요
- 회원 목록 (UserList)
    - 객체 배열을 map()으로 렌더링
    - 각 객체의 id를 key로 사용 (권장 방식)

```jsx
<div id="root"></div>
    
    <script type="text/babel">
        
        const fruits =['사과', '바나나', , '포도', '오렌지'];

        function FruitList() {
            return (
                <div>
                    <h2>과일 목록</h2>    
                    <ul>
                        {fruits.map((fruit, index) => (
                            <li key={index}>{fruit}</li>
                        ) )}
                    </ul>
                </div>
            )
        }

        function UserList() {
            return (
                <>
                    <h2>회원 목록</h2>
                    <ul>
                        {users.map(user => (
                            <li key={user.id}>
                                {user.name}
                            </li>
                        ))}
                    </ul>
                </>
            );
        }

        const users = [
            { id: 1, name: '홍길동' },
            { id: 2, name: '유관순' },
            { id: 3, name: '이순신' }
        ]

        function App() {
            return (
                <>
                    <FruitList />
                    <UserList />
                </>
            )
        }
        ReactDOM.createRoot(document.getElementById('root')).render(<App />);
    </script>
```

## ✏️list-add-remove

- 리스트 동적으로 추가, 삭제 (입력값을 state로 관리하고, 배열에 추가/삭제하면서 리스트를 동적으로 렌더링)

#### 상태(state) 3개

```html
const [inputValue,setInputValue]=useState('');
const [names,setNames]=useState([{id:1, text:'판다'}]);
const [nextId,setNextId]=useState(2);
```

- inputValue : 입력창 값
- names : 이름 목록 (객체 배열)
- nextId : 다음에 추가될 id

#### 입력 처리

```jsx
consthandleChange= (e) =>setInputValue(e.target.value);
```

- input에 입력할 때마다 값 업데이트

#### 추가 기능

```jsx
consthandleAdd= () => {
if (inputValue.trim()==='')return;

constnewName= {id:nextId, text:inputValue};

setNames([...names,newName]);// 기존 + 새 항목
setNextId(nextId+1);// id 증가
setInputValue('');// 입력창 초기화
}
```

핵심 포인트:

- 빈 값 방지 (trim)
- 배열은 **불변성 유지** (…names)

#### 삭제 기능

```jsx
consthandleRemove= (idToRemove) => {
constupdateNames=names.filter(name =>name.id!==idToRemove);
setNames(updateNames);
}
```

핵심 포인트:

- filter() 로 특정 id 제거
- 기존 배열 수정 ❌ → 새 배열 생성 ✔

#### 화면 렌더링

```html
<inputvalue={inputValue}onChange={handleChange}/>
<button onClick={handleAdd}>추가</button>

<ul>
    {names.map(name => (
<likey={name.id}>
            {name.text}
<buttononClick={() =>handleRemove(name.id)}>삭제</button>
</li>
    ))}
</ul>
```

핵심 포인트:

- map() 으로 리스트 출력
- key는 반드시 필요 (여기선 id)
- 삭제 버튼에 id 전달

# 📝useEffect

## ✏️useEffect

- 함수형 컴포넌트에서 렌더링 된 이후 시점에 수행 할 내용이 필요한 경우 사용할 수 있는 기능을 hooks로 제공
- 함수형 컴포넌트는 **함수가 실행되는 순간 = 렌더링**
- 렌더링이 끝난 후 실행됨
- 의존성 배열 없으면 → 매 렌더링마다 실행
- return 아래 코드는 실행 안 됨 (함수 종료됨)

### 1. Counter 컴포넌트

#### 상태(state)

- count: 클릭 횟수
- otherCount: 또 다른 카운트 (비교용)

```jsx
function Counter() {

            const [count, setCount] = useState(0);
            const [otherCount, setOtherCount] = useState(0);

            // useEffect는 첫 마운트 직후에는 무조건 한번 실행 된다

            // 1. 의존성 배열 생략 : 매 렌더링마다 실행
            useEffect(() => {
                //랜더링 '완료된 후' 실행된다
                console.log('렌더링될 때마다 실행')
                document.title = `넌 ${ count }번 클릭했어`;
            });

            // 2. 빈 의존성 배열 [] : Mount(첫 렌더링) 시에만 실행
            // 주로 초기 데이터 로딩(API호출) 등에 사용
            useEffect(() => {
                console.log('컴포넌트 처음 화면에 나타남(Mount)');
            }, []);

            // 3. 의존성 배열에 특정 값 지정 [count] : 해당 값이 변경될 때만 실행
            useEffect(() => {
                console.log(`count 값이 ${count}로 변경되어, 이 useEffect가 실행`);
            }, [count]);
            
            console.log('Counter 컴포넌트 렌더링');

            return (
                <div>
                    <h2>단순 카운터</h2>    
                    <p>{count}번 클릭 했습니다.</p>
                    <button onClick={() => setCount(count + 1)}>클릭</button>
                    <p>Other Count : {otherCount}</p>
                    <button onClick={() => setOtherCount(otherCount + 1)}>클릭</button>
                </div>
                
            );
        }
```

### 🔗useEffect 3가지 패턴

#### 1) 의존성 배열 생략

- useEffect(() => {...})
- 실행 시점: **렌더링 될 때마다**
- 역할: document.title 변경, 콘솔 출력

#### 2) 빈 배열 []

- useEffect(() => {...}, [])
- 실행 시점: **처음 렌더링(Mount) 한 번만**
- 역할: 초기 실행 (예: API 호출 자리)

#### 3) 특정 값 [count]

- useEffect(() => {...}, [count])
- 실행 시점: **count가 바뀔 때만**
- 역할: count 변화 감지 (otherCount 바뀔 땐 실행 안됨)

### 🔗렌더링 흐름

버튼 클릭 → state 변경 → 렌더링 → useEffect 실행

그리고 중요한 점:

- console.log('Counter 컴포넌트 렌더링') → 렌더링 "중"
- useEffect → 렌더링 "끝난 후"

### 2. Timer 컴포넌트

```jsx
function Timer() {

            useEffect(() => {
                console.log('타이머가 시작되었습니다(Mount');

                const timerId = setInterval(() => {
                    console.log('...1초 경과...');
                }, 1000);

                // Cleanup 함수: useEffect가 반환하는 함수이다
                // 컴포넌트가 화면에서 사라지기 직전(Unmount)에 호출된다
                return () => {
                    console.log('타이머를 정리합니다.(Unmount)');
                    clearInterval(timerId); // 생성했던 타이머 제거
                }
            }, []);
            return (
                <h3>타이머가 동작 중입니다. (콘솔 확인)</h3>
            )
        }
```

#### 🔗Mount 시 실행

- useEffect(..., [])
- setInterval :  1초마다 로그 출력

#### 🔗Cleanup 함수 (핵심)

useEffect에서 return한 함수는 컴포넌트가 사라질 때(Unmount) 실행됨

하는 일: clearInterval(timerId) ⇒ 타이머 정리

### 4. App 컴포넌트

```jsx
function App() {
            const [showTimer, setShowTimer] = useState(true);

            return (
                <>
                    <Counter />
                    <hr />
                    <button onClick={() => setShowTimer(!showTimer)}>타이머 보이기/숨기기</button>
                    {showTimer && <Timer />}
                </>
            )
        }
```

#### 상태

- showTimer: Timer 보이기 여부

#### 동작

- 버튼 클릭 → showTimer 토글
- true → Timer 나타남 (Mount)
- false → Timer 사라짐 (Unmount)

#### 🔗정리

- useEffect는 "렌더링 이후" 실행
- 의존성 배열에 따라 실행 타이밍이 달라짐 useEffect(() ⇒ {} ,  )
    - useEffect(() ⇒ {} ) → 매번 실행
    - useEffect(() ⇒ {} ,  [])  → 처음만
    - useEffect(() ⇒ {} ,  [값]) → 값 바뀔 때만
- return 함수는 "정리(cleanup)" 용도 (Unmount 시 실행)
