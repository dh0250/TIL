# 🧢 fitory

### Supabase 기반 Next.js 패션 커머스 플랫폼

---

# 📌 1. 프로젝트 개요

* Next.js (App Router) 기반으로 개발함
* Supabase(PostgreSQL)로 백엔드 및 DB 구성함
* Zustand로 전역 상태 관리함
* 브랜드/카테고리 기반 상품 탐색 + 장바구니/좋아요/주문 기능 구현함

---

# 🛍 2. 주요 기능

## A. 상품 탐색

* 메인 페이지에서 배너, 신상품, 랭킹, 브랜드/카테고리 섹션 노출함
* 카테고리/브랜드 기준으로 상품 필터링 조회 가능함
* 상품 상세 페이지에서 이미지, 가격, 리뷰 조회 및 작성 가능함

---

## B. 쇼핑 기능

* 장바구니에 상품 추가, 수량 변경, 삭제 및 총 금액 계산 가능함
* 좋아요(찜) 기능으로 관심 상품 저장 가능함
* 구매 상품 기반 리뷰 작성 및 관리 가능함

---

## C. 사용자 / 주문

* 로그인/로그아웃 및 인증 상태 관리함
* 마이페이지에서 사용자 정보 및 메뉴 제공함
* 주문 내역 조회 및 주문 상세 정보 확인 가능함

---

# 📄 3. 페이지 구조

| 페이지                           | 설명                   | 컴포넌트                                                                          |
| ----------------------------- | -------------------- | ----------------------------------------------------------------------------- |
| 🏠 `/`                        | 메인 화면 (프로모션 + 상품 노출) | HeroBanner, NewArrivalsSection, RankingSection, BrandSection, CategorySection |
| 📦 `/products/[productId]`    | 상품 상세 + 구매/좋아요       | ProductDetail, ProductReviews, ReviewForm, AddToCartButton, LikesButton       |
| 🗂 `/category`                | 카테고리 목록              | CategoryItem                                                                  |
| 🛍 `/category/[categoryId]`   | 카테고리별 상품 리스트         | ProductCard                                                                   |
| 🛒 `/cart`                    | 장바구니 + 결제 금액 계산      | CartItem, OrderSummarySection                                                 |
| ❤️ `/likes`                   | 좋아요 상품 목록            | ProductCard                                                                   |
| 👤 `/mypage`                  | 사용자 정보 및 메뉴          | ProfileSection, MyPageMenu                                                    |
| 📑 `/mypage/orders`           | 주문 내역                | OrderCard                                                                     |
| 🔍 `/mypage/orders/[orderId]` | 주문 상세                | OrderItemCard, OrderSummarySection                                            |

---

# 🧠 4. 상태 관리 (Zustand)

## 👤 useUserStore

* 사용자 정보 및 로그인 상태 관리함
* session, user 상태 저장함

---

## 🛒 useCartStore

* 장바구니 상품 목록 및 수량 관리함
* 상품 추가/삭제/수량 변경 처리함
* 총 금액 및 수량 계산함

---

## ❤️ useLikesStore

* 좋아요(위시리스트) 상태 관리함
* 좋아요 토글 및 여부 판단 처리함
* 로그인 시 DB와 동기화함

---

## 🧩 useUiStore

* 전역 UI 상태 관리함
* 모달 열기/닫기 및 상태 제어함

---

# ⚙️ 5. 기술적 특징

* lib/data로 데이터 접근 로직 분리하여 관리함
* Supabase migration으로 DB 스키마 변경 이력 관리함
* UUID → Integer ID 구조로 최적화 진행함
* 컴포넌트 단위로 UI 쪼개서 재사용성 높임

---

# 🔄 6. 데이터 흐름

* Supabase Client로 서버/클라이언트 양쪽에서 DB 접근함
* 도메인별 데이터 로직을 모듈화하여 관리함
* 로그인 필요한 페이지는 Auth Guard로 접근 제한함
