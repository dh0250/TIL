## Git & GitHub

### Git & GitHub 기초
--------------------------------------

git - 소스코드 버전 관리 시스템, 협업 프로젝트를 위한 버전관리 시스템

명령어 모음

- Git Bash : CLI 방식으로 Git을 사용할 수 있는 환경
- git bash :명령어
- git init : 해당 폴더를 깃으로 버전관리하기위해 씀
- git add hello.txt :작업 디렉토리(working directory) 상의 변경 내용을 스테이징 영역(staging area)에 추가함.
- git revert : reset과 다르게 커밋을 삭제하는 것이 아닌 커밋을 추가함
- git push origin feature/myname : 브랜치에 코드 변경 이력을 origin이라는 원격저장소에 올림
- git pull upstream main : 받아오기
- git commit -m “메세지 등록” : 버전 관리를 통해 생성 된 파일
- git branch feature/greeting : 브랜치 생성
- git switch feature/greeting : 브랜치 변경
- git merge feature/conflict : 브랜치 병합
- git switch -c feature/myname : 브랜치 생성 & 스위치
- git clone : 원격 저장소를 로컬에 복제하는 것
- git remote add upstream https://github.com/*****/git-practice.git : folk 한 repository의 주소를 가져와 원격 저장소로 설정
- pull request : 수정코드 적용 요청
- push/pull -folk 방식 : 오픈소스에 기여할 때 사용
