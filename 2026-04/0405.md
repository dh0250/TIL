## 📝event-handler

### ✏️이벤트 핸들러 등록 및 제거

- 이벤트 핸들러 어트리뷰트
    - HTML 태그에 어트리뷰트 직접 작성, HTML과 JavaScript 코드가 섞여서 유지보수가 어렵기 때문에 사용하지 않는 것이 좋음

```html
<button onclick="alert('첫 번째 버튼 클릭!');">방법 1 버튼</button>
```

- 이벤트 핸들러 프로퍼티
    - 요소 객체의 ‘onclick’ 프로퍼티에 함수를 직접 할당
    - 단점: 하나의 이벤트에 오직 하나의 임무만 부여

```html
<button id="btn2">방법 2 버튼</button>
<script>
    const $btn2 = document.getElementById('btn2');

    $btn2.onclick = function() {
        console.log('온클릭 이벤트 발생!');
    }

    $btn2.onclick = function() {
        console.log('두번째 이벤트 발생!');
    }
</script>
```

- addEventListener
    - element.addEventListener('이벤트종류', 실행할함수)` 형태로 사용
    - 가장 유연하고 현대적인 표준 방식

```html
<button id="btn3">방법 3 버튼</button>
<script>
   const $btn3 = document.getElementById('btn3');

   // 하나의 이벤트에 여러 개의 핸들러(클릭했을때 실행될 함수)를 부여할 수 있다.
   $btn3.addEventListener('click', function() {
        console.log('addEventListner = 첫 번째 이벤트');
   });

   $btn3.addEventListener('click', function() {
        alert('두 번째 이벤트 수행!');
   })
</script>
```

### ✏️이벤트 핸들러 제거

- 제거하려는 함수는 반드시 이름이 있거나 변수에 담겨있어야 함

```html
<button id="btn4">추가/제거 버튼</button>
<button id="addBtn">임무 추가</button>
<button id="removeBtn">임무 제거</button>

<script>
    const $btn4 = document.getElementById('btn4');
    const $addBtn = document.getElementById('addBtn');
    const $removeBtn = document.getElementById('removeBtn');

    // 제거하기 위한 핵심: 함수를 변수에 저장하여 '참조'를 유지
    const mission = () => {
        alert('특수 임수 수행 완료!');
    };
    
    // 임무 추가 버튼을 누르면 $btn4에 mission 이벤트 핸들러 함수 등록
    $addBtn.addEventListener('click', () => {
        $btn4.addEventListener('click', mission);
        console.log('btn4번에 임무가 추가 되었습니다.');
    });

    // 임수 제거 버튼을 누르면 $btn4에서 mission 이벤트 제거
    $removeBtn.addEventListener('click', () => {
        $btn4.removeEventListener('click', mission);
        console.log('btn4번에 임무가 제거 되었습니다.');
    });

    // 익명 함수는 추가는 되지만, 제거할 방법이 없다.
    // $btn4.addEventListener('click', () => alert('이 이벤트는 제거할 수 없다!'));
    // $btn4.removeEventListener('click', () => alert('이 이벤트는 제거할 수 없다!'));

</script>
```

## 📝이벤트 객체와 this

### ✏️이벤트 객체: 상세한 '사건 보고서’

- 이벤트가 발생하면, 브라우저는 해당 이벤트의 모든 정보가 담긴 '이벤트 객체'를 생성
- 이 객체는 이벤트 핸들러(우리가 등록한 함수)의 첫 번째 인자로 자동으로 전달

```html
<h2 id="msg">여기를 클릭하면, 클릭한 곳의 마우스 좌표를 알려드립니다.</h2>

<script>
   const $msg = document.getElementById('msg');

   // 핸들러 함수의 첫 번째 매개변수 (이름은 e, event 등 자유롭게)로 이벤트 객체를 받는다.
   document.addEventListener('click', function(e) {
        console.log(e);

        $msg.textContent = `마우스 좌표 -> X: ${e.clientX}, Y: ${e.clientY}`;

        // e.target: '실제로' 이벤트가 시작된 요소
        // e.currentTarget: 이벤트를 등록한 요소
        console.log('실제 클릭 대상(e.target):', e.target);
   })
</script>
```

### ✏️핸들러 내부의 ‘this’ : '사건 당사자'는 누구인가?

- this`는 '누가 이 함수를 호출했는가'에 따라 가리키는 대상이 달라져서 혼란을 줄 수 있음
- 특히 화살표 함수를 쓸 때 주의

```html
<button id="btn">클릭해보세요</button>

<script>
    const $btn = document.getElementById('btn');

    // 1. 일반 function 함수 사용 시
    $btn.addEventListener('click', function(e) {
        console.log('일반 function');
        //이때의 'this'는 이벤트를 부착한 요소($btn)를 가리킨다.
        console.log('this:', this); 
        // e.currentTarget 또한 이벤트를 부착한 요소($btn)를 가리킨다.
        console.log('e.currentTarget:', e.currentTarget); 
        console.log(this === e.currentTarget);
        this.style.backgroundColor = 'green';
    })

    // 2. 화살표 함수 사용 시
    $btn.addEventListener('mouseover', (e) => {
        console.log('화살표 함수');
        // 화살표 함수의 'this'는 바깥 스코프(여기서는 전역 객체 window)를 가리킨다.
        console.log('this:', this);
        console.log('e.currentTarget:', e.currentTarget); 
        console.log(this === e.currentTarget);
    })
</script>
```

- function과 화살표 함수 사이에서 this가 다르게 동작하여 생기는 혼란을 피하기 위해, e.currentTarget을  사용하는 것이 좋음

```html
<button id="best-practice-btn">Best Practice</button>
<script>
   const $bestBtn = document.getElementById('best-practice-btn');

   $bestBtn.addEventListener('click', (e) => {
        const btn = e.currentTarget;

        btn.textContent = '클릭되었습니다';
        btn.style.backgroundColor = 'skyblue';
        
   })
</script>
```

## 📝이벤트 전파와 위임

- 이벤트 버블링 : 하위 요소에서 발생한 이벤트는, 그 파문이 부모 요소를 따라 최상위까지 올라가는 현상

```html
<ul id="parent"> 부모 (ul)
    <li id="child"> 자식 (li) </li>
</ul>

<script>
    const $parent = document.getElementById('parent');
    const $child = document.getElementById('child');
    
    $parent.addEventListener('click', (e) => {
            alert('부모가 클릭(ul)을 감지했습니다.');
            console.log('부모 핸들러');

            // 이벤트가 최초로 발생한 지점 (parent / child버튼 누르면 child)
            console.log('e.target: ', e.target);
            
            // 이벤트 리스너가 부착된 요소 (parent)
            console.log('e.currentTarget: ', e.currentTarget);
    })

    $child.addEventListener('click', (e) => {
            alert('자식(li)이 클릭되었습니다.');
    })
</script>
```

- 이벤트 위임 : 부모 요소에 이벤트 핸들러를 단 하나만 등록하여 모든 자식의 이벤트를 처리

```html
<ul id="drink-list">
    <li>커피</li>
    <li>콜라</li>
    <li>우유</li>
</ul>
<button id="add-btn">음료 추가</button>

<script>
    const $drinkList = document.getElementById('drink-list');
    const $addBtn = document.getElementById('add-btn');

    function highlight(element) {
        element.classList.toggle('highlight');
    }

    // 이벤트 위임 적용
    // 각 li가 아닌, 부모인 ul에 이벤트 핸들러 딱 한번 등록
    $drinkList.addEventListener('click', (e) => {
        console.log('클릭된 요소(e.target):', e.target);

        if (e.target.matches('li')) {
            highlight(e.target);
        }
    });
    
    // 이벤트 위임의 가장 큰 장점은 
    // 나중에 동적으로 추가되는 자식 요소에도 이벤트 핸들러가 자동으로 동작한다.
    $addBtn.addEventListener('click', () => {
        const $newLi = document.createElement('li');
        $newLi.textContent = '사이다';
        $drinkList.appendChild($newLi);
    })
</script>
```

## 📝이벤트 제어

### ✏️preventDefault와 stopPropagation

- preventDefault : 브라우저 자동 반응 멈추기

```html
<form id="login-form">
    <div class="form-group">
        <label for="username">아이디:</label>
        <input type="text" id="username">
    </div>
    <button type="submit">로그인</button>
    <p class="error-message"></p>
</form>

<script>
    const $loginForm = document.getElementById('login-form');
    const $usernameInput = document.getElementById('username');
    const $errorMessage = document.querySelector('.error-message');

    $loginForm.addEventListener('submit', (e) => {
        // '자동 새로고침' 기본 동작 멈추기
        e.preventDefault();
        console.log('자동 새로고침을 막았습니다.');

        // 유효성 검사 수행
        if ($usernameInput.value.trim() === '') {
            $errorMessage.textContent = '아이디를 입력해주세요!';
        }  else {
            $errorMessage.textContent = '';
            console.log(`서버로 제출할 아이디: ${usernameInput.value}`);
            // 여기에 실제 서버로 데이터를 보내는 코드 삽입
            alert('로그인 성공!');
        }
    });

</script>
```

- stopPropagation : 이벤트의 ‘연쇄 보고’(버블링) 멈추기

```html
<div id="modal-background" class="modal-background">
    <div id="modal-content" class="modal-content">
        <h3>모달 창</h3>
        <p>이 흰색 영역을 클릭해도 모달은 닫히지 않아야 합니다.</p>
        <p>하지만 바깥 회색 영역을 클릭하면 닫힙니다.</p>
    </div>
</div>

<script>
    const $modalBg = document.getElementById('modal-background');
    const $modalContent = document.getElementById('modal-content');

    $modalBg.addEventListener('click', () => {
        console.log('배경 클릭됨 -> 모달을 닫습니다.');
        $modalBg.style.display = 'none';
    });

    $modalContent.addEventListener('click', (e) => {
        // 이벤트 버블링을 멈춘다!!
        e.stopPropagation();
        console.log('흰색 자식 클릭됨 -> 버블링 막아 부모에게 가지 않는다');

        // 내용 부분 클릭했을 때만 수행할 다른 작업을 여기에 추가할 수 있다.
        
    })
</script>
```

## 📝마우스와 키보드 이벤트

### ✏️마우스 이벤트

- `click`(클릭), `dblclick`(더블클릭), `mousedown`(누를때), `mouseup`(뗄때), `mousemove`(움직일때)

```html
<button id="mouse-btn">여기를 클릭/우클릭 해보세요</button>
<div id="mouse-area" class="area"></div>
<script>
    const $mouseBtn = document.getElementById('mouse-btn');
    const $mouseArea = document.getElementById('mouse-area');

    // click: 마우스 왼쪽 버튼을 눌렀다 뗐을 때 발생하는 이벤트
    $mouseBtn.addEventListener('click', (e) => {
        $mouseArea.innerHTML += 'click 이벤트 발생!<br>';
    })

    // mousedown: 마우스 버튼을 누르는 순간 발생
    // e.button -> 0: 왼쪽, 1: 가운데(휠), 2: 오른쪽
    $mouseBtn.addEventListener('mousedown', (e) => {
        $mouseArea.innerHTML += `mousedown, button=${e.button}<br>`;
    });

    // contextMenu: 마우스 오른쪽 버튼을 눌렀을 때
    $mouseBtn.addEventListener('contextmenu', (e) => {
        e.preventDefault(); // 브라우저 기본 메뉴가 뜨는 것을 막는다.
        $mouseArea.innerHTML += `contextmenu 이벤트 발생! (기본 메뉴 차단)<br>`;
    })
</script>
```

### ✏️키보드 이벤트

```html
    - keydown: 키를 누르는 순간 (누르고 있는 동안 계속 발생)
    - keyup: 키에서 손을 떼는 순간
    - event.key: 입력된 문자를 알려줌 (ex: 'a', 'A', 'Enter')
    - event.code: 실제 물리적인 키 코드를 알려줌 (ex: 'KeyA', 'Enter')
```

```html
<input type="text" id="keyboard-input" placeholder="아무 키나 입력하세요">
<div id="keyboard-area" class="area"></div>
<script>
    const $keyboardInput = document.getElementById('keyboard-input');
    const $keyboardArea = document.getElementById('keyboard-area');

    $keyboardInput.addEventListener('keydown', (e) => {
        $keyboardArea.innerHTML = `
            keydown 이벤트 발생!<br>
            e.key: ${e.key}<br>
            e.code: ${e.code}`;

        // 실무 예 : 'Enter' 키를 눌렀을 때 검색 실행
        if (e.key === 'Enter') {
            console.log('검색을 실행합니다!');
            // 실제 검색 함수를 여기에 호출
        }
    });

    $keyboardInput.addEventListener('keyup', (e) => {
        console.log('keyup 이벤트 발생!');
    });
</script>
```

## 📝폼(form) 이벤트

### ✏️폼 이벤트 다루기

- 사용자가 입력한 데이터를 받고, 검증하고, 서버에 제출하는 웹의 핵심 기능

```html
<form name="joinForm" action="https://example.com/join" method="GET">
    <div class="form-group">
        <label for="username">이름</label>
        <input type="text" name="username" id="username" value="홍길동">
    </div>

    <div class="form-group">
        <label>관심사 (2개 이상 선택)</label><br>
        <input type="checkbox" name="hobby" value="exercise"> 운동
        <input type="checkbox" name="hobby" value="music"> 음악
        <input type="checkbox" name="hobby" value="movie"> 영화
    </div>

    <div class="form-group">
        <label for="introduce">자기소개 (<span id="char-counter">0</span>/100자)</label>
        <textarea name="introduce" id="introduce" rows="4" maxlength="100"></textarea>
    </div>

    <button type="submit">회원가입</button>
    <p class="error"></p>
</form>

<script>
        // 폼과 요소들 선택하기
    const $form = document.forms.joinForm;   // name 속성 기준
    const $username = $form.username;        // name 속성 기준
    // const $username = document.querySelector('[name="username"]');
    const $hobbies = $form.hobby; // name이 같은 요소들은 컬렉션(NodeList)으로 가져옴
    const $introduce = $form.introduce;
    const $counter = document.getElementById('char-counter');
    const $error = document.querySelector('.error');

    // 폼 요소 이벤트 활용
    // focus & blur : 요소에 포커스가 가거나 벗어났을 때 
    $username.addEventListener('focus', (e) => e.target.classList.add('focused'));
    $username.addEventListener('blur', (e) => e.target.classList.remove('focused'));

    // input: 사용자가 값을 변경할 때마다(실시간!)
    $introduce.addEventListener('input', (e) => {
        const currentLength = e.target.value.length;
        $counter.textContent = currentLength;
    });

    // 폼 제출(submit) 이벤트 제어
    $form.addEventListener('submit', (e) => {
        console.log('submit 이벤트 발생!');

        // 1. 이름 유효성 검사
        if($username.value.trim().length < 2) {
            e.preventDefault(); // 기본 동작(서버 제출)을 막는다.
            $error.textContent = '이름은 2글자 이상 입력해주세요.';
            return; // 함수 종료
        } 

        // 2. 관심사 유효성 검사
        let checkedCount = 0;
        $hobbies.forEach(hobby => {
            if (hobby.checked) {   // check 상태일 경우 true 반환
                checkedCount++;
            }
        });

        if (checkedCount < 2) {
            e.preventDefault(); // 서버 제출 막는다.
            $error.textContent = '관심사는 2개 이상 선택해주세요.';
            return;
        };

        // 모든 유효성 검사를 통과하면, 기본 동작(제출)이 실행됨
        $error.textContent = '';
        alert('회원가입 성공! 서버로 제출합니다.'); 
    });
</script>
```

## 📝비동기 프로그래밍

- 블로킹(Blocking) : '동기' 처리는 앞선 작업이 끝날 때까지 다음 작업이 기다리는 방식

```html
<button id="sync-btn">동기 작업 시작 (3초 멈춤!)</button>
<button id="test-btn">반응 테스트</button>
<script>
    function sleep(delay) {
        const delayUntil = Date.now() + delay;
        while(Date.now() < delayUntil);
    };

    document.getElementById('sync-btn').addEventListener('click', () => {
        console.log('동기 작업 시작...');
        sleep(3000); // 이 함수가 실행되는 3초 동안은 멈춤
        console.log('동기 작업 완료!');
    });

</script>
```

- 타이머 함수 (`setTimeout`과 `setInterval`)

```html
<h4>setTimeout: "조금 이따가, 딱 한 번만 해줘!"</h4>
<script>
    const timerId = setTimeout(() => {
        console.log('3초가 지났습니다.');
    }, 3000);

    console.log('타이머는 예약 되었습니다.');

    clearTimeout(timerId); // 예약을 취소할 수도 있다.
   
</script>

<h4>setInterval: "계속 반복해서 해줘!" (feat. clearInterval)</h4>
<script>
    let count = 1;
    const intervalId = setInterval(() => {
        console.log(`${count}초`);

        // 5초가 되면 반복을 멈춥니다.
        if(count++ === 5) {
            clearInterval(intervalId);
            console.log('반복이 중지되었습니다.');
        }
    }, 1000);
</script>
```

- 이벤트 루프
    - 호출 스택(Call Stack): 코드가 실행되는 작업 공간
    - Web API: setTimeout 같은 오래 걸리는 작업은 브라우저에 맡겨 처리
    - 태스크 큐 (Task Queue): Web API에서 끝난 작업(콜백 함수)이 순서대로 대기하는 줄
    - 이벤트 루프 (Event Loop): Call Stack이 비었을 때, Task Queue의 작업을 Call Stack으로 옮겨 실행

## 📝브라우저 제어 (BOM)

- BOM (Browser Object Model)
    - `window` 객체를 통해 자바스크립트는 HTML 문서(`document`)를 넘어 브라우저 창 자체를 제어하고, 사용자와 직접 대화
- 사용자와 대화하기

```html
<button id="confirmBtn">진행 여부 묻기</button>
<button id="promptBtn">이름 입력받기</button>
<p id="message"></p>
<script>
    //alert('페이지 로딩 완료!');

    const $confirmBtn = document.getElementById('confirmBtn');
    const $promptBtn = document.getElementById('promptBtn');
    const $message = document.getElementById('message');

    // confirm: 확인/취소 질문. boolean 값 반환(true/false)
    $confirmBtn.onclick = () => {
        const answer = confirm('결제를 진행하시겠습니까?');
        if (answer) {
            $message.textContent = '결제를 진행합니다.';
        } else {
            $message.textContent = '결제가 취소되었습니다';
        }

    };
    // propmt : 사용자에게 텍스트 입력받음. 입력된 문자열 또는 null 반환.
    $promptBtn.onclick = () => {
        const name = prompt('당신의 이름은 무엇인가요?', '홍길동');
        if (name) {
            $message.textContent = `반갑습니다, ${name}님`;
        } else {
            $message.textContent = '이름이 입력되지 않았습니다.';
        }
    }
</script>

```

- 페이지 제어하기
    - location 객체 : 현재 페이지의 주소(URL) 정보를 담고 있다. 다른 페이지로 이동(redirect) 시키거나 새로고침할 때 사용함

```html
<button id="moveBtn">구글로 이동</button>
<button id="replaceBtn">네이버로 이동 (뒤로가기 불가)</button>
<button id="reloadBtn">새로고침</button>
<script>
    document.getElementById('moveBtn').onclick = () => {
        location.href = "http://www.google.com";
    };

    document.getElementById('replaceBtn').onclick = () => {
        location.replace("http://www.naver.com");
    };

    document.getElementById('reloadBtn').onclick = () => {
        location.reload();
    };
    

</script>
```

- 방문 기록 제어하기
    - history 객체 : 브라우저의 '뒤로가기', '앞으로가기' 버튼 기능을 흉내

```html
<button onclick="history.back()">뒤로 가기</button>
<button onclick="history.forward()">앞으로 가기</button>
<button onclick="history.go(-2)">2페이지 뒤로 가기</button>
```

- 브라우저/화면 정보 얻기 (참고용)
    - navigator, screen 객체 : 사용자의 브라우저 종류, 운영체제, 화면 크기 등의 정보를 얻음

```html
<script>
    console.log(navigator);
    console.log(screen);
</script>
```
