### ✏️getElementById, getElementsByClassName

- HTMLCollection이라는 특별한 목록을 반환
    - 반환 시 문제점

```html
    <div class="box">
        <p class="text">첫 번째 문단</p>
        <p class="text">두 번째 문단</p>
        <p class="text">세 번째 문단</p>
    </div>
```

```html
    <script>
        const $box = document.querySelector('.box');

        const $text = $box.getElementsByClassName('text');
        console.log($text);
        console.log('반복문 시작 전 길이:', $text.length);

        //for문으로 순회하며 클래스 이름 바꾸기
        for(let i = 0; i < $text.length; i++) {
            console.log(`${i}번째 반복문 시작! 현재 $text의 길이는?`, $text.length);
            // text 클래스 제거
            $text[i].classList.remove('text'); //실시간반영됨.length길이가 줄어서 전체 클래스 삭제 안됨
        }
        // HTMLCollection은 첫 번째 반복문을 돌면서 클래스가 제거 되었을때, 실시간으로 반영되어
        // 두번째 반복문때는 세번째 문단의 인덱스가 1번 인덱스가 되어 의도치 않게 두 번째 문단 클래스 제거가 건너뛰어 지게 된다.
    </script>
```

- querySelectorAll은 이런 문제가 없는 '고정된' NodeList를 반환하기 때문에 훨씬 안전함
- getElementsBy… 메서드 써야할 때 가져온 목록을 배열로 변환(스프레드 문법 이용) 후 사용

```html
 <div class="box">
        <p class="text2">첫 번째 문단</p>
        <p class="text2">두 번째 문단</p>
        <p class="text2">세 번째 문단</p>
 </div>
```

```html
<script>
        const $texts2 = document.getElementsByClassName('text2');

        // 스프레드 문법(...)을 이용해 일반 배열로 변환
        const textArray = [...$texts2];
        console.log(textArray);

        textArray.forEach(item => {
            item.classList.remove('text2');
        })
        console.log('작업 후 $text2의 길이: ', $texts2.length); // 0, 실시간으로 비어있음
        console.log('작업 후 textArray.length의 길이: ', textArray.length); // 3, 찍어둔 시점 그대로
    </script>
```

### ✏️HTML 가계도 탐색하기

- querySelector로 특정 요소를 찾은 뒤, 그 요소를 기준으로 가족(부모, 자식, 형제) 요소를 찾기
- 눈에 보이지 않는 '공백'이나 '줄바꿈' (텍스트 노드) 때문에 겪는 혼란을 피하기 위해 '요소(Element)'만 찾아주는 프로퍼티들을 사용

```html
    <ul id="family">
        <!-- 이 주석과 줄바꿈도 노드입니다! -->
        <li id="me">나 (기준)</li>
        <li>나의 첫째 형제</li>
        <li>나의 둘째 형제</li>
    </ul>
```

```html
    <script>
        const $me = document.getElementById('me');

        // 자식 요소 탐색
        const $family = document.getElementById('family');
        // 눈에 보이지 않는 텍스트 노드(줄바꿈), 주석 까지 모두 포함하여 혼란을 준다.
        // console.log($family.childNodes);

        // children: 모든 '자식 요소'들을 HTMLCollection으로 반환(텍스트 노드 제외!)
        const $children = $family.children;
        console.log($children);

        // firstElementChild: 첫 번째 '자식 요소' 반환
        const $first = $family.firstElementChild;
        console.log($first);
        $first.style.color = 'red';

        // lastElementChild: 마지막 '자식 요소' 반환
        const $last = $family.lastElementChild;
        console.log($last);

        // 부모 요소 탐색하기
        // parentNode: 나의 '부모 요소'를 반환
        const $parent = $me.parentNode;
        console.log($parent);
        $parent.style.border = '1px solid blue';

        // 형제 요소 탐색하기

        // nextElementSibling: 나의 바로 다음 '형제 요소'를 반환
        const $next = $me.nextElementSibling;
        console.log($next);
        $next.style.color = 'green';

        // previousElementSibling: 나의 바로 이전 '형제 요소'를 반환
        const $prev = $next.previousElementSibling;
        console.log($prev); // 이전 형제 요소가 없기 때문에 '나' 자신을 가리키게 된다.
    </script>
```

### ✏️요소 정보 확인 및 텍스트 조작

- 요소 종류 확인

```html
<div id="info-area">div 안의 텍스트</div>

<script>
    const $infoArea = document.querySelector('#info-area');
    const $textInside = $infoArea.firstChild;

    // .nodeType: 노드의 종류를 숫자로 알려줌(1: 요소, 3: 텍스트, 9: 문서)
    console.log($infoArea.nodeType);
    console.log($textInside.nodeType);

    console.log($infoArea.nodeName);
    console.log($textInside.nodeName);
</script>
```

- 요소 텍스트 조작
    - .nodeValue
        - 오직 '텍스트 노드'의 값만 읽고 쓸 수 있음
        - '요소 노드' 자체에는 텍스트 정보가 없다고 판단하여 null을 반환
    
    ```html
    <div id="value-area">nodeValue</div>
    <script>
       const $valueArea = document.getElementById('value-area');
       const $valueText = $valueArea.firstChild;
    
       // 요소 노드 자체의 nodeValue는 null 이다.
       console.log($valueArea.nodeValue);
       // 텍스트를 바꾸려면, 자식인 '텍스트 노드'를 직접 찾아가야만 한다.
       $valueText.nodeValue = 'nodeValue로 변경 완료!'
    </script>
    ```
    
    - .textContent (현대적인 표준)
        - 요소 노드 안에 바로 사용 가능
        - 요소 안의 모든 자손 노드들의 텍스트까지 전부 가져옴
        - 값을 할당하면, 내부의 모든 자식 노드를 지우고 새로운 텍스트로 교체
    
    ```html
    <div id="content-area">여기는 <b>굵은</b> 글씨가 있는 영역입니다.</div>
    
    <script>
        const $contentArea = document.querySelector('#content-area');
    
        // 내부 HTML 태그는 무시하고 텍스트만 전부 가져온다
        console.log($contentArea.textContent);
    
        // 값을 할당하면, 내부의 모든 것을 지우고 이 텍스트로 교체(HTML 태그 인식 못함)
        $contentArea.textContent = 'textContent로 쉽게 변경! <span>태그는 무시됨<span>';
    
    </script>
    ```

### ✏️요소 조작 및 추가

- innerHTML :요소 내부의 모든 것을 지우고, 새로운 HTML 문자열로 완전히 교체

```html
<div id="area1">
    <p>기존에 있던 내용입니다.</p>
</div>

<script>
   const $area1 = document.querySelector('#area1');

   // innerHTML로 내부를 완전히 새로운 내용으로 교체
   $area1.innerHTML = '<ul><li>새로운 리스트 아이템</li></ul>';
   // 단점: 사용자의 입력을 그대로 넣으면 XSS 공격에 취약하다.
   //$area1.innerHTML = `<img src = 'x' onerror='alert("해킹 공격!")'>`;
</script>
```

- insertAdjacentHTML : 기존 내용은 유지하면서 원하는 위치(4군데 중 하나)에 HTML요소를 끼워넣음

```html
<pre>
    `insertAdjacentHTML`은 기존 내용은 그대로 유지하면서,
    원하는 위치 4곳 중 하나를 지정해 새로운 HTML 요소를 '끼워 넣는' 효율적인 방법입니다.
    </pre>
<!-- beforebegin -->
<div id="area2" style="border: 2px solid blue; padding: 10px;">
    <!-- afterbegin -->
    <p>기존 컨텐츠 </p>
    <!-- beforeend -->
</div>
<!-- afterend -->

<script>
   const $area2 = document.getElementById('area2');
   
   // 4가지 위치에 HTML 삽입하기
   $area2.insertAdjacentHTML('beforebegin', '<p>div 시작 전(beforebegin)</p>');
   $area2.insertAdjacentHTML('afterbegin', '<p>div 내부의 가장 앞(afterbegin)</p>');
   $area2.insertAdjacentHTML('beforeend', '<p>div 내부의 가장 뒤(beforeend)</p>');
   $area2.insertAdjacentHTML('afterend', '<p>div 끝난 후(afterend)</p>');
</script>
```

- createElement / appendChild : 직접 요소를 만들고 만들어진 요소를 원하는 위치에 조립(append)

```html
<ul id="drink-list">
    <li>커피</li>
</ul>

<script>
   const $drinkList = document.getElementById('drink-list');

   // 'li' 요소 노드를 만든다.
   const $newLi = document.createElement('li'); //<li></li> 만듦

   // textContent로 안전하게 텍스트를 넣는다.
   $newLi.textContent = '콜라'; //<li>콜라</li> 텍스트 넣음

   // appendChild로 ul 리스트의 마지막 자식으로 추가한다.
   $drinkList.appendChild($newLi);

   // 여러 요소 효율적으로 추가(DocumentFragment)
   const $fragment = document.createDocumentFragment(); // 임시 조립 공간
   ['사이다', '우유', '환타'].forEach(text => {
        const $li = document.createElement('li');
        $li.textContent = text;
        $fragment.appendChild($li); // 임시 공간에 먼저 조립
   });

   // 마지막에 한 번만 실제 DOM에 조립
   $drinkList.appendChild($fragment);
   
</script>
```

- createElement 이용해서 삽입, 이동, 교체, 삭제

```html
<ul id="food-list">
    <li id="pasta">파스타</li>
    <li id="pizza">피자</li>
</ul>

<script>
   const $foodList = document.getElementById('food-list');
   const $drinkList2 = document.getElementById('drink-list');
   const $pasta = document.getElementById('pasta');
   const $pizza = document.getElementById('pizza');

   // 노드 삽입 (insertBefore)
   const $newFood = document.createElement('li');
   $newFood.textContent = '스테이크';
   // 피자 앞에 스테이크 삽입
   $foodList.insertBefore($newFood, $pizza);

   // 노드 이동 (appendChild)
   // 이미 존재하는 '피자'를 drink-list로 옮기기
   $drinkList2.appendChild($pizza); //원래있던 자리에서 없어지고 이동

   // 노드 교체 (reaplaceChild)
   const $newDrink = document.createElement('li');
   $newDrink.textContent = '오렌지주스';
   // drink-list의 첫 요소를 오렌지주스로 교체
   $drinkList2.replaceChild($newDrink, $drinkList2.firstElementChild);

   // 노드 삭제 (removeChild)
   $foodList.removeChild($pasta);
</script>
```

### ✏️Attribute / Property

- HTML 태그에 부여된 속성(attribute)을 이용해서 조작

```html
<label for="username">유저명</label>
<input type="text" id="username" value="user01">

<script>
    const $input = document.getElementById('username');

    // 읽기: getAttribute('어트리뷰트이름');
    const inputValue = $input.getAttribute('value');
    console.log(inputValue);

    // 쓰기: setAttribute('어트리뷰트이름', '값');
    $input.setAttribute('value', 'new-user');

    // 확인: hasAttribute('어트리뷰트이름')
    console.log(`class 어트리뷰트 존재 여부: ${$input.hasAttribute('class')}`);

    // 지우기: removeAttribute('어트리뷰트이름')
    $input.removeAttribute('value');
</script>
```

- Attribute vs Property
    - Attribute : HTML 문서에 적힌 '초기 설정값’
    - Property : 자바스크립트 객체가 가진 '실시간 최신값’

```html
<label for="nickname">닉네임</label>
<input type="text" id="nickname" value="JSBeginner">

<script>
    const $nickname = document.getElementById('nickname');

    // 사용자가 input에 글자를 입력할 때마다 비교
    $nickname.oninput = () => {
        // .value 프로퍼티는 사용자의 입력을 실시간으로 반영
        console.log('프로퍼티(최신값):', $nickname.value);
        // .getAttribute('value')는 HTML에 적힌 초기값을 그대로 유지
        console.log('어트리뷰트(초기값): ', $nickname.getAttribute('value'));
    }
    /*
    - 어트리뷰트는 초기 상태를 기억하고, 프로퍼티는 현재 상태를 기억한다.
    - 사용자의 입력 값을 가져올 때는 반드시 '프로퍼티(.value)'를 사용해야 한다. 
    */

   // (참고)
   const $checkbox = document.createElement('input');
   $checkbox.type = 'checkbox';
   $checkbox.setAttribute('checked', '');

   console.log(`어트리뷰트: ${$checkbox.getAttribute('checked')}`); // '' (빈 문자열)
   console.log(`프로퍼티: ${$checkbox.checked}`) // true (boolean)
</script>
```

- 실무 활용: data 어트리뷰트와 dataset 프로퍼티
    - data-이름  형태로 HTML에 어트리뷰트를 작성, 자바스크립트에서는 .dataset.이름  프로퍼티로 쉽게 접근

```html
<ul class="board-list">
    <li data-board-id="13" data-category-name="맛집">망원 맛집 추천 부탁드려요!</li>
    <li data-board-id="15" data-category-name="운동">한강 러닝 크루 구해요!</li>
    <li data-board-id="18" data-category-name="경제">요즘 재태크 어떻게 하시나요?</li>
</ul>

<script>
   const $boardList = document.querySelector('.board-list');

   $boardList.onclick = () => {
    // 클릭된 요소가 li가 아니면 무시
        if(event.target.tagName !== 'LI') return;

        const boardId = event.target.dataset.boardId; // data-board-id -> boardId(카멜케이스)
        const categoryName = event.target.dataset.categoryName;

        console.log(`선택한 게시글의 ID는 ${boardId}이고, 카테고리는 ${categoryName}입니다.`);
   }
</script>
```

### ✏️CSS 조작 (인라인 스타일 vs 클래스 제어)
