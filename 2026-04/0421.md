## 🔹 1. fetch 기본 개념

```jsx
let promise = fetch(url, [option]);
```

- `fetch()`는 **비동기 함수**
- 바로 결과를 주는 게 아니라 **Promise 객체**를 반환

---

## 🔹 2. Promise 상태 확인

```jsx
const promise = fetch('https://jsonplaceholder.typicode.com/users');
console.log(promise);
console.log(promise['[[PromiseResult]]']); // undefined
```

- `promise`는 아직 결과가 안 나온 상태
- `[[PromiseResult]]`는 내부 슬롯이라 직접 접근 불가 → `undefined`
- **await 없이 결과 못 씀**

---

## 🔹 3. await로 응답 받기

```jsx
const response = await fetch('https://jsonplaceholder.typicode.com/users');
console.log(response);
```

- `await`를 쓰면 Promise가 끝날 때까지 기다림
- `response`는 **Response 객체**

---

## 🔹 4. body 사용 여부 확인

```jsx
console.log(response.bodyUsed);
```

- `false` → 아직 본문 안 읽음
- `true` → 이미 한 번 읽음

---

## 🔹 5. JSON 변환

```jsx
const responseJson = await response.json();
console.log(responseJson);
```

- `response.json()`도 **비동기**
- 반드시 `await` 필요
- 결과: **JS 객체 (배열/객체)**

---

## 🔹 6. body는 한 번만 사용 가능

```jsx
console.log(response.bodyUsed);
```

- `true`로 바뀜
- 같은 response에서 `.json()` 두 번 못 씀 ❌

---

## 🔹 전체 흐름 요약

1. `fetch()` → Promise 반환
2. `await fetch()` → Response 객체 획득
3. `await response.json()` → 실제 데이터 획득
4. body는 한 번만 읽기 가능

---

## 🔹 전체 흐름 (공통)

둘 다 내부적으로 같은 작업을 함:

```
1. fetch 요청
2. Response 받기
3. response.json()으로 데이터 변환
4. 결과 사용
5. 에러 처리
```

## 🔹 방법 1: `.then()` 체이닝 방식

```jsx
fetch(API_URL)
  .then(response => {
    if (!response.ok) throw new Error();
    return response.json();
  })
  .then(data => console.log(data))
  .catch(error => console.log(error));
```

### 특징

- Promise를 **체인으로 연결**
- 함수형 스타일
- 오래된 방식 (하지만 여전히 많이 쓰임)

### 흐름

1. fetch 완료 → 첫 번째 then 실행
2. json 변환 → 두 번째 then 실행
3. 중간 에러 → catch로 이동

---

## 🔹 방법 2: `async/await`

```jsx
async function fetchData() {
  try {
    const response = await fetch(API_URL);

    if (!response.ok) throw new Error();

    const data = await response.json();
    console.log(data);
  } catch (error) {
    console.log(error);
  }
}
```

### 특징

- **동기 코드처럼 보임**
- 가독성 훨씬 좋음
- try-catch로 에러 처리

---

## 🔹 차이 핵심 비교

| 구분 | then 방식 | async/await |
| --- | --- | --- |
| 코드 스타일 | 체이닝 | 동기처럼 |
| 가독성 | 낮음 | 높음 |
| 에러 처리 | .catch() | try-catch |
| 추천 | ❌ | ✅ (현대 표준) |

---

## 🔹 중요한 포인트

### 1. `response.ok` 체크

```jsx
if (!response.ok) {
  throw new Error(`HTTP error: ${response.status}`);
}
```

- 서버 에러 (404, 500 등) 직접 처리해야 함
- fetch는 **에러 자동 throw 안 함**

---

### 2. `response.json()`도 비동기

```jsx
const data = await response.json();
```

- 이것도 Promise라서 **await 필요**

---

### 3. 에러 처리 위치 차이

- `.then()` → `.catch()`
- `async/await` → `try-catch`

---

### 🔹 추천 방법

```jsx
async function fetchData() {
  try {
    const res = await fetch(API_URL);
    if (!res.ok) throw new Error();

    return await res.json();
  } catch (e) {
    console.error(e);
  }
}
```

## 🔹 2. fetch 방식

```jsx
async functioncallApiWithFetch() {
	try{
		const response = await fetch(API_URL);
	
		if (!response.ok) {
			throw new Error(`HTTP error:${response.status}`);
		 }
	
		const data = await response.json();
		console.log(data);
	} catch (error) {
		console.log(error);
	}
}
```

### ✔ 특징

- 기본 내장 API (추가 설치 필요 없음)
- `response.json()` 직접 호출해야 함
- HTTP 에러를 자동으로 잡지 않음 → `ok 체크 필수`

## 🔹 3. axios 방식

```jsx
async  functioncallApiWithAxios() {
	try {
		const response = await axios.get(API_URL);
		const data = response.data;
		console.log(data);
	}catch (error) {
		console.log(error);
	    }
}
```

### ✔ 특징

- 외부 라이브러리 (CDN / npm 필요)
- JSON 자동 파싱 → `response.data`
- HTTP 에러 자동으로 catch 처리됨
- `.get()`, `.post()` 등 직관적인 메소드 제공

| JSON 파싱 | 직접 `.json()` | 자동 |
| 에러 처리 | 수동 (`ok 체크`) | 자동 |
| 사용성 | 약간 불편 | 편함 |

## 🔹 3. SearchBox (입력 컴포넌트)

```jsx
function SearchBox({onSearch}) {
      const [inputText, setInputText] = useState('');

      const handleSearchClick = () => {
          // 전달 받은 onSearch 함수 사용
          onSearch(inputText);
      }

      return (
          <>
              <input 
                  type="text"
                  placeholder='이모지 이름을 입력하세요'
                  value={inputText}
                  onChange={e => setInputText(e.target.value)}
                  //onKeyDown={}
              />
              <button onClick={handleSearchClick}>검색</button>
          </>
      )
        }
```

### 역할

- 사용자 입력값 관리 (`useState`)
- 버튼 클릭 시 부모에게 전달

### 핵심 로직

```jsx

```

- 부모(App)의 `handleSearch` 실행

## 🔹 4. ImageBox (출력 컴포넌트)

```jsx
function ImageBox({imgUrl, loading}) {
    if (loading) {
        return <p>로딩 중...</p>
    }
    if (imgUrl) {
        return <img src={imgUrl} alt="emoji" width="100" height="100"/>
    }

    return <p>이모지를 검색해주세요</p>;
}
```

### 화면 분기 처리

```jsx

else → 안내 메시지
```

## 🔹 5. App (핵심 로직)

```jsx
function App() {

    const [loading, setLoading] = useState(true);  // 로딩 상태
    const [emojis, setEmojis] = useState(null);    // 전체 이모지 목록
    const [imgUrl, setImgUrl] = useState(null);    // 검색 결과 이미지 URL

    useEffect(() => {
        // 마운트시 딱 한 번 api 호출해서 전체 이모지 목록을 가져온다.
        async function fetchEmojis() {
            try {
                const response = await axios.get('https://api.github.com/emojis');
                setEmojis(response.data); // 가져온 데이터를 state에 저장
            } catch(error) {
                console.log(error);
            } finally {
                setLoading(false); // 로딩 완료
            }
        }
        fetchEmojis();
    }, [])

    // 검색된 이모지 이름을 이용하여 일치하는 이미지를 찾아, ImgUrl 설정
    const handleSearch = (emojiName) => {
        if(!emojis || emojiName.trim() === '') {
            setImgUrl(null);
            return;
        }
        // emojis 객체에서 emojiName을 키로 가진 값을 찾음
        const foundUrl = emojis[emojiName]; //객체 키값 찾아올때 []표기법사용
        setImgUrl(foundUrl || null); // 찾았으면 url을, 못 찾았으면 null 설정
    };

    return (
        <>
            <h1>이모지 검색기</h1>
            <SearchBox onSearch={handleSearch}/>    
            <br/>
            <ImageBox imgUrl={imgUrl} loading={loading}/>
        </>
    );
        }
```

### 상태 관리

```jsx
const [loading, setLoading] = useState(true);
const [emojis, setEmojis] = useState(null);
const [imgUrl, setImgUrl] = useState(null);
```

| 상태 | 역할 |
| --- | --- |
| loading | API 로딩 상태 |
| emojis | 전체 이모지 데이터 |
| imgUrl | 검색 결과 이미지 |

## 🔹 2. 전체 코드

```html
<script type="text/babel">
        
  const { useState, useEffect } = React;

  function Title() {
      return <h1>회원 목록</h1>;
  }

  function ItemList() {
      const [users, setUsers] = useState([]);  // 서버에서 받아올 유저 목록
      const [loading, setLoading] = useState(true); // 로딩 상태
      const [error, setError] = useState(null); // 에러 상태

      useEffect(() => {
          fetch('https://jsonplaceholder.typicode.com/users')
              .then(response => {
                  if (!response.ok) {
                      throw new Error('네트워크 응답 문제가 있습니다.');
                  }
                  return response.json();
              })
              .then(responseUsers => setUsers(responseUsers))
              .catch(error => {
                  console.error('데이터를 불러오는 데 실패했습니다.', error);
                  setError(error);  // 에러 발생 시 에러 상태에 저장
              })
              .finally(() => setLoading(false));
      }, []);

      // Early Return 패턴
      if (loading) {
          return <p>로딩 중...</p>
      } 
      if (error) {
          return <p>오류가 발생했습니다: {error.message}</p>
      }

      return (
          <div>
              {users.map(user => <Item key={user.id} user={user}/>)}    
          </div>
      )
  }

  function Item({user}) {
      return (
          <div className="item">  
              <h4>{user.name}</h4>
              userId : {user.id} <br/>
              email : {user.email} <br/>
              phone : {user.phone} <br/>
              {/* 옵셔널 체이닝 연산자 사용(에러 방지) */}
               company name : {user.company?.name} 
          </div>
      )
  }

  function App() {
      return (
          <>
              <Title />
              <ItemList />
          </>
      )
  }

  ReactDOM.createRoot(document.getElementById('root')).render(<App />);
  
    </script>
```
