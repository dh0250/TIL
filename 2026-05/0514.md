# MySQL
## chap01-select

### 01. SELECT (조회)

데이터베이스의 특정 테이블에서 원하는 데이터를 조회하는 가장 기본적인 구문.

---

✔ 01. Basic Select (기본 조회 및 연산)

SELECT 키워드로 컬럼을 선택하고, FROM 절로 대상 테이블을 지정.

* 모든 컬럼 조회: * 와일드카드를 통한 테이블 내 전체 데이터 추출.
* 산술 연산: FROM 절 없이 간단한 사칙연산 및 결과 출력 가능.
* 내장 함수: NOW(), CONCAT() 등 MySQL 제공 함수 활용.

```sql
-- 모든 열 데이터 검색
SELECT * FROM tbl_menu;

-- 간단한 연산 및 함수 사용
SELECT 6 * 3;
SELECT NOW(); -- 현재 일시
SELECT CONCAT('홍', '길동') AS 'Full name';
```

---

👀 핵심 요약

* 조회의 기초: SELECT와 FROM은 조회의 필수 요소.
* 와일드카드(*): 편리성 대비 가독성과 성능을 고려한 컬럼명 명시 권장.
* 별칭(Alias): AS를 통한 컬럼명 부여 및 계산 결과의 명확화.

---

## chap02-order-by

### 01. ORDER BY (정렬)

조회된 결과 집합을 특정 기준에 따라 정렬하며, SELECT문의 최하단에 위치.

---

✔ 01. Sort Direction (정렬 방향 및 다중 기준)

* ASC (Ascending): 오름차순 정렬 (기본값, 생략 가능).
* DESC (Descending): 내림차순 정렬.
* 다중 정렬: 선행 정렬 기준이 동일할 경우, 후행 기준을 적용한 순서 결정.

```sql
-- 가격 내림차순, 이름 오름차순 정렬
SELECT menu_name, menu_price
FROM tbl_menu
ORDER BY menu_price DESC, menu_name ASC;

-- 연산 결과(별칭)로 정렬
SELECT menu_code * menu_price AS val
FROM tbl_menu
ORDER BY val DESC;
```

✔ 02. Advanced Sorting (특수 정렬)

* FIELD(): 나열된 목록 순서에 따른 사용자 정의 정렬 수행.
* NULL 정렬: IS NULL을 활용한 NULL 값의 최상단 또는 최하단 배치 제어.

```sql
-- 'N' 상태를 1순위로 정렬
SELECT menu_name, orderable_status
FROM tbl_menu
ORDER BY FIELD(orderable_status, 'N', 'Y');
```

---

👀 핵심 요약

* 최종 정렬: 데이터 가시성 확보를 위한 마지막 처리 단계.
* 다중 조건: 콤마(,)를 활용한 복합 정렬 조건 구성.
* NULL 제어: 정렬 시 비어있는 값의 위치를 논리적으로 조정 가능.

---

## chap03-where

### 01. WHERE (조건 필터링)

특정 조건에 부합하는 레코드만 선택하여 결과를 제한하는 역할.

---

✔ 01. Operators (비교 및 논리 연산자)

* 비교 연산자: =, <>(불일치), <, >, <= 등 사용.
* 논리 연산자: AND, OR를 조합한 복합 조건 구성.
* BETWEEN: 특정 범위(A 이상 B 이하) 포함 여부 확인.

```sql
-- 10,000원 ~ 25,000원 범위 메뉴 조회
SELECT menu_name, menu_price
FROM tbl_menu
WHERE menu_price BETWEEN 10000 AND 25000;
```

✔ 02. Pattern & Set (패턴 매칭 및 집합)

* LIKE: % 와일드카드를 이용한 부분 문자열 일치 검색.
* IN: 지정된 값 목록 중 하나라도 일치하는 데이터 선택.
* IS NULL: 데이터가 비어있는 상태(NULL) 검색.

```sql
-- '마늘' 포함 메뉴 검색
SELECT menu_name
FROM tbl_menu
WHERE menu_name LIKE '%마늘%';

-- 특정 카테고리(4, 5, 6) 소속 데이터 조회
SELECT *
FROM tbl_menu
WHERE category_code IN (4, 5, 6);
```

---

👀 핵심 요약

* 데이터 정제: 불필요한 행 제외 및 핵심 데이터 추출.
* NULL 처리: 비교 연산자가 아닌 IS NULL 또는 IS NOT NULL 사용 필수.
* 가독성: 복합 OR는 IN으로, 범위는 BETWEEN으로 간결하게 표현.

---

## chap04-distinct

### 01. DISTINCT (중복 제거)

조회 결과에서 중복 행을 제외하고 고유한(Unique) 값만 추출.

---

✔ 01. Unique Values (단일 및 다중 열 중복 제거)

* 단일 열: 특정 컬럼에 존재하는 데이터 종류 파악 시 활용.
* 다중 열: 명시된 모든 컬럼의 조합이 동일한 경우에만 중복으로 간주.

```sql
-- 카테고리 종류 확인
SELECT DISTINCT category_code
FROM tbl_menu;

-- 코드와 상태의 고유 조합 조회
SELECT DISTINCT category_code, orderable_status
FROM tbl_menu;
```

---

👀 핵심 요약

* 범주 파악: 전체적인 데이터 도메인 및 종류 확인에 유용.
* 전체 조합 기준: 선택된 모든 컬럼이 하나의 묶음으로 중복 체크 기준이 됨.

---

## chap05-limit

### 01. LIMIT (결과 개수 제한)

전체 결과 집합 중 필요한 만큼의 행만 분리하여 조회.

---

✔ 01. Pagination (페이징 처리)

* LIMIT [개수]: 상위 N개 데이터만 반환.
* LIMIT [오프셋], [개수]: 시작 위치(Offset) 기준 지정 개수 추출 및 페이징 로직 구현.

```sql
-- 가격 상위 5개 메뉴
SELECT menu_name, menu_price
FROM tbl_menu
ORDER BY menu_price DESC
LIMIT 5;

-- 2번째 데이터부터 4개 조회
SELECT *
FROM tbl_menu
LIMIT 1, 4;
```

---

👀 핵심 요약

* 성능 최적화: 대량 조회 시 필요한 구간만 추출하여 시스템 부하 감소.
* 페이징 핵심: 목록형 인터페이스에서 페이지 분할을 위한 필수 문법.

---

## chap06-join

### 01. JOIN (테이블 결합)

서로 다른 테이블을 공통 컬럼 기준으로 연결하여 통합된 결과 생성.

---

✔ 01. Inner & Outer Join (내부 및 외부 조인)

* INNER JOIN: 교집합 기준으로 양쪽 테이블에 매칭 데이터가 모두 존재할 때 결합.
* LEFT JOIN: 왼쪽 테이블 기준 데이터 유지 및 오른쪽 불일치 영역 NULL 처리.
* RIGHT JOIN: 오른쪽 테이블 기준 데이터 유지 및 결합.

```sql
-- 메뉴와 카테고리 테이블 연결
SELECT a.menu_name, b.category_name
FROM tbl_menu a
JOIN tbl_category b
ON a.category_code = b.category_code;
```

✔ 02. Self Join (자체 조인)

* Self Join: 동일 테이블 내 계층 구조(상위-하위) 표현을 위한 별칭 기반 결합.

```sql
-- 하위 카테고리와 상위 카테고리 매칭 조회
SELECT a.category_name AS child,
       b.category_name AS parent
FROM tbl_category a
JOIN tbl_category b
ON a.ref_category_code = b.category_code;
```

---

👀 핵심 요약

* RDB의 핵심: 정규화된 데이터를 논리적으로 연결하여 정보 재구성.
* 별칭 활용: 컬럼 소속 명확화를 위한 테이블 별칭(a, b) 사용 권장.
* 조인 선택: 데이터 유실 방지는 OUTER JOIN, 정확한 매칭은 INNER JOIN 사용.
