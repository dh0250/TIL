## chap01-connection

1. CONNECTION (DB 연결)

자바 애플리케이션과 데이터베이스 사이의 통로를 생성하고 관리하는 핵심 단계.

---

✔ 01. DriverManager & Connection객체

* Class.forName(): 사용하는 DB의 JDBC 드라이버를 로딩.
* DriverManager.getConnection(): URL, ID, Password 정보를 전달하여 Connection 객체 획득.
* close(): 사용이 끝난 연결 통로는 자원 해제를 위해 반드시 닫아주어야 함.

```java
// 직접 정보를 입력하여 연결
con = DriverManager.getConnection("jdbc:mysql://localhost/employeedb", "user", "password");
```

✔ 02. Configuration (설정 파일 활용)

* .properties: DB 접속 정보를 하드코딩하지 않고 별도 파일로 분리하여 관리.
* Properties.load(): 설정 파일을 읽어와 동적으로 접속 정보를 로드함으로써 유지보수성 향상.

```java
// 설정 파일을 통한 연결 정보 로드
Properties prop = new Properties();
prop.load(new FileReader("connection-info.properties"));
con = DriverManager.getConnection(prop.getProperty("url"), prop);
```

---

👀 핵심 요약

* 연결 통로: Connection은 DB 작업의 가장 기본이 되는 객체.
* 자원 관리: 연결 생성 후 예외 발생 여부와 상관없이 finally 블록 등을 통해 반드시 close() 호출.
* 유연성: 접속 정보를 소스 코드와 분리하여 환경 변화에 대응하기 쉽게 설계.

---

## chap01-connection

### 1. JDBC TEMPLATE (공통 모듈화)

반복되는 DB 연결 및 자원 해제 로직을 별도 클래스로 분리하여 코드 중복을 제거하는 방식.

---

✔ 01. Connection Static Method

* getConnection(): Connection 객체를 생성하여 반환하는 공통 메소드.
* close(Connection con): 전달받은 객체의 유효성을 검사한 후 안전하게 종료 처리.
* static 활용: 객체 생성 없이 클래스명으로 즉시 호출 가능하도록 설계.

```java
// JDBCTemplate 활용 예시
Connection con = JDBCTemplate.getConnection();
// ... DB 작업 수행
JDBCTemplate.close(con);
```

---

👀 핵심 요약

* 코드 재사용: 프로젝트 전체에서 동일한 연결/해제 로직을 공유하여 생산성 증대.
* 유지보수 단일화: 접속 정보나 로직 변경 시 JDBCTemplate 클래스만 수정하면 전체 적용 가능.
---
## chap02-statements

### 1. STATEMENT (쿼리 실행)

자바 프로그램에서 SQL 문을 데이터베이스로 전송하고 결과를 받아오기 위해 사용되는 기본적인 객체.

---

✔ 01. Statement 객체

* Connection.createStatement(): 쿼리를 실행할 Statement 객체 생성.
* executeQuery(sql): SELECT 문을 실행할 때 사용하며, 결과로 ResultSet을 반환.
* executeUpdate(sql): INSERT, UPDATE, DELETE 문을 실행할 때 사용하며, 영향받은 행의 수를 반환.

✔ 02. ResultSet 객체

* select 쿼리의 결과 집합을 담는 객체.
* next(): 커서를 다음 행으로 이동시키며, 데이터가 있으면 true를 반환.
* getXXX("컬럼명"): 현재 커서가 가리키는 행의 특정 컬럼 데이터를 해당 타입으로 가져옴.

```java
// Statement를 이용한 기본 조회 예시
stmt = con.createStatement();
rset = stmt.executeQuery("SELECT MENU_NAME, MENU_PRICE FROM TBL_MENU");

while (rset.next()) {
    System.out.println(rset.getString("MENU_NAME") + ", " + rset.getInt("MENU_PRICE"));
}
```

---

👀 핵심 요약

* 정적 쿼리: 실행 시점에 완성된 SQL 문을 전달해야 하며, 매번 구문을 분석하므로 성능 면에서 대량 처리에 불리할 수 있음.
* 자원 해제: 생성된 순서의 역순(ResultSet -> Statement -> Connection)으로 반드시 close()를 호출.
* 단순성: 파라미터가 없는 간단한 쿼리를 실행할 때 직관적임.

---

### 2. PREPARED STATEMENT (동적 쿼리 처리)

SQL 문을 미리 컴파일하여 캐싱해두고, 실행 시에 파라미터(위치 홀더)만 전달하여 효율성과 보안성을 높인 방식.

---

✔ 01. PreparedStatement 객체

* Connection.prepareStatement(sql): 실행할 SQL 문을 인자로 전달하며 객체 생성.
* ? (위치 홀더): SQL 문 내부에서 값이 들어갈 자리를 ?로 표시.
* setXXX(순서, 값): 위치 홀더에 들어갈 실제 데이터를 설정 (1번부터 시작).

✔ 02. External Configuration (XML/Properties)

* SQL 문을 소스 코드에 하드코딩하지 않고 별도의 파일(.xml, .properties)로 관리.
* Properties.loadFromXML(): XML 파일에 저장된 쿼리를 읽어와 동적으로 실행 가능.

```java
// PreparedStatement 활용 및 위치 홀더 설정
String query = "SELECT * FROM TBL_MENU WHERE MENU_CODE = ?";
pstmt = con.prepareStatement(query);
pstmt.setString(1, "4"); // 1번 위치 홀더에 값 설정
rset = pstmt.executeQuery();
```

---

👀 핵심 요약

* 효율성: 동일한 구조의 쿼리를 반복 실행할 때 컴파일 비용이 줄어 성능이 향상됨.
* 보안성: 사용자 입력값이 SQL 문법으로 해석되지 않도록 처리하여 SQL Injection을 방어함.
* 가독성: 문자열 더하기(+)를 통한 쿼리 조립보다 코드가 깔끔하고 유지보수가 쉬움.

---

### 3. SQL INJECTION (보안 취약점과 방어)

사용자의 입력값에 SQL 구문을 주입하여 데이터베이스를 비정상적으로 조작하거나 정보를 탈취하는 공격 기법.

---

✔ 01. Statement의 취약성

* 문자열 결합 방식: 사용자가 입력한 문자열이 SQL 구문의 일부가 되어 쿼리 논리 구조가 변경될 수 있음.
* 입력값 격리: 위치 홀더(?)를 사용하면 입력된 데이터는 항상 단순한 '데이터 값'으로만 처리됨.
* 자동 이스케이프: 특수 문자나 예약어 등이 포함되어도 SQL 엔진은 이를 구문으로 해석하지 않음.

```java
// SQL Injection 방어 코드 (PreparedStatement 사용)
String query = "SELECT * FROM EMPLOYEE WHERE EMP_ID = ? AND EMP_NAME = ?";
pstmt = con.prepareStatement(query);
pstmt.setString(1, empId);   // 입력값이 쿼리 구조에 영향을 주지 않음
pstmt.setString(2, empName);
```

---

👀 핵심 요약

* 입력 데이터 신뢰 금지: 외부에서 들어오는 모든 데이터는 잠재적인 위협으로 간주해야 함.
* 표준 권장: JDBC 개발 시 보안과 성능을 모두 고려하여 Statement보다는 PreparedStatement 사용을 강력히 권장함.
* 로직 보호: 데이터베이스의 구조를 보호하고 의도하지 않은 데이터 노출을 막는 핵심 보안 단계.
---
## chap03-crud

### 1. INSERT (데이터 추가)

테이블에 새로운 데이터를 삽입하는 DML(Data Manipulation Language) 작업 수행 방식.

---

✔ 01. PreparedStatement 활용

* 위치 홀더(?): SQL 쿼리 내에 변수가 들어갈 자리를 ?로 표시하여 보안성(SQL Injection 방지)과 가독성 확보.
* setXXX(): setString(), setInt() 등의 메소드를 통해 각 홀더(?)에 매핑할 데이터를 동적으로 설정.
* executeUpdate(): DML 작업 수행 후 영향받은 행의 갯수(int)를 반환하며, 성공 시 보통 1 이상의 값을 가짐.

```java
pstmt = con.prepareStatement(query);
pstmt.setString(1, "봉골레청국장");
pstmt.setInt(2, 13000);
pstmt.setInt(3, 1);
pstmt.setString(4, "Y");

result = pstmt.executeUpdate();
```

✔ 02. External Query Management (XML)

* XML Mapper: SQL 쿼리를 자바 소스 코드와 분리하여 외부 XML 파일(menu-query.xml)에서 관리.
* loadFromXML(): Properties 객체의 기능을 활용해 XML 형식으로 저장된 쿼리 문장을 동적으로 읽어옴.
* 분리 목적: 쿼리 수정 시 자바 코드를 컴파일하지 않고도 XML 파일만 수정하여 즉시 반영 가능한 유지보수 환경 구축.

```java
Properties prop = new Properties();
prop.loadFromXML(new FileInputStream("src/main/java/com/ohgiraffers/mapper/menu-query.xml"));
String query = prop.getProperty("insertMenu");
```

---

👀 핵심 요약

* 보안 및 효율: PreparedStatement를 통해 SQL 인젝션을 방지하고 쿼리 실행의 안정성을 확보.
* 관리 편의성: 비즈니스 로직(Java)과 데이터베이스 로직(SQL)을 분리하여 코드의 청결도 향상.
* 자원 해제: JDBCTemplate을 활용해 사용이 끝난 Connection과 PreparedStatement를 안전하게 종료.
