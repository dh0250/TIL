## 02. Object Array Uses (객체 배열 활용)

객체 배열을 활용하여 데이터를 관리(저장, 조회, 검색)하는 회원 관리 프로그램의 구조와 원리

---

## ✔ 계층별 역할 분담 (Architecture)

프로그램을 논리적 단위로 나누어 유지보수와 확장을 용이하게 함

- DTO (`Member`)

  회원 데이터를 담는 객체 (캡슐화 적용)

- Repository (`MemberRepository`)

  데이터 저장소 (객체 배열 관리, 데이터 입출력)

- Service (`MemberService`)

  비즈니스 로직 처리 (등록, 조회, 검색 로직 제어)

- Application

  사용자 인터페이스 (메뉴 출력, 입력 처리)


---

## ✔ 주요 기술 요소

### Static을 활용한 저장소 공유

`MemberRepository`에서 `static` 배열을 사용하여 프로그램 실행 동안 데이터를 유지하고

모든 인스턴스가 동일한 데이터 공간을 공유하게 함

```java
public class MemberRepository {

    private final static Member[] members; // static 배열 (전체 공유)
    private static int count;              // 저장된 회원 수 관리

    static {
        members = new Member[10];
    } // 정적 초기화 블럭
}
```

---

### 객체 배열의 순회와 Null 체크

배열의 크기는 고정되어 있지만 실제 데이터가 다 차지 않았을 경우를 대비해

반복문 사용 시 반드시 `null` 체크가 필요함

```java
for (Member member : allMembers) {

    if (member != null) { // NullPointerException 방지
        System.out.println(member.getInformation());
    }
}
```

---

## ✔ 핵심 로직 활용

### 일괄 등록 (Bulk Insert)

여러 객체를 배열에 담아 한 번에 저장소로 전달하여 처리 효율을 높임

```java
Member[] members = {
    new Member(1, "user01", "pass01", 20, '남'),
    new Member(2, "user02", "pass02", 16, '여')
};

MemberRepository.store(members);
```

---

### 데이터 검색 (Search)

배열을 순회하며 특정 조건(예: 아이디 일치 여부)을 만족하는 객체를 찾아 결과를 반환

```java
for (Member member : foundMember) {

    if (member != null && searchId.equals(member.getId())) {

        System.out.println(member.getInformation());
        isFound = true;
        break; // 찾으면 반복 종료
    }
}
```

---

## 👀 핵심 요약

- 계층 분리

  데이터(DTO), 저장(Repository), 로직(Service)을 분리하여 관리

- 데이터 영속성

  `static` 배열을 통해 프로그램 실행 중 메모리 상에서 데이터 유지

- 안정성

  객체 배열 사용 시 `if (member != null)` 체크 필수

- 효율성

  반복문(`for`)과 조건문(`if`)을 조합하여 대량 데이터 처리 가능

# chap08-inheritance

## 01. Inheritance (상속 - extends)

기존 클래스(부모)의 멤버를 물려받아 새로운 클래스(자식)를 만드는 Java의 핵심 기술

---

## ✔ 주요 개념 및 활용

- 코드의 재사용성

  부모 클래스의 필드와 메소드를 그대로 사용하여 중복 코드를 제거함

- 기능의 확장

  부모의 기능을 기반으로 자식만의 독특한 기능을 추가(Extension)함

- protected 접근 제한자

  상속 관계라면 다른 패키지에서도 접근 가능하도록 허용함


```java
public class FireCar extends Car { // extends 키워드 사용

    @Override
    public void soundHorn() { // 부모 기능 재정의
        System.out.println("빠아아아아아아앙!!!");
    }

    public void sprayWater() { // 자식만의 기능 추가
        System.out.println("물을 뿌립니다 =============>>");
    }
}
```

---

## 02. Super Keyword (부모 참조 - super)

자식 인스턴스 내에서 부모 인스턴스 영역에 접근하기 위해 사용하는 키워드

---

## ✔ 주요 개념 및 활용

- `super()`

  부모 생성자를 호출함

  자식 생성자의 첫 줄에 반드시 작성해야 하며, 생략 시 기본 생성자가 자동 호출됨

- `super.`

  부모 클래스의 필드나 메소드에 직접 접근할 때 사용함

  오버라이딩된 부모 메소드를 호출할 때 주로 활용됨


```java
public class Computer extends Product {

    private String cpu;
    private int ram;

    public Computer(String code, String brand, String name,
                    int price, Date date, String cpu, int ram) {

        super(code, brand, name, price, date); // 부모 생성자 호출

        this.cpu = cpu;
        this.ram = ram;
    }

    @Override
    public String getInformation() {

        return super.getInformation()
                + " [CPU=" + cpu + ", RAM=" + ram + "]";
    }
}
```

---

## 03. Method Overriding (메소드 재정의)

자식 클래스가 부모로부터 물려받은 메소드를 자신에게 맞게 다시 정의하는 것

---

## ✔ 오버라이딩 성립 조건 및 제약

- 동일성

  메소드명, 반환 타입, 매개변수 목록이 부모와 완전히 일치해야 함

- 접근 제한

  부모보다 더 좁은 범위로 변경 불가

  예: `protected → private` 불가능

- 예외 사항

  `private` 메소드(상속 불가)와 `final` 메소드(수정 불가)는 오버라이딩 불가능


```java
public class CoffeeVendingMachine extends VendingMachine {

    @Override // 오버라이딩 규칙 검사
    public String vend() {

        return "따듯한 아메리카노";
    }

    // 부모의 final 메소드는 오버라이딩 불가
    // public final void powerOn()
}
```

---

## 👀 핵심 요약

- `extends`를 사용하여 클래스 간 계층 구조 형성 및 기능 확장 가능
- `super`를 통해 부모 생성자 및 부모 멤버에 접근 가능
- 오버라이딩을 통해 다형성 구현의 기초를 만들고 메소드를 재정의함

---

# chap09-polymorphism

## 01. Polymorphism (다형성)

하나의 객체가 여러 가지 타입을 가질 수 있는 성질로,

부모 타입의 참조 변수로 자식 인스턴스를 참조하는 Java의 핵심 기술

---

## ✔ 주요 개념 및 활용

### 업캐스팅 (Upcasting)

자식 객체를 부모 타입의 변수에 할당하는 것

(자동 형변환)

### 동적 바인딩 (Dynamic Binding)

컴파일 시점에는 부모 메소드를 호출하는 것처럼 보이지만,

실행 시점(Runtime)에는 실제 객체의 오버라이딩된 메소드가 실행됨

### 다운캐스팅 & instanceof

부모 타입 변수를 다시 자식 타입으로 강제 형변환하여

자식만의 기능을 사용할 수 있음

`instanceof`를 사용하여 타입 확인 후 형변환하는 것이 안전함

```java
Animal[] animals = new Animal[2];

animals[0] = new Cat();
animals[1] = new Tiger();

for (Animal animal : animals) {

    animal.cry(); // 동적 바인딩

    if (animal instanceof Cat) {

        ((Cat) animal).jump(); // 다운캐스팅
    }
}
```

---

## 02. Abstract Class (추상 클래스)

미완성된 클래스로, 직접 객체를 생성할 수 없으며

자식 클래스에서 반드시 구현해야 할 기능을 정의하는 설계도

---

## ✔ 주요 개념 및 활용

### 추상 메소드 (Abstract Method)

몸통(`{}`)이 없는 메소드로 `abstract` 키워드를 사용함

자식 클래스는 반드시 오버라이딩해야 함

### 객체 생성 불가

추상 클래스는 `new`를 통한 객체 생성이 불가능함

하지만 다형성을 위한 부모 타입으로는 사용 가능함

```java
public abstract class Player {

    public abstract void attack(); // 추상 메소드

    public void levelUp() { // 일반 메소드

        System.out.println("레벨업!");
    }
}
```

```java
public class Warrior extends Player {

    @Override
    public void attack() {

        System.out.println("칼을 휘둘러 공격합니다!");
    }
}
```

---

## 03. Interface (인터페이스)

추상 클래스보다 더 추상화된 개념으로,

클래스 간의 표준 규격(계약)을 정의하여 결합도를 낮추는 기술

---

## ✔ 주요 개념 및 활용

### 순수 규격 정의

- 모든 메소드는 기본적으로 `public abstract`
- 모든 변수는 `public static final` 상수로 처리됨

### 다중 구현 (Multiple Implements)

클래스는 단일 상속만 가능하지만

인터페이스는 여러 개를 동시에 구현할 수 있음

```java
public interface IConnectable {

    void connect(); // public abstract

    default void showStatus() {

        System.out.println("연결 대기 중...");
    }
}
```

```java
public class Computer {

    // 인터페이스 타입 활용
    public void connectDevice(IConnectable device) {

        device.connect();
    }
}
```

---

## 👀 핵심 요약

- 다형성

  부모 타입 하나로 여러 자식 객체를 관리하여 유연성과 확장성 확보

- 추상 클래스

  공통 기능은 상속하고, 추상 메소드로 구현을 강제함

- 인터페이스

  클래스 간 규격을 정의하여 결합도를 낮추고 유연한 설계 가능

- 형변환 주의

  다운캐스팅 전에는 반드시 `instanceof`로 타입 확인 필요
