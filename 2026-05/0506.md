# 06. Static, Final & Singleton

## ✔ Static Keyword

프로그램 시작 시 **Static 영역**에 할당되어 모든 객체가 공유

- **Static 필드**
    - 클래스당 하나만 존재 (공용 변수)
    - 인스턴스마다 생성되지 않음
- **Static 메소드**
    - 객체 생성 없이 `클래스명.메소드명()`으로 호출

```java
public class UniversityStudent {
    private int personalLocker;      // 인스턴스 변수
    private static int sharedLocker; // 클래스 변수

    public void increaseSharedLocker() {
        UniversityStudent.sharedLocker++; // 클래스명으로 접근
    }
}
```

---

## ✔ Final Keyword

값이나 기능을 **변경 불가능하게 제한**

- **Final 필드**
    - 초기화 이후 값 변경 불가 (상수)
- **Final 메소드**
    - 오버라이딩 불가
- **Final 클래스**
    - 상속 불가

```java
public class Person {
    private final String ssn;

    public Person(String ssn) {
        this.ssn = ssn;
    }
}
```

---

## ✔ Singleton Pattern

어플리케이션 전체에서 **인스턴스를 하나만 생성하여 공유**

- 생성자를 `private`으로 제한
- `static 메소드`로 객체 반환

```java
public class EagerSingleton {
    private static EagerSingleton instance = new EagerSingleton();

    private EagerSingleton() {}

    public static EagerSingleton getInstance() {
        return instance;
    }
}
```

# 07. Variables & Init Block

## ✔ 변수의 종류

| 종류 | 선언 위치 | 생성 시점 | 소멸 시점 |
| --- | --- | --- | --- |
| 인스턴스 변수 | 클래스 영역 | 객체 생성 시 (Heap) | 객체 소멸 시 |
| 클래스 변수 (static) | 클래스 영역 | 프로그램 시작 시 (Static) | 프로그램 종료 시 |
| 지역 변수 | 메소드/블록 | 메소드 호출 시 (Stack) | 메소드 종료 시 |

---

## ✔ 초기화 방법

- **명시적 초기화**

  → 선언과 동시에 값 설정

- **정적 초기화 블럭 (`static {}`)**

  → 클래스 로딩 시 한 번 실행 (static 필드 초기화)

- **인스턴스 초기화 블럭 (`{}`)**

  → 객체 생성 시마다 실행 (생성자보다 먼저)

- **생성자**

  → 객체 생성 시 최종 초기화


```java
public class Product {
    private String name = "갤럭시"; // 명시적 초기화
    private static String brand;

    static { brand = "삼성"; }   // 정적 초기화 블럭

    { name = "아이폰"; }         // 인스턴스 초기화 블럭

    public Product() {
        this.name = "갤럭시S24";
    }
}
```

---

## ✔ 실행 순서

👉 **클래스 로딩(static) → 명시적 초기화 → 초기화 블럭 → 생성자**

---

## 👀 핵심 정리

- **static** → 공유
- **final** → 변경 불가
- **singleton** → 하나만 생성
- **초기화 순서** → static → 필드 → 블럭 → 생성자

# 01. Object Array (객체 배열)

동일한 타입의 여러 객체를 하나로 묶어서 **효율적으로 관리**하는 구조

---

## ✔ 객체 배열의 선언 및 생성

객체 배열을 만들면 Heap에 **객체 주소를 담는 공간**이 생성되고,

각 요소는 기본적으로 `null` 상태

```java
Car[] carArr = new Car[3];
```

⚠️ 주의:

```java
carArr[0].driveMaxSpeed(); // NullPointerException 발생
```

👉 객체 생성 안하면 예외 발생

---

## ✔ 객체 배열의 초기화

### 인덱스별 개별 초기화

```java
carArr[0] = new Car("페라리", 300);
carArr[1] = new Car("람보르기니", 350);
carArr[2] = new Car("롤스로이스", 250);
```

---

### 선언과 동시에 초기화

```java
Car[] carArr2 = {
    new Car("페라리", 300),
    new Car("람보르기니", 350),
    new Car("롤스로이스", 250)
};
```

---

## ✔ 객체 배열 활용

### 향상된 for문

```java
for (Car car : carArr2) {
    car.driveMaxSpeed();
}
```

👉 배열 안의 객체들을 한 번에 처리

---

### 최댓값 찾기 (알고리즘)

```java
Car fastestCar = carArr[0];

for (int i = 1; i < carArr.length; i++) {
    if (carArr[i].getMaxSpeed() > fastestCar.getMaxSpeed()) {
        fastestCar = carArr[i];
    }
}

System.out.println("가장 빠른 차: " + fastestCar.getModelName());
```

👉 기준 잡고 비교 → 더 크면 교체

---

## ✔ DTO (Car 클래스 구조)

데이터 전달용 객체 (캡슐화 적용)

```java
public class Car {
    private String modelName;
    private int maxSpeed;

    public Car(String modelName, int maxSpeed) {
        this.modelName = modelName;
        this.maxSpeed = maxSpeed;
    }

    public String getModelName() { return modelName; }
    public int getMaxSpeed() { return maxSpeed; }

    public void driveMaxSpeed() {
        System.out.println(modelName + "이(가) 최고시속 " + maxSpeed + "km/h로 달려갑니다.");
    }
}
```

---

## 👀 핵심 요약

- 객체 배열 = **객체 주소를 담는 배열 (초기값 null)**
- 반드시 `new`로 **각 요소 초기화 필요**
- 반복문으로 전체 처리 가능
- 알고리즘 문제에 자주 활용됨
