# chap15-enum

## 01. Enum (열거형)

서로 연관된 상수들의 집합을 의미하며, 자바에서는 `enum` 키워드를 통해 클래스처럼 사용할 수 있는 열거 타입을 제공

장점 - 가독성 향상, 타입 안전성 보장, 싱글톤 패턴 적용 등

---

## ✔ 01. Integer Enum (정수 열거 패턴의 한계)

`public static final int`를 사용하여 상수 정의 후 문제점

- 타입 안전성 부족

  단순 정수값이므로 엉뚱한 값이 전달되어도 컴파일 시점에서 확인할 수 없음

- 의미의 중복

  서로 다른 그룹의 상수가 같은 정수값을 가질 경우, 논리적으로 같다고 판단하는 오류 발생

- 가독성 저하

  상수 이름만으로 구분이 어려워 접두어(prefix)를 붙여야 하는 등 코드가 지저분해짐


```java
// Subject.java (인터페이스 내 상수 정의)
public interface Subject {
    int JAVA = 0;
    int MYSQL = 1;
    int HTML = 0; // JAVA와 값이 같음
}

// Application.java
int s1 = Subject.JAVA;
int s2 = Subject.HTML;

if(s1 == s2) { // 논리적으로 다른 과목이지만 값이 같아 '참'이 됨
    System.out.println("두 과목은 같은 과목입니다.");
}
```

---

## ✔ 02. Enum Type (열거 타입의 도입)

`enum`을 사용하여 정수 열거 패턴의 문제를 해결하고 더 강력한 기능을 사용 가능

- 타입 안전성

  정의된 Enum 타입의 상수만 인스턴스로 전달 가능

- 고유성

  각 상수는 해당 열거형의 유일한 인스턴스임이 보장됨

- 제공 메서드
    - `values()` : 모든 열거 상수를 배열로 반환
    - `name()` : 상수의 이름(문자열) 반환
    - `ordinal()` : 상수의 선언 순서(0부터 시작) 반환

```java
public enum Subjects {
    JAVA, MYSQL, JDBC, HTML, CSS
}

// 사용 예시
Subjects subject = Subjects.JAVA;
System.out.println("선택된 과목: " + subject.name());

for(Subjects s : Subjects.values()) {
    System.out.println(s.ordinal() + "번 : " + s);
}
```

## ✔ 03. Enum Grammar (열거형의 고급 문법)

Enum은 클래스와 유사하게 필드, 생성자, 메서드를 가질 수 있어 상수에 추가적인 데이터를 바인딩할 수 있음

- 데이터 바인딩

  각 상수에 설명, 코드값 등을 부여하여 관리 가능

- private 생성자

  Enum 상수는 프로그램 시작 시 한 번만 생성되므로 생성자는 외부에서 호출할 수 없음 (싱글톤)

- EnumSet

  Enum 상수를 집합(Set) 형태로 다루기 위한 전용 컬렉션으로, 메모리 효율과 성능이 매우 뛰어남


```java
public enum UserRole {
    GUEST("게스트"),
    CONSUMER("구매자"),
    ADMIN("관리자");

    private final String description;

    private UserRole(String description) { // 묵시적으로 private
        this.description = description;
    }

    public String getDescription() {
        return description;
    }
}

// EnumSet 활용
EnumSet<UserRole> roles =
        EnumSet.of(UserRole.CONSUMER, UserRole.ADMIN);
```

# 👀 핵심 요약

- 열거형 사용 이유
    - 코드의 가독성을 높이고 상수의 의미를 명확히 함
    - 컴파일 타임에 타입 체크를 수행하여 런타임 에러 방지
- 상수 이상의 가치
    - 단순한 값이 아닌 객체(Instance)로 취급되므로 메서드와 필드를 가질 수 있음
- 싱글톤(Singleton)
    - 각 상수는 JVM 내에서 유일하게 존재하므로 `==` 연산자로 안전하게 비교 가능
- 유틸리티 활용
    - `EnumSet`을 사용하여 권한 관리나 그룹화된 상수 처리를 효율적으로 수행 가능

# chap16-lambda

## 01. Lambda (람다식)

자바 8부터 도입된 기능으로, 메서드를 하나의 식(expression)으로 표현하여 코드의 간결성을 높이고 함수형 프로그래밍을 지원함

### 장점

- 코드의 간결함
- 불필요한 익명 클래스 제거
- 컬렉션 처리(Stream) 효율 증대 등

---

## ✔ 01. Intro (람다식의 기본)

메서드가 하나뿐인 인터페이스(함수형 인터페이스)의 구현을 짧게 쓰는 문법

- 함수형 인터페이스

  `@FunctionalInterface` 어노테이션을 사용하여 단 하나의 추상 메서드만 가짐을 보장하며, 이는 람다식 사용의 필수 조건임

- 구현 방식의 변화

  전통적인 클래스 구현(`implements`) → 익명 클래스 방식 → 람다식 순으로 코드가 점진적으로 간결해짐

- 지역 변수 제약 (Variable Capture)

  람다식 내부에서 사용하는 외부 지역 변수는 `final`이거나 effectively final(값의 재할당이 없는 상태)이어야 함


```java
// Functional Interface 정의
@FunctionalInterface
public interface Calculator {
    int sumTwoNumber(int a, int b);
}

// 람다식 활용
Calculator c = (x, y) -> x + y;

System.out.println(c.sumTwoNumber(10, 20));
```

---

## ✔ 02. Method Reference (메서드 참조)

람다식이 단순히 기존 메서드를 호출만 하는 경우, 이를 더욱 간결하게 표현할 수 있는 문법

- 정적 메서드 참조

  `클래스이름::정적메소드이름`

  예: `Integer::parseInt`

- 인스턴스 메서드 참조

  `참조변수::메소드이름`

  예: `System.out::println`

- 임의 객체의 인스턴스 메서드 참조

  `클래스이름::메소드이름`

  예: `String::equals`

- 생성자 참조

  `클래스이름::new`

  예: `Account::new`


```java
// 람다 표현식
Function<String, Integer> f1 =
        str -> Integer.parseInt(str);

// 메서드 참조 (정적)
Function<String, Integer> f2 =
        Integer::parseInt;

// 생성자 참조
Supplier<Account> supplier =
        Account::new;

Account acc = supplier.get();
```

---

## ✔ 03. Functional Interface (표준 함수형 인터페이스)

`java.util.function` 패키지에서 제공하는 자주 쓰이는 함수 형태를 미리 정의해둔 인터페이스

- `Consumer<T>`

  입력을 받아 소비(출력, 저장 등)하고 반환값이 없음 (`accept`)

- `Supplier<T>`

  인자 없이 호출 시 값을 생성하여 반환함 (`get`)

- `Function<T, R>`

  입력을 받아 특정 타입으로 변환하여 반환함 (`apply`)

- `Predicate<T>`

  입력을 받아 조건을 검사한 후 boolean 결과를 반환함 (`test`)

- `Operator`

  입력과 출력 타입이 같은 특수한 형태 (`UnaryOperator`, `BinaryOperator`)


```java
// 1. Consumer: 출력 또는 소비
Consumer<String> printer = str -> System.out.println(str);
printer.accept("Hello World");

// 2. Supplier: 객체 생성 및 공급
Supplier<LocalDateTime> now = () -> LocalDateTime.now();
System.out.println(now.get());

// 3. Function: 타입 변환 (String -> Integer)
Function<String, Integer> parser = Integer::parseInt;
int result = parser.apply("123");

// 4. Operator: 매개변수와 반환 타입이 동일한 연산
BinaryOperator<Integer> sum = (a, b) -> a + b;
System.out.println(sum.apply(10, 20));

// 5. Predicate: 조건 검사 및 판단
Predicate<String> isEmpty = str -> str.isEmpty();
System.out.println(isEmpty.test("")); // true
```

    - "어떻게(How)" 연산할지보다 "무엇을(What)" 수행할지에 집중하게 하여 현대적인 자바 코딩 스타일(Stream 등)의 기반이 됨

# chap17-stream

## 01. Stream (스트림)

자바 8부터 도입된 기능으로, 배열이나 컬렉션 등의 데이터를 선언적으로 처리할 수 있는 흐름(Stream)을 의미

데이터 소스를 변경하지 않고, 일회용으로 사용하며, 함수형 인터페이스를 활용해 복잡한 루프문을 간결하게 대체할 수 있음

---

## ✔ 01. Intro & Creation (스트림 개요 및 생성)

스트림 -  '데이터의 흐름', 데이터 소스(배열, 컬렉션 등)로부터 스트림을 형성하고 여러 번의 중간 연산과 하나의 최종 연산을 거쳐 결과를 도출

- 데이터 소스의 불변성

  스트림은 원본 데이터를 읽기만 할 뿐 직접 수정하지 않음

- 일회용 (Consumption)

  한 번 사용(최종 연산)한 스트림은 재사용할 수 없으며 다시 생성해야 함

- 지연 연산 (Lazy Evaluation)

  최종 연산이 수행되기 전까지 중간 연산은 실행되지 않음


```java
// 배열 및 컬렉션으로부터 스트림 생성
List<String> stringList =
        Arrays.asList("java", "oracle", "jdbc");

Stream<String> stringStream =
        stringList.stream();

// Stream.of() 활용
Stream<String> stream =
        Stream.of("hello", "world");
```

---

## ✔ 02. Intermediate & Terminal Operations (중간 연산 및 최종 연산)

스트림 -  데이터를 가공하는 중간 연산과 가공된 데이터를 소모하여 결과를 내는 최종 연산으로 나뉨.

### 중간 연산 (Intermediate Operations)

- `filter(Predicate)` : 조건에 맞는 요소만 추출
- `map(Function)` : 요소를 다른 형태나 값으로 변환
- `sorted()` : 요소 정렬
- `distinct()` : 중복 제거

### 최종 연산 (Terminal Operations)

- `forEach(Consumer)` : 각 요소에 대해 특정 작업 수행
- `collect(Collector)` : 스트림 결과를 컬렉션(List, Set 등)으로 변환
- `reduce()` : 모든 요소를 소모하여 단일 결과 도출 (합계, 최댓값 등)
- `count()`, `anyMatch()`, `allMatch()` 등

```java
// OnlineCourse 리스트에서
// 'Java'를 포함하고 통과한(isPassed) 강의 이름만 추출
List<String> passedJavaNames =
        courseList.stream()
                .filter(course ->
                        course.getName().contains("Java"))
                .filter(OnlineCourse::isPassed)
                .map(OnlineCourse::getName)
                .collect(Collectors.toList());

passedJavaNames.forEach(System.out::println);
```

---

## ✔ 03. Optional & Statistics (옵셔널 및 통계 처리)

스트림의 최종 연산 결과가 비어있을 가능성이 있을 때 `Optional`을 사용하며, 숫자 타입 스트림은 전용 메서드를 제공

- `Optional<T>`

  `null`이 발생할 수 있는 객체를 감싸는 래퍼 클래스로, `NPE(NullPointerException)` 방지

- Primitive Stream (기본형 스트림)

  `IntStream`, `LongStream`, `DoubleStream` 등을 통해 오토박싱 비용을 줄이고 `sum()`, `average()` 등의 통계 메서드 사용 가능


```java
// IntStream을 활용한 통계 계산
int sum =
        IntStream.rangeClosed(1, 10).sum(); // 1~10까지 합

// Optional 활용
Optional<OnlineCourse> firstCourse =
        courseList.stream().findFirst();

firstCourse.ifPresent(c ->
        System.out.println(c.getName()));
```

# 👀 핵심 요약

- 선언적 코드 작성
    - "어떻게" 반복할지가 아니라 "무엇을" 필터링하고 변환할지에 집중하여 가독성이 비약적으로 향상됨
- 파이프라인 구조
    - 생성 → 중간 연산(N번) → 최종 연산(1번)의 구조를 가지며, 최종 연산이 없으면 중간 연산도 수행되지 않음
- 함수형 인터페이스 결합
    - 람다식과 메서드 참조를 적극 활용하여 로직을 인자로 전달하는 현대적인 프로그래밍 방식 구현
- 병렬 처리 용이성
    - `.parallelStream()`을 호출하는 것만으로 데이터 소스를 병렬로 처리할 수 있어 대량의 데이터 처리 성능 최적화 가능

      (단, 스레드 안전성 고려 필요)
