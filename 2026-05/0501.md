## 02. dimensional array (다차원 배열)

- 2차원 이상의 배열로, 배열의 인덱스마다 또 다른 배열의 주소를 보관하는 구조
- 주로 1차원 배열 여러 개를 하나로 묶어 관리할 때 사용

---

✔ 2차원 배열의 선언 및 할당

```java
// 선언과 동시에 할당 (정변 배열)
int[][] iarr = new int[3][5]; // 3행 5열

// 선언과 동시에 초기화
int[][] iarr2 = {
    {1, 2, 3, 4, 5},
    {6, 7, 8},
    {11, 12, 13}
};
```

---

✔ 가변 배열 (Variable Array)

```java
int[][] iarr = new int[3][]; // 행의 수만 먼저 지정

iarr[0] = new int[3]; // 0행은 3열
iarr[1] = new int[2]; // 1행은 2열
iarr[2] = new int[5]; // 2행은 5열
```

---

✔ 중첩 반복문을 이용한 출력

```java
for (int i = 0; i < iarr.length; i++) {
    for (int j = 0; j < iarr[i].length; j++) {
        System.out.print(iarr[i][j] + " ");
    }
    System.out.println();
}
```

## 03. array copy (배열 복사)

- 배열 -> 참조 자료형
- 얕은 복사 - 주소값만 복사
- 깊은 복사 - 실제 데이터를 복사하여 새로운 배열을 생성

---

✔ 얕은 복사 (Shallow Copy)

```java
int[] originArr = {1, 2, 3};
int[] copyArr = originArr; // 주소값 복사

copyArr[0] = 99;
System.out.println(originArr[0]); // 99 출력 (원본 영향)
```

---

✔ 깊은 복사 (Deep Copy)

```java
int[] originArr = {1, 2, 3, 4, 5};

// 1. for문을 이용한 복사
int[] copy1 = new int[originArr.length];
for(int i = 0; i < originArr.length; i++) {
    copy1[i] = originArr[i];
}

// 2. Arrays.copyOf() 사용
int[] copy2 = java.util.Arrays.copyOf(originArr, originArr.length);

// 3. System.arraycopy() 사용
int[] copy3 = new int[originArr.length];
System.arraycopy(originArr, 0, copy3, 0, originArr.length);

// 4. clone() 사용
int[] copy4 = originArr.clone();
```
- clone()은 기본형 배열에서는 깊은 복사처럼 동작, 참조형(객체, 2차원 배열 등)에서는 얕은 복사함
---

## 04. array sort (배열 정렬)

배열에 저장된 데이터를 특정 기준(오름차순, 내림차순)에 따라 재배열하는 과정

---

✔ 선택 정렬

```java
int[] arr = {2, 5, 4, 6, 1, 3};

for (int i = 0; i < arr.length - 1; i++) {
    int minIndex = i;
    for (int j = i + 1; j < arr.length; j++) {
        if (arr[minIndex] > arr[j]) {
            minIndex = j;
        }
    }
    int temp = arr[minIndex];
    arr[minIndex] = arr[i];
    arr[i] = temp;
}
```

---

✔ 버블 정렬

```java
int[] arr = {2, 5, 4, 6, 1, 3};

for (int i = arr.length - 1; i >= 0; i--) {
    for (int j = 0; j < i; j++) {
        if (arr[j] > arr[j + 1]) {
            int temp = arr[j];
            arr[j] = arr[j + 1];
            arr[j + 1] = temp;
        }
    }
}
```

---

✔ Arrays.sort() : 로또 번호 생성

```java
// Arrays.sort() : 오름차순 정렬
int[] arr = {3, 1, 5, 2, 4};
java.util.Arrays.sort(arr);

// 중복 없는 로또 번호 생성
int[] lotto = new int[6];
for (int i = 0; i < lotto.length; i++) {
    lotto[i] = (int) (Math.random() * 45) + 1;
    for (int j = 0; j < i; j++) {
        if (lotto[i] == lotto[j]) { i--; break; }
    }
}
java.util.Arrays.sort(lotto);
```

## 01. User Defined Type (사용자 정의 자료형)

클래스 : 다양한 자료형의 데이터를 하나로 묶어 관리하기 위해 사용자가
직접 만드는 새로운 자료형

---

✔ 클래스와 객체 기본 사용법

```java
// 클래스 선언 (Member.java)
public class Member {
    String id;
    String pwd;
    String name;
    int age;
    char gender;
    String[] hobbies;
}

// 객체 생성 및 활용 (Application.java)
Member member = new Member(); // 인스턴스 생성

// 필드 접근 (도트 연산자 . 사용)
member.name = "홍길동";
member.age = 20;

System.out.println("이름: " + member.name);
```

---

✔ 사용자 정의 자료형의 필요성

* 기본 자료형: 하나의 값만 저장 가능 (int, double 등)
* 배열: 같은 자료형의 여러 데이터 저장 가능
* 클래스: 서로 다른 자료형의 데이터를 하나의 그룹으로 묶어 실세계의 객체를 표현 가능

---

## 02. Encapsulation (캡슐화)

객체의 상세한 구현 내용을 외부에 숨기고, 필요한 정보만을 인터페이스(메소드)를
통해 노출하여 데이터의 직접적인 접근을 제한하는 객체지향의 핵심 원칙

---

✔ 유지보수와 필드 직접 접근의 문제

```java
// 직접 접근 시 발생하는 위험 (Encapsulation 적용 전)
Children child = new Children();
child.name = "이순신";
child.age = -5; // 논리적으로 말이 안 되는 값도 대입 가능 (제어 불가)
```

```java
// 필드를 외부에서 직접 수정하지 못하도록 차단
public class Children {
    private String name;
    private int age;

    // 메소드를 통해 간접적으로 접근하도록 유도
    public void setAge(int age) {
        if (age > 0) {
            this.age = age;
        } else {
            System.out.println("나이는 0보다 작을 수 없습니다.");
        }
    }
}
```

---

✔ 캡슐화의 장점

* 데이터 보호: 외부에서 데이터의 무분별한 수정 방지 (유효성 검사 가능)
* 코드 독립성: 내부 로직이 변경되어도 이를 사용하는 외부 코드는 수정할 필요가 없음 (결합도 낮춤)
* 은닉화: 복잡한 내부 동작을 감추고 사용자에게는 단순한 사용법만 제공

## 02. Access Modifier (접근 제한자)

클래스나 클래스의 멤버(필드, 메소드)에 외부에서 접근할 수 있는 범위를 제한하는 키워드입니다.

---

## ✔ 접근 제한자의 종류와 범위

| 제한자           | 클래스 내부 | 동일 패키지 | 후손 클래스 | 전체(외부) |
| ------------- | ------ | ------ | ------ | ------ |
| **public**    | ✅      | ✅      | ✅      | ✅      |
| **protected** | ✅      | ✅      | ✅ (상속) | ❌      |
| **default**   | ✅      | ✅      | ❌      | ❌      |
| **private**   | ✅      | ❌      | ❌      | ❌      |

* **public** → 어디서든 접근 가능
* **protected** → 같은 패키지 + 상속 관계
* **default** → 같은 패키지만
* **private** → 클래스 내부만

---
👉 범위 크기 순서

```text
public > protected > default > private
```

## 03. Class & Object Memory (메모리 구조)

객체가 생성되고 관리되는 과정에서의 메모리 할당 원리

---

✔ Stack과 Heap 영역

* Stack (스택): 클래스 타입의 레퍼런스 변수(주소값 저장)가 생성되는 공간
* Heap (힙): new 연산자를 통해 실제 데이터가 올라가는 인스턴스가 생성되는 공간

```java
Member member = new Member();
```

---

✔ 필드 기본값 (Heap 영역의 특징)

* Heap 영역에 생성되는 인스턴스 변수(필드)는 자료형에 맞는 기본값으로 자동 초기화됩니다.

    * 정수: 0
    * 실수: 0.0
    * 참조형: null
    * 논리형: false
