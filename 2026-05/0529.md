# chap03 — Jackson & JSON

## 개념

▎ Spring Boot에서 Java 객체 ↔ JSON 간 자동 직렬화/역직렬화를 담당하는 라이브러리.

▎ `@RestController`가 내부적으로 Jackson의 `ObjectMapper`를 사용한다.

---

# 프로젝트 구조

```
Application.java       (Spring Boot 진입점)
MemberDTO.java         (직렬화 대상 도메인 객체)
JacksonController.java (Jackson 동작 확인용 컨트롤러)
api-test.http          (API 테스트 파일)
```

---

# 도메인 — MemberDTO

```java
public class MemberDTO {
    private int no;
    private String name;
    private int age;

    @JsonFormat(shape = JsonFormat.Shape.STRING,
                pattern = "yyyy-MM-dd HH:mm:ss",
                timezone = "Asia/Seoul")
    private Date enrollDate;
}
```

## @JsonFormat

날짜(Date) 타입을 JSON으로 직렬화할 때 형식을 지정하는 어노테이션.

지정하지 않으면 Unix timestamp(숫자)로 출력됨.

---

# 핵심 흐름

| 구분 | 설명 |
| --- | --- |
| 직렬화 | Java 객체 → (Jackson ObjectMapper) → JSON 문자열 |
| 역직렬화 | JSON 문자열 → (Jackson ObjectMapper) → Java 객체 |

`@RestController` 안에서 이 두 변환은 자동으로 처리된다.

---

# API 엔드포인트

| HTTP | URL | 설명 |
| --- | --- | --- |
| GET | `/api/v1/members` | `List<MemberDTO>` → JSON 배열 |
| GET | `/api/v1/members/{memberNo}` | `MemberDTO` → JSON 객체 |
| POST | `/api/v1/members` | JSON 바디 → `MemberDTO` 역직렬화 후 저장 |
| GET | `/api/v1/members/response-wrapper` | `Map`으로 message + data 묶어 응답 |
| GET | `/api/v1/members/object-mapper` | `ObjectMapper` 직접 사용해 JSON 문자열 변환 확인 |

---

# 코드 포인트

## 자동 직렬화 (List → JSON 배열)

```java
@GetMapping
public ResponseEntity<List<MemberDTO>> findMembers() {
    return ResponseEntity.ok(memberList);  // Jackson이 알아서 JSON 배열로 변환
}
```

---

## 자동 역직렬화 (JSON → DTO)

```java
@PostMapping
public ResponseEntity<Map<String, Object>> createMember(@RequestBody MemberDTO member) {
    memberList.add(member);  // 요청 Body JSON이 MemberDTO로 자동 변환됨
    ...
}
```

---

## 응답 래핑 패턴 (message + data)

```java
@GetMapping("/response-wrapper")
public ResponseEntity<Map<String, Object>> findMembersWithWrapper() {
    Map<String, Object> response = new LinkedHashMap<>();
    response.put("message", "회원 목록 조회 성공");
    response.put("count", memberList.size());
    response.put("data", memberList);
    return ResponseEntity.ok(response);
}
```

---

## ObjectMapper 직접 사용

```java
ObjectMapper mapper = new ObjectMapper();
String jsonString = mapper.writeValueAsString(memberList);  // Java 객체 → JSON 문자열
```

▎ `@RestController`에서는 반환 타입만 맞추면 Jackson이 자동 처리하므로,

▎ `ObjectMapper`를 직접 쓸 일은 거의 없다. 원리 확인용 예제.

# 핵심 요약

| 항목 | 내용 |
| --- | --- |
| Jackson | Spring Boot 기본 내장 JSON 라이브러리 |
| 직렬화 | `@RestController` 반환값 → JSON 자동 변환 |
| 역직렬화 | `@RequestBody` → DTO 자동 변환 |
| `@JsonFormat` | 날짜 출력 형식 지정 (기본값은 Unix timestamp) |
| `ObjectMapper` | Jackson의 핵심 클래스, 수동 변환 시 직접 사용 |
| 응답 래핑 | `Map<String, Object>`로 message·data 구조화 |

# 커스텀 예외 클래스

## 공통 패턴

```java
public class MemberNotFoundException extends RuntimeException {
    public MemberNotFoundException(String message) {
        super(message);
    }
}
```

| 클래스 | 상위 클래스 | 사용 상황 |
| --- | --- | --- |
| `MemberNotFoundException` | `RuntimeException` | 해당 번호의 회원이 존재하지 않을 때 |
| `InvalidMemberRequestException` | `RuntimeException` | 요청 값이 유효하지 않을 때 |

`RuntimeException`을 상속하면 Unchecked Exception이 되어 메서드 시그니처에 `throws` 선언 없이도 던질 수 있다.

# 코드 포인트

## 예외 발생 흐름

```java
@GetMapping("/{memberNo}")
public ResponseEntity<MemberDTO> findMember(@PathVariable int memberNo) {
    MemberDTO foundMember = members.stream()
            .filter(member -> member.getNo() == memberNo)
            .findFirst()
            .orElseThrow(() -> new MemberNotFoundException(memberNo + "번 회원을 찾을 수 없습니다."));

    return ResponseEntity.ok(foundMember);
}
```

### orElseThrow()

`Optional`이 비어있을 때 예외를 던지는 메서드.

스트림으로 조회 → 없으면 커스텀 예외 → 호출자(혹은 전역 핸들러)가 처리.

---

# 핵심 요약

| 항목 | 내용 |
| --- | --- |
| 커스텀 예외 | `RuntimeException` 상속 → Unchecked, throws 불필요 |
| `orElseThrow()` | `Optional` 비어있을 때 예외 투척 |
| 현재 동작 | 예외 발생 시 Spring 기본 500 응답 |
| 다음 단계 | `@ControllerAdvice` + `@ExceptionHandler`로 전역 예외 처리 |
