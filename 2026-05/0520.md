# 📌 [Servlet & JSP]

## 1. Servlet 개요 및 웹 통신 구조 (01._개요.pdf)

- **서블릿(Servlet)**: Server + Applet의 합성어로, 자바(JAVA) 언어를 사용하여 사용자의 요청을 받아 동적인 웹 페이지를 생성하고 처리 결과를 전송하는 서버측 프로그램(Class 파일)
- **CGI vs WAS**:
    - **CGI(Common Gateway Interface)**: 사용자 요청이 있을 때마다 새로운 프로세스(프로그램)를 생성하여 1:1로 매칭 실행 ➔ 대용량 요청 시 서버 과부하 발생.
    - **WAS(Web Application Server)**: 동일한 프로그램에 대한 다수 요청을 한 개의 프로세스 내에서 **멀티스레딩(Multi-Threading)** 방식으로 처리 ➔ 자원 효율성 극대화.
- **웹 통신 흐름 (Web Architecture)**:

  `Client (브라우저)` ➔ `Web Server (Static HTML 처리)` ➔ `WAS (Dynamic Servlet/JSP 처리)` ➔ `DB Server`


## 2. Servlet Container & LifeCycle (03._Servlet_LifeCycle.pdf)

- **서블릿 컨테이너(Servlet Container)**: 서블릿의 생명주기를 관리하고, `HttpServletRequest`/`HttpServletResponse` 객체를 생성하며 요청에 따른 멀티스레딩 환경을 지원
- **서블릿 생명주기 (LifeCycle)**:
1. **인스턴스 생성 및 초기화 (`init()`)**: 클라이언트의 **첫 요청 시** 딱 한 번 실행되며 서블릿 객체를 메모리에 로드
2. **요청 서비스 수행 (`service()`)**: 클라이언트의 요청이 들어올 때마다 호출되며, 요청 방식(GET/POST)에 따라 `doGet()` 또는 `doPost()`를 분기 처리
3. **자원 해제 및 소멸 (`destroy()`)**: 서블릿 수정, 서버 종료 등 애플리케이션이 종료될 때 컨테이너가 호출하여 자원을 반납
- **초기화 파라미터 설정**:
    - **ServletConfig**: 특정 서블릿 내에서만 사용할 초기값을 `<servlet>` 태그 내부에 설정.
    - **ServletContext**: 모든 서블릿(애플리케이션 전체)이 공용으로 사용할 초기값을 `<web-app>` 내부에 설정.

## 3. Servlet Method & 데이터 전송 (04._Servlet_Method.pdf)

- **데이터 전송 방식(Method) 비교**:
    - **GET 방식**: URL 창의 쿼리스트링(`?key=value&...`)을 통해 데이터를 전송합니다. 주로 조회/검색에 사용되며, 크기 제한이 있고 보안에 취약(기본값)
    - **POST 방식**: HTTP Header(Body)에 포함하여 데이터를 전송합니다. 크기 제한이 없고 패킷이 노출되지 않아 보안성이 뛰어남
- **핵심 매개변수 객체**:
    - **HttpServletRequest**: 클라이언트의 요청 정보(파라미터, 쿠키, 세션 등)를 담고 있는 객체.
    - **HttpServletResponse**: 클라이언트에게 보낼 응답 정보(MIME 타입, 인코딩, 스트림 등)를 설정하는 객체.
- **페이지 이동 기술 비교 (중요)**:
    - **sendRedirect**: 클라이언트 브라우저에게 "다른 페이지로 재요청하라(301/302)"고 지시. **URL 주소가 변경**되며, 새로운 요청이 발생하므로 이전 Request 객체의 데이터는 유지되지 않음
    - **Forward (RequestDispatcher)**: 컨테이너 내부에서 다른 서블릿/JSP로 요청을 위임. 브라우저는 이동 사실을 모르기 때문에 **URL 주소가 변하지 않으며**, 원래 가지고 있던 **Request와 Response 객체가 그대로 유지**

## 📝 핵심 개념

### 서블릿(Servlet)이란?

- Serve + Applet: 웹 서버에서 실행되는 작은 애플리케이션
- HttpServlet: HTTP 요청을 처리하기 위해 상속받는 클래스
- 관리 주체: 톰캣(Tomcat)과 같은 서블릿 컨테이너가 서블릿의 생성부터 소멸까지 관리 (Singleton 패턴)

### 서블릿 생명주기 (Life Cycle)

- 객체 생성(Constructor): 서블릿 객체가 메모리에 로드될 때 호출됨
- 초기화(init): 객체 생성 후 최초 1회만 호출되어 설정 작업 수행
- 서비스(service): 요청이 올 때마다 호출되며 요청 방식(GET, POST 등)에 따라 doGet, doPost로 분기 처리
- 소멸(destroy): 서버 종료나 서블릿 교체 시 호출되어 자원 해제

## 💻 주요 코드

### LifeCycleTestServlet.java

생명주기 메소드가 호출되는 순서를 콘솔 로그로 확인

```java
@WebServlet(value = "/lifecycle", loadOnStartup = 1)
public class LifeCycleTestServlet extends HttpServlet {

    // 1. 생성자: 서블릿 인스턴스화
    public LifeCycleTestServlet() {
        System.out.println("constructor 생성자!");
    }

    // 2. init(): 최초 1회 초기화
    @Override
    public void init(ServletConfig config) throws ServletException {
        System.out.println("init() 호출");
    }

    // 3. service(): 요청마다 실행 (핵심 로직 처리)
    @Override
    protected void service(HttpServletRequest req, HttpServletResponse resp) {
        System.out.println("service() 호출");
    }

    // 4. destroy(): 서블릿 종료 시 호출
    @Override
    public void destroy() {
        System.out.println("destroy() 호출");
    }
}
```

- `@WebServlet`: 서블릿 등록 및 URL 매핑 설정함
- `loadOnStartup = 1`: 서버 기동 시 우선적으로 서블릿을 미리 생성하도록 설정함

### 실행 흐름

- 서버 시작: `Constructor → init()` 실행됨
- `/lifecycle` 접속: 접속할 때마다 `service()` 실행됨
- 서버 종료: `destroy()` 실행되어 서블릿 소멸됨

## 🟢 Section 01. 서비스 메소드 (Service Method)

서블릿 컨테이너가 요청을 받았을 때 실행되는 핵심 로직

### 핵심 개념

- `service()`: 모든 HTTP 요청(GET, POST, PUT, DELETE 등)이 가장 먼저 거쳐가는 메소드
- 분기 처리: `request.getMethod()`를 통해 요청 방식을 확인하고, `doGet()` 또는 `doPost()`로 처리를 위임

### 💡 주요 코드 (ServiceMethodTestServlet)

```java
@WebServlet("/request-service")
public class ServiceMethodTestServlet extends HttpServlet {

    @Override
    protected void service(HttpServletRequest req, HttpServletResponse resp) {

        String httpMethod = req.getMethod(); // 요청 방식 확인

        if ("GET".equals(httpMethod)) {
            doGet(req, resp);

        } else if ("POST".equals(httpMethod)) {
            doPost(req, resp);
        }
    }
}
```

## 🔵 Section 02. 요청 파라미터 (Request Parameter)

클라이언트가 보낸 데이터를 서버에서 추출하고 응답하는 과정

### 주요 API 메소드

| 메소드 | 설명 |
| --- | --- |
| `getParameter(key)` | 지정한 키에 해당하는 단일 값(String)을 가져옴 |
| `getParameterValues(key)` | 체크박스 등 다중 값(String[])을 배열 형태로 가져옴 |
| `getHeader(key)` | HTTP 요청 헤더(User-Agent 등) 정보를 가져옴 |
| `setContentType(type)` | 응답할 문서의 MIME 타입과 인코딩을 설정 |
| `getWriter()` | 브라우저에 보낼 출력 스트림을 생성 |

    // 2. 응답 설정

## 📝 핵심 개념

### JSON (JavaScript Object Notation)이란?

- 데이터를 교환하기 위한 경량의 텍스트 기반 형식
- 키(Key)와 값(Value)의 쌍으로 이루어져 있으며, 언어에 독립적

### 직렬화 & 역직렬화 (Serialization & Deserialization)

- 역직렬화(Deserialization): 클라이언트가 보낸 JSON 문자열을 자바 객체(DTO)로 변환하는 과정
- 직렬화(Serialization): 자바 객체를 클라이언트가 이해할 수 있는 JSON 문자열로 변환하는 과정
- Jackson Library: 자바에서 JSON 처리를 돕는 대표적인 라이브러리로 ObjectMapper 클래스를 주로 사용함

### JsonTestServlet.java

ObjectMapper를 이용한 JSON 처리 로직

```java
@WebServlet("/json")
public class JsonTestServlet extends HttpServlet {

    @Override
    protected void doPost(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {

        // 1. 요청 바디(Body)에서 JSON 문자열 읽기
        BufferedReader reader = req.getReader();
        StringBuilder sb = new StringBuilder();

        String line;
        while ((line = reader.readLine()) != null) {
            sb.append(line);
        }

        // 2. Jackson ObjectMapper 생성
        ObjectMapper mapper = new ObjectMapper();

        // 3. 역직렬화: JSON 문자열 -> 자바 객체(UserDTO)
        UserDTO user = mapper.readValue(sb.toString(), UserDTO.class);

        // 4. 응답 설정 (MIME 타입: application/json)
        resp.setContentType("application/json; charset=UTF-8");
        PrintWriter out = resp.getWriter();

        // 5. 직렬화: 자바 객체 -> JSON 문자열 변환 및 전송
        String jsonResponse = mapper.writeValueAsString(user);
        out.print(jsonResponse);

        out.close();
    }
}
```

## 📝 요약

1. 데이터 추출: `req.getReader()`를 통해 HTTP 요청 바디에 담긴 JSON 문자열을 직접 읽어옴
2. 역직렬화: `mapper.readValue(jsonStr, Class)`를 사용하여 JSON을 자바 객체로 변환
3. 직렬화: `mapper.writeValueAsString(Object)`를 사용하여 자바 객체를 JSON 문자열로 변환
4. 응답 설정: JSON으로 응답할 때는 반드시`resp.setContentType("application/json")`을 설정

## 📝 핵심 개념

### JSON (JavaScript Object Notation) :

- 데이터를 구조적으로 표현하기 위한 가벼운 텍스트 형식의 표준
- 키-값(Key-Value) 쌍으로 구성되며, 다양한 프로그래밍 언어에서 지원

### Jackson Library :

- 자바에서 JSON을 처리하기 위한 가장 대중적인 라이브러리
- `ObjectMapper`: JSON 문자열을 자바 객체로 변환하거나, 자바 객체를 JSON 문자열로 변환하는 핵심 클래스

    - `writeValue()`: Java Object → JSON(Output) (직렬화)

### MemoAPIServlet.java

JSON API를 제공하는 서블릿 로직

```java
@WebServlet("/api/memos")
public class MemoAPIServlet extends HttpServlet {

    private final ObjectMapper mapper = new ObjectMapper();
    private final List<MemoDTO> memos = new ArrayList<>();

    // 1. GET: 목록 조회 (Java List -> JSON Array)
    @Override
    protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws IOException {

        resp.setContentType("application/json; charset=UTF-8");
        mapper.writeValue(resp.getWriter(), memos);
    }

    // 2. POST: 데이터 등록 (JSON Body -> Java Object)
    @Override
    protected void doPost(HttpServletRequest req, HttpServletResponse resp) throws IOException {

        resp.setContentType("application/json; charset=UTF-8");

        // 요청 본문의 JSON을 DTO 객체로 변환
        MemoDTO requestMemo = mapper.readValue(req.getReader(), MemoDTO.class);

        MemoDTO savedMemo = new MemoDTO(sequence++, requestMemo.getContent());
        memos.add(savedMemo);

        resp.setStatus(HttpServletResponse.SC_CREATED);
        mapper.writeValue(resp.getWriter(), savedMemo);
    }
}
```

## 📝 요약

1. 응답 설정: JSON을 응답할 때는 반드시 `resp.setContentType("application/json; charset=UTF-8")`을 설정해야 합니다.
2. 직렬화(Serialization): `mapper.writeValue(output, object)`를 사용하여 자바 객체를 JSON으로 변환해 클라이언트로 보냅니다.
3. 역직렬화(Deserialization): `mapper.readValue(input, Class)`를 사용하여 클라이언트가 보낸 JSON 데이터를 자바 객체로 변환합니다.
4. 클라이언트 통신: 브라우저에서는 `fetch()` API를 통해 비동기적으로 JSON 데이터를 주고받으며 화면을 갱신합니다.
