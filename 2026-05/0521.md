## 1. Servlet 개요 및 웹 통신 구조

- 서블릿(Servlet): 자바 기반 서버 사이드 프로그램으로 동적 웹 페이지 처리
- WAS (Web Application Server): 다수 요청을 멀티스레드로 처리 (CGI 단점 개선)
- 통신 구조

  Client ↔ Web Server(Static) ↔ WAS(Servlet) ↔ DB


---

## 2. 서블릿 컨테이너 및 생명주기 (LifeCycle)

서블릿: Tomcat 같은 컨테이너가 생명주기를 관리

1. **init()**
    - 최초 요청 시 1회 실행
    - 초기화 작업 수행
    - `@WebServlet(loadOnStartup=1)` → 서버 실행 시 즉시 생성
2. **service()**
    - 모든 요청마다 실행
    - HTTP Method에 따라 `doGet()`, `doPost()`로 분기
3. **destroy()**
    - 서버 종료 시 1회 실행
    - 자원 해제

## 1) Backend: MemoAPIServlet.java

```java
@WebServlet("/api/memos")
public class MemoAPIServlet extends HttpServlet {

    private final ObjectMapper mapper = new ObjectMapper();
    private static final List<MemoDTO> memos = new ArrayList<>();

    // GET: 전체 조회
    @Override
    protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws IOException {
        resp.setContentType("application/json; charset=UTF-8");
        mapper.writeValue(resp.getWriter(), memos);
    }

    // POST: 등록
    @Override
    protected void doPost(HttpServletRequest req, HttpServletResponse resp) throws IOException {
        req.setCharacterEncoding("UTF-8");
        resp.setContentType("application/json; charset=UTF-8");

        MemoDTO requestMemo = mapper.readValue(req.getReader(), MemoDTO.class);

        MemoDTO savedMemo = new MemoDTO(sequence++, requestMemo.getContent());
        memos.add(savedMemo);

        resp.setStatus(HttpServletResponse.SC_CREATED);
        mapper.writeValue(resp.getWriter(), savedMemo);
    }
}
```

## 2) Frontend: memoApi.js

```jsx
// GET: 목록 조회
export async function getMemos() {
    const response = await fetch('/api/memos');
    if (!response.ok) throw new Error('조회 실패');
    return response.json();
}

// POST: 등록
export async function createMemo(content) {
    const response = await fetch('/api/memos', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ content })
    });

    if (!response.ok) throw new Error('등록 실패');
    return response.json();
}
```

## 🔄 데이터 흐름 (Data Communication Flow)

애플리케이션의 핵심 동작:

클라이언트(브라우저) → 백엔드(서블릿) → 데이터베이스(DB) 순으로 이어지는 비동기 통신

---

### 1. 메모 조회 (Read Flow)

1. Frontend: useEffect가 실행되며 fetchMemos() 호출 → memoApi.js에서 GET /api/memos 요청
2. Backend: MemoApiServlet이 요청을 받아 MemoService → MemoDAO 호출
3. Database: SELECT 실행 후 ResultSet 반환
4. Backend: List 변환 → Jackson으로 JSON Array 응답
5. Frontend: Zustand store(memos)에 저장 → UI 자동 갱신

---

### 2. 메모 등록 (Create Flow)

1. Frontend: addMemo(content) → POST /api/memos (JSON Body 포함)
2. Backend: MemoApiServlet이 JSON을 MemoDTO로 역직렬화 → Service 전달
3. Database: INSERT 실행 + 생성된 PK 반환
4. Backend: 저장된 데이터 다시 조회 후 JSON Object 응답
5. Frontend: `[...state.memos, savedMemo]`로 UI 즉시 반영

## 2. memoStore.js (State Layer - Zustand)

- 역할
    - 전역 상태(State) 관리
    - 비동기 Action 처리
    - UI 자동 리렌더링 유도

| 상태(State) | 역할 |
| --- | --- |
| memos | 메모 목록 |
| loading | 로딩 상태 |
| error | 에러 메시지 |

| Action | 역할 |
| --- | --- |
| fetchMemos() | 서버 데이터 조회 |
| addMemo() | 새 메모 추가 |

```jsx
import { create } from "zustand";
import { getMemos, createMemo } from "../api/memoApi";

export const useMemoStore = create((set) => ({

    // 상태(State)
    memos: [],
    loading: false,
    error: '',

    // 메모 목록 조회
    fetchMemos: async () => {

        set({
            loading: true,
            error: ''
        });

        try {

            const data = await getMemos();

            // 상태 변경 -> 자동 리렌더링
            set({
                memos: data
            });

        } catch (err) {

            set({
                error: err.message
            });

        } finally {

            set({
                loading: false
            });
        }
    },

    // 메모 등록
    addMemo: async (content) => {

        try {

            const savedMemo =
                    await createMemo(content);

            // 불변성 유지
            set((state) => ({
                memos: [
                    ...state.memos,
                    savedMemo
                ]
            }));

        } catch (err) {

            set({
                error: err.message
            });
        }
    }
}));
```

# 🔴 Backend  (Servlet-Service-DAO)

Controller(Servlet) → Service → DAO로 이어지는 계층 구조를 가지며, 각 층의 역할이 명확히 구분

---

## 1. MemoApiServlet (Controller Layer)

- 역할
    - HTTP 요청 수신
    - JSON 역직렬화
    - Service 호출
    - 결과 JSON 직렬화 응답

```java
@WebServlet("/api/memos")
public class MemoApiServlet extends HttpServlet {

    private final ObjectMapper mapper = new ObjectMapper();
    private final MemoService memoService = new MemoService();

    // GET: 메모 전체 조회
    @Override
    protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws IOException {

        resp.setContentType("application/json; charset=UTF-8");

        List<MemoDTO> memos = memoService.findAllMemos();

        // List -> JSON Array
        mapper.writeValue(resp.getWriter(), memos);
    }

    // POST: 새 메모 등록
    @Override
    protected void doPost(HttpServletRequest req, HttpServletResponse resp) throws IOException {

        resp.setContentType("application/json; charset=UTF-8");

        // JSON Body -> DTO 객체
        MemoDTO requestMemo =
                mapper.readValue(req.getReader(), MemoDTO.class);

        MemoDTO savedMemo =
                memoService.registMemo(requestMemo.getContent());

        resp.setStatus(HttpServletResponse.SC_CREATED);

        // DTO -> JSON Object
        mapper.writeValue(resp.getWriter(), savedMemo);
    }
}
```

## 2. MemoService (Service Layer)

- 역할
    - 비즈니스 로직 수행
    - DB Connection 관리
    - 트랜잭션 처리
    - 자원 해제

| 역할 | 설명 |
| --- | --- |
| Connection 생성 | getConnection() |
| DAO 호출 | 실제 DB 작업 위임 |
| 트랜잭션 관리 | commit / rollback |
| 자원 정리 | close(con) |

```java
public class MemoService {

    private final MemoDAO memoDAO = new MemoDAO();

    // 전체 조회
    public List<MemoDTO> findAllMemos() {

        Connection con = getConnection();

        try {
            return memoDAO.selectAllMemos(con);

        } finally {
            close(con);
        }
    }

    // 메모 등록
    public MemoDTO registMemo(String content) {

        Connection con = getConnection();

        try {

            MemoDTO memo =
                    memoDAO.insertMemo(con, content);

            commit(con);

            return memo;

        } catch (Exception e) {

            rollback(con);

            throw e;

        } finally {
            close(con);
        }
    }
}
```
