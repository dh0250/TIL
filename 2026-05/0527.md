# chap05 - Spring AOP (Aspect-Oriented Programming)

---

# 📐 AOP 개념

> 애플리케이션 전반에 반복 등장하는 공통 관심사를 핵심 비즈니스 로직에서 분리하여, 필요한 실행 지점에 자동으로 끼워 넣는 방식.

---

# 관심사의 분리

| 구분                             | 설명               | 예시                                  |
| ------------------------------ | ---------------- | ----------------------------------- |
| 핵심 관심사 (Core Concern)          | 실제 업무 기능         | `selectMembers()`, `selectMember()` |
| 공통 관심사 (Cross-Cutting Concern) | 여러 곳에 반복되는 부가 기능 | 로깅, 트랜잭션, 보안                        |

---

# AOP 주요 용어

| 용어        | 설명                                                        |
| --------- | --------------------------------------------------------- |
| Aspect    | 공통 관심사를 모아 놓은 클래스 (`@Aspect`)                             |
| Pointcut  | Advice를 적용할 실행 지점을 선택하는 조건식                               |
| JoinPoint | Advice가 실행되는 실제 시점 (메서드 호출 등)                             |
| Advice    | JoinPoint에서 실행되는 공통 동작 (`@Before`, `@After`, `@Around` 등) |
| Proxy     | 실제 객체 앞에 Spring이 세우는 대리 객체                                |

---

# 동작 흐름

```text id="w4y0lc"
클라이언트
```

# MemberDAO.java

```java id="v7n1xr"
@Repository  // DB 커넥션을 가지는 계층
public class MemberDAO {

    private final Map<Long, MemberDTO> memberMap;

    public MemberDAO() {
        memberMap = new HashMap<>();
        memberMap.put(1L, new MemberDTO(1L, "신짱구"));
        memberMap.put(1L, new MemberDTO(2L, "신짱아"));
    }

    public Map<Long, MemberDTO> selectMembers() {
        return memberMap;
    }

    public MemberDTO selectMember(Long id) {
        MemberDTO result = memberMap.get(id);

        if (result == null) {
            throw new RuntimeException(
                "해당하는 id의 회원이 없습니다."
            );
        }

        return result;
    }
}
```

# MemberService.java

```java id="y5r9mq"
@Service  // DAO를 호출하는 비즈니스 로직 계층
public class MemberService {

    private final MemberDAO memberDAO;

    // 생성자 하나 → @Autowired 생략 가능
    public MemberService(MemberDAO memberDAO) {
        this.memberDAO = memberDAO;
    }

    public Map<Long, MemberDTO> selectMembers() {

        // 공통 관심사(로그 출력)는 Aspect로 이전
        return memberDAO.selectMembers();   // 핵심 관심사만 남음
    }

    public MemberDTO selectMember(Long id) {
        return memberDAO.selectMember(id);
    }
}
```

# LoggingAspect.java

```java id="m3v8xw"
@Aspect     // AOP 클래스 선언
@Component  // 컨테이너에 Bean으로 등록
public class LoggingAspect {

    // 어떤 메서드에 공통 기능을 적용할지 결정하는 조건식
    // → section01 패키지 내 이름이 "Service"로 끝나는
    //   클래스의 모든 메서드, 모든 파라미터
    @Pointcut("execution(* com.ohgiraffers.section01.*Service.*(..))")
    public void logPointCut() {}

    // logPointCut() 조건에 해당하는 메서드 실행 "전"에 동작
    @Before("LoggingAspect.logPointCut()")
    public void logBefore(JoinPoint joinPoint) {

        // 실행 대상 객체 정보
        System.out.println(
            "Before joinPoint.getTarget() : "
            + joinPoint.getTarget()
        );

        // 실행 대상 메서드 시그니처 정보
        System.out.println(
            "Before joinPoint.getSignature() : "
            + joinPoint.getSignature()
        );
    }
}
```

# Application.java

```java id="o5f0sq"
ApplicationContext context =
    new AnnotationConfigApplicationContext(
        "com.ohgiraffers.section01"
    );

MemberService memberService =
    context.getBean("memberService", MemberService.class);

System.out.println("====== selectMembers ======");
System.out.println(memberService.selectMembers());

System.out.println("====== selectMember =======");
System.out.println(memberService.selectMember(1L));
```

# ✅ 핵심 요약

| 항목                        | 내용                                    |
| ------------------------- | ------------------------------------- |
| AOP                       | 공통 관심사를 핵심 로직에서 분리해 자동 삽입             |
| `@Aspect`                 | 공통 기능을 담는 클래스임을 선언                    |
| `@Pointcut`               | Advice를 적용할 메서드를 선택하는 표현식             |
| `@Before`                 | 대상 메서드 실행 전에 Advice 동작                |
| JoinPoint                 | 현재 실행 지점의 대상 객체·메서드 정보를 담은 객체         |
| Proxy                     | Spring이 `@Aspect` 대상 앞에 자동 생성하는 대리 객체 |
| `@EnableAspectJAutoProxy` | AOP Proxy 자동 생성 활성화 (설정 클래스에 선언)      |

# ⚠️ Advice 종류 비교

| 어노테이션             | 실행 시점                |
| ----------------- | -------------------- |
| `@Before`         | 메서드 실행 전             |
| `@After`          | 메서드 실행 후 (예외 여부 무관)  |
| `@AfterReturning` | 메서드가 정상 반환 후         |
| `@AfterThrowing`  | 메서드에서 예외 발생 후        |
| `@Around`         | 실행 전·후 모두 제어 (가장 강력) |

# ⚠️ `@EnableAspectJAutoProxy` 옵션

```java id="i1r7mz"
// 인터페이스가 없는 클래스(CGLIB 프록시)도 AOP 적용 가능하게 설정
@EnableAspectJAutoProxy(proxyTargetClass = true)
```

```text id="h5d9vu"
proxyTargetClass = false (기본값)
→ JDK 동적 프록시, 인터페이스 필수

proxyTargetClass = true
→ CGLIB 프록시, 인터페이스 없어도 적용 가능
```
