# chap03 - Spring DI — `@Primary` / `@Qualifier` / Collection 주입

---

## 공통 — Domain & Interface

> 세 가지 주입 해결 방식이 공유하는 공통 클래스.
>

---

## Pokemon.java (인터페이스)

```java
public interface Pokemon {

    void attack();
}
```

---

## 구현체 구조

```
Pokemon (interface)
    ├── Pikachu    (@Component)
    ├── Squirtle   (@Component)
    └── Charmander (@Component + @Primary)
```

---

## Pikachu.java

```java
@Component
public class Pikachu implements Pokemon {

    @Override
    public void attack() {
        System.out.println("피카츄 백만볼트!!");
    }
}
```

---

## Squirtle.java

```java
@Component
public class Squirtle implements Pokemon {

    @Override
    public void attack() {
        System.out.println("꼬부기 물대포!");
    }
}
```

---

## Charmander.java

```java
@Component
@Primary   // 동일 타입 Bean이 여러 개일 때 우선 선택
public class Charmander implements Pokemon {

    @Override
    public void attack() {
        System.out.println("파이리 화염방사!");
    }
}
```

---

## `@Primary`

> 같은 타입의 Bean이 여러 개 등록되어 있을 때,
>
>
> `@Autowired` 주입 대상을 자동으로 이 Bean으로 결정
>

---

## 🟢 Section 02-1 — `@Primary`

> 같은 인터페이스를 구현한 Bean이 여러 개일 때,
>
>
> `@Primary`가 붙은 Bean을 우선 주입.
>

---

### 문제 상황

```
Pokemon 타입 Bean이 3개
→ Pikachu, Squirtle, Charmander

@Autowired만 사용하면
→ NoUniqueBeanDefinitionException 발생
```

---

### 해결

```
@Primary가 붙은 Charmander 자동 선택
```

---

## PokemonService.java

```java
@Service("pokemonServicePrimary")
public class PokemonService {

    private final Pokemon pokemon;

    @Autowired
    public PokemonService(Pokemon pokemon) {

        // Pokemon 타입 Bean 중
        // @Primary 가 붙은 Charmander 주입
        this.pokemon = pokemon;
    }

    public void pokemonAttack() {
        pokemon.attack();
    }
}
```

---

## Application.java

```java
ApplicationContext context =
    new AnnotationConfigApplicationContext(
        "com.ohgiraffers.section02"
    );

PokemonService pokemonService =
    context.getBean(
        "pokemonServicePrimary",
        PokemonService.class
    );

pokemonService.pokemonAttack();

// 출력: 파이리 화염방사!
```

---

## 🟡 Section 02-2 — `@Qualifier`

> Bean 이름을 직접 지정해서 특정 구현체를 명시적으로 주입.
>

---

## 특징

- `@Autowired`만으로 해결이 안 될 때 Bean 이름으로 대상을 특정
- `@Primary`보다 우선순위 높음
- 필드·생성자·세터 어디에도 적용 가능
- `@Component`의 기본 Bean 이름 = 클래스명 첫 글자 소문자

```
Pikachu    → "pikachu"
Squirtle   → "squirtle"
Charmander → "charmander"
```

---

## PokemonService.java

```java
@Service("pokemonServiceQualifier")
public class PokemonService {

    @Autowired
    @Qualifier("pikachu")   // Bean 이름이 "pikachu"인 Bean 주입
    private Pokemon pokemon;

    public void pokemonAttack() {
        pokemon.attack();
    }
}
```

---

## Application.java

```java
ApplicationContext context =
    new AnnotationConfigApplicationContext(
        "com.ohgiraffers.section02"
    );

PokemonService pokemonService =
    context.getBean(
        "pokemonServiceQualifier",
        PokemonService.class
    );

pokemonService.pokemonAttack();

// 출력: 피카츄 백만볼트!!
```

---

## 🔴 Section 02-3 — Collection 주입

> 같은 타입의 Bean 전체를 List로 한꺼번에 주입.
>

---

## 특징

- `List<Pokemon>` 타입으로 선언하면

  Spring이 해당 타입을 구현한 모든 Bean을 자동 수집

- `@Primary`, `@Qualifier` 없이도 충돌 없이 동작
- `forEach + 메서드 참조`로 일괄 호출 가능

---

## PokemonService.java

```java
@Service("pokemonServiceCollection")
public class PokemonService {

    private List<Pokemon> pokemonList;

    @Autowired
    public PokemonService(List<Pokemon> pokemonList) {

        // Pokemon 타입 Bean 전체 주입
        // [Pikachu, Squirtle, Charmander]
        this.pokemonList = pokemonList;
    }

    public void pokemonAttack() {

        // 메서드 참조로 일괄 호출
        pokemonList.forEach(Pokemon::attack);
    }
}
```

---

## Application.java

```java
ApplicationContext context =
    new AnnotationConfigApplicationContext(
        "com.ohgiraffers.section02"
    );

PokemonService pokemonService =
    context.getBean(
        "pokemonServiceCollection",
        PokemonService.class
    );

pokemonService.pokemonAttack();
```

### 출력

```
피카츄 백만볼트!!
꼬부기 물대포!
파이리 화염방사!
```

---

## ✅ 핵심 요약

| 방식 | 어노테이션 | 역할 | 우선순위 |
| --- | --- | --- | --- |
| 기본 타입 주입 | `@Autowired` | 타입이 1개일 때 자동 주입 | — |
| 우선 Bean 지정 | `@Primary` | 같은 타입 중 기본값 지정 | 낮음 |
| 이름으로 특정 | `@Qualifier` | Bean 이름으로 대상 명시 | 높음 |
| 전체 수집 | `List<T>` | 동일 타입 Bean 전부 주입 | — |

---

## ⚠️ 중복 Bean 주입 오류 해결 흐름

```
Pokemon 타입 Bean 3개 존재
        ↓
@Autowired 단독 사용
        ↓
NoUniqueBeanDefinitionException 발생!

해결 방법 선택
   ├── 하나만 주입하고 싶다 → @Primary 또는 @Qualifier
   └── 전부 주입하고 싶다  → List<Pokemon>
```

---

## ⚠️ `@Primary` vs `@Qualifier` 우선순위

```java
// @Primary 선언 → Charmander

@Autowired
@Qualifier("pikachu")   // @Qualifier가 @Primary보다 우선
private Pokemon pokemon;

// 결과: Pikachu 주입
```

```
→ @Qualifier는 명시적 지정이므로 @Primary를 덮어씀.
```

---

## ⭐

| 상황 | 추천 |
| --- | --- |
| 구현체가 자주 교체됨 | `@Primary` (기본값 설정) |
| 특정 구현체를 고정 사용 | `@Qualifier` (이름 명시) |
| 모든 구현체를 순회 처리 | `List<T>` 주입 |

# 🟢 Section 01 — Bean Scope

> Bean의 인스턴스 생성 범위(Scope)를 제어하는 방식.
>

---

## 🔵 Singleton (기본값)

> 컨테이너에 Bean이 딱 하나만 생성되고 모든 요청에서 동일 객체를 공유.
>

---

## 특징

- Spring Bean의 기본 스코프
- `getBean()` 을 여러 번 호출해도 같은 인스턴스 반환
- `hashCode()` 비교 시 동일 값 출력
- ⚠️ 상태를 가지는 객체를 Singleton으로 만들면

  의도하지 않은 데이터 공유 문제 발생 가능


---

## ContextConfiguration.java (Singleton)

```java
@Configuration
public class ContextConfiguration {

    @Bean
    public Product carpBread() {
        return new Bread("붕어빵", 1000, new Date());
    }

    @Bean
    public Product milk() {
        return new Beverage("딸기우유", 1500, 500);
    }

    @Bean
    public Product water() {
        return new Beverage("지리반암반수", 3000, 500);
    }

    @Bean
    // @Scope 미선언 → 기본값 singleton
    public ShoppingCart cart() {
        return new ShoppingCart();
    }
}
```

---

## Application.java (Singleton)

```java
ApplicationContext context =
    new AnnotationConfigApplicationContext(
        ContextConfiguration.class
    );

ShoppingCart cart1 = context.getBean("cart", ShoppingCart.class);
cart1.addItem(carpBread);
cart1.addItem(milk);

System.out.println("cart1 : " + cart1.getItems());
// cart1 : [붕어빵, 딸기우유]

ShoppingCart cart2 = context.getBean("cart", ShoppingCart.class);
cart2.addItem(water);

System.out.println("cart2 : " + cart2.getItems());
// cart2 : [붕어빵, 딸기우유, 지리반암반수]  ← cart1 내용까지 공유됨!

System.out.println(cart1.hashCode() == cart2.hashCode());
// true → 동일 객체
```

---

# 🔴 Prototype

> `getBean()` 호출 시마다 새로운 인스턴스를 생성하여 반환.
>

---

## 특징

- `@Scope("prototype")` 선언 필요
- 요청마다 독립적인 객체 생성

  → 장바구니처럼 상태를 가지는 객체에 적합

- `hashCode()` 비교 시 서로 다른 값 출력

---

## ContextConfiguration.java (Prototype)

```java
public class ContextConfiguration {

    @Bean
    public Product carpBread() { ... }

    @Bean
    public Product milk() { ... }

    @Bean
    public Product water() { ... }

    @Bean
    @Scope("prototype")   // getBean() 호출마다 새 객체 생성
    public ShoppingCart cart() {
        return new ShoppingCart();
    }
}
```

---

## Application.java (Prototype)

```java
ShoppingCart cart1 = context.getBean("cart", ShoppingCart.class);
cart1.addItem(carpBread);
cart1.addItem(milk);

System.out.println("cart1 : " + cart1.getItems());
// cart1 : [붕어빵, 딸기우유]

ShoppingCart cart2 = context.getBean("cart", ShoppingCart.class);
cart2.addItem(water);

System.out.println("cart2 : " + cart2.getItems());
// cart2 : [지리반암반수]  ← 완전히 독립된 객체!

System.out.println(cart1.hashCode() == cart2.hashCode());
// false → 서로 다른 객체
```

# 🟡 Section 02 — Bean Lifecycle (Init & Destroy)

> Bean 생성 직후 초기화, 컨테이너 종료 직전 소멸 시 실행할 메서드를 지정.
>

---

## 방법 1 — 어노테이션 방식 (`@PostConstruct` / `@PreDestroy`)

> Bean 클래스 내부 메서드에 직접 선언. 권장 방식.
>

---

## Owner.java

```java
public class Owner {

    @PostConstruct
    public void openShop() {
        // Bean 생성 후 의존성 주입까지 완료된 시점에 실행
        System.out.println("가게 문을 열었습니다.");
    }

    @PreDestroy
    public void closeShop() {
        // 컨테이너 종료 직전 실행
        System.out.println("가게 문을 닫았습니다.");
    }
}
```

---

## 방법 2 — `@Bean` 속성 방식 (`initMethod` / `destroyMethod`)

> `@Bean` 어노테이션의 속성으로 메서드명을 문자열로 지정.
>

```java
@Bean(initMethod = "openShop", destroyMethod = "closeShop")
public Owner owner() {
    return new Owner();
}
```

---

## ContextConfiguration.java

```java
@Configuration
public class ContextConfiguration {

    @Bean
    public Product carpBread() { ... }

    @Bean
    public Product milk() { ... }

    @Bean
    public Product water() { ... }

    @Bean
    public ShoppingCart cart() { ... }

    @Bean
    // initMethod · destroyMethod 는 Owner 클래스의
    // @PostConstruct · @PreDestroy 로 처리
    public Owner owner() {
        return new Owner();
    }
}
```

---

## Application.java

```java
// 컨테이너 생성 → initMethod(@PostConstruct) 자동 호출
ApplicationContext context =
    new AnnotationConfigApplicationContext(
        ContextConfiguration.class
    );

// 컨테이너 종료 → destroyMethod(@PreDestroy) 자동 호출
((AnnotationConfigApplicationContext) context).close();
```

---

## 실행 흐름

```
컨테이너 생성
    └── @PostConstruct → "가게 문을 열었습니다."

context.close()
    └── @PreDestroy   → "가게 문을 닫았습니다."
```

## 🔴 Section 03 — Properties 외부 설정 주입

> `.properties` 파일의 값을 `@Value`로 읽어 Bean 생성 시 주입.
>

---

## product-info.properties

```
bread.carpbread.name=붕어빵
bread.carpbread.price=1000

beverage.milk.name=딸기우유
beverage.milk.price=1500
beverage.milk.capacity=500

beverage.water.name=지리반암반수
beverage.water.price=3000
beverage.water.capacity=500
```

---

## ContextConfiguration.java

```java
@PropertySource("section03/properties/product-info.properties")
@Configuration
public class ContextConfiguration {

    // 방법 1 — 필드 주입
    @Value("${bread.carpbread.name}")
    private String carpBreadName;

    @Value("${bread.carpbread.price}")
    private int carpBreadPrice;

    @Value("${Beverage.milk.name}")
    private String milkName;

    @Value("${beverage.milk.price}")
    private int milkPrice;

    @Value("${beverage.milk.capacity}")
    private int milkCapacity;

    @Bean
    public Product carpBread() {
        return new Bread(carpBreadName, carpBreadPrice, new Date());
    }

    @Bean
    public Product milk() {
        return new Beverage(milkName, milkPrice, milkCapacity);
    }

    // 방법 2 — 메서드 파라미터 주입
    @Bean
    public Product water(
        @Value("${beverage.water.name}")     String waterName,
        @Value("${beverage.water.price}")    int waterPrice,
        @Value("${beverage.water.capacity}") int waterCapacity
    ) {
        return new Beverage(waterName, waterPrice, waterCapacity);
    }

    @Bean
    public ShoppingCart cart() {
        return new ShoppingCart();
    }

    @Bean
    public Owner owner() {
        return new Owner();
    }
}
```

---

## Application.java

```java
ApplicationContext context =
    new AnnotationConfigApplicationContext(
        ContextConfiguration.class
    );

Bread carpBread  = context.getBean("carpBread", Bread.class);
Beverage milk    = context.getBean("milk",      Beverage.class);
Beverage water   = context.getBean("water",     Beverage.class);

ShoppingCart cart1 = context.getBean("cart", ShoppingCart.class);
cart1.addItem(carpBread);
cart1.addItem(milk);

System.out.println("cart1 : " + cart1.getItems());
// cart1 : [붕어빵 1000원, 딸기우유 1500원]

ShoppingCart cart2 = context.getBean("cart", ShoppingCart.class);
cart2.addItem(water);

System.out.println("cart2 : " + cart2.getItems());
// Singleton이므로 cart1 데이터까지 포함됨
```

---

### ✅ 핵심 요약

| 항목 | 내용 |
| --- | --- |
| Singleton (기본) | 컨테이너당 인스턴스 1개, 모든 요청이 동일 객체 공유 |
| Prototype | `getBean()` 호출마다 새 인스턴스 생성 |
| 상태 객체 주의 | 상태를 가진 Bean은 Singleton이면 공유 문제 → Prototype 고려 |
| `@PostConstruct` | Bean 생성·주입 완료 후 자동 실행되는 초기화 메서드 |
| `@PreDestroy` | 컨테이너 종료 직전 자동 실행되는 소멸 메서드 |
| `@PropertySource` | `.properties` 파일을 컨텍스트에 로드 |
| `@Value` | `${key}` 형식으로 프로퍼티 값을 필드 또는 파라미터에 주입 |

---

# ⚠️ Scope 선택 기준

## Singleton을 쓰면 안 되는 경우

```java
// ❌ 상태를 가지는 객체를 Singleton으로 → 공유 문제
ShoppingCart cart1 = context.getBean("cart", ShoppingCart.class);
cart1.addItem(itemA);

ShoppingCart cart2 = context.getBean("cart", ShoppingCart.class);

// cart2 == cart1 (동일 객체)
// cart2 에도 itemA 가 담겨 있음!
```

---

## Prototype으로 해결

```java
// ✅ @Scope("prototype") → 독립 객체 보장
@Bean
@Scope("prototype")
public ShoppingCart cart() {
    return new ShoppingCart();
}
```

---

## ⚠️ Lifecycle 메서드 비교

| 방식 | 선언 위치 | 특징 |
| --- | --- | --- |
| `@PostConstruct` | Bean 클래스 메서드 | 표준 Jakarta 어노테이션, 권장 |
| `@PreDestroy` | Bean 클래스 메서드 | 표준 Jakarta 어노테이션, 권장 |
| `initMethod` | `@Bean` 속성 | 클래스 수정 없이 외부에서 지정 가능 |
| `destroyMethod` | `@Bean` 속성 | 클래스 수정 없이 외부에서 지정 가능 |

---

## ⚠️ `@Value` 주입 두 가지 방식

## 방법 1 — 필드 주입

```java
@Value("${bread.carpbread.name}")
private String carpBreadName;

@Bean
public Product carpBread() {
    return new Bread(carpBreadName, ...);
}
```

---

## 방법 2 — 메서드 파라미터 주입

```java
@Bean
public Product water(
    @Value("${beverage.water.name}") String name,
    @Value("${beverage.water.price}") int price
) {
    return new Beverage(name, price, ...);
}
```

# chap05 - Spring AOP (Aspect-Oriented Programming)

---

## 📐 AOP 개념

> 애플리케이션 전반에 반복 등장하는 공통 관심사를 핵심 비즈니스 로직에서 분리하여, 필요한 실행 지점에 자동으로 끼워 넣는 방식.

---

## 관심사의 분리

| 구분                             | 설명               | 예시                                  |
| ------------------------------ | ---------------- | ----------------------------------- |
| 핵심 관심사 (Core Concern)          | 실제 업무 기능         | `selectMembers()`, `selectMember()` |
| 공통 관심사 (Cross-Cutting Concern) | 여러 곳에 반복되는 부가 기능 | 로깅, 트랜잭션, 보안                        |

---

## AOP 주요 용어

| 용어        | 설명                                                        |
| --------- | --------------------------------------------------------- |
| Aspect    | 공통 관심사를 모아 놓은 클래스 (`@Aspect`)                             |
| Pointcut  | Advice를 적용할 실행 지점을 선택하는 조건식                               |
| JoinPoint | Advice가 실행되는 실제 시점 (메서드 호출 등)                             |
| Advice    | JoinPoint에서 실행되는 공통 동작 (`@Before`, `@After`, `@Around` 등) |
| Proxy     | 실제 객체 앞에 Spring이 세우는 대리 객체                                |

## MemberDAO.java

```java id="v7n1xr"
@Repository  // DB 커넥션을 가지는 계층
public class MemberDAO {

    private final Map<Long, MemberDTO> memberMap;

    public MemberDAO() {
        memberMap = new HashMap<>();
        memberMap.put(1L, new MemberDTO(1L, "신짱구"));
        memberMap.put(1L, new MemberDTO(2L, "신짱아"));
    }

    public Map<Long, MemberDTO> selectMembers() {
        return memberMap;
    }

    public MemberDTO selectMember(Long id) {
        MemberDTO result = memberMap.get(id);

        if (result == null) {
            throw new RuntimeException(
                "해당하는 id의 회원이 없습니다."
            );
        }

        return result;
    }
}
```

## MemberService.java

```java id="y5r9mq"
@Service  // DAO를 호출하는 비즈니스 로직 계층
public class MemberService {

    private final MemberDAO memberDAO;

    // 생성자 하나 → @Autowired 생략 가능
    public MemberService(MemberDAO memberDAO) {
        this.memberDAO = memberDAO;
    }

    public Map<Long, MemberDTO> selectMembers() {

        // 공통 관심사(로그 출력)는 Aspect로 이전
        return memberDAO.selectMembers();   // 핵심 관심사만 남음
    }

    public MemberDTO selectMember(Long id) {
        return memberDAO.selectMember(id);
    }
}
```

## LoggingAspect.java

```java id="m3v8xw"
@Aspect     // AOP 클래스 선언
@Component  // 컨테이너에 Bean으로 등록
public class LoggingAspect {

    // 어떤 메서드에 공통 기능을 적용할지 결정하는 조건식
    // → section01 패키지 내 이름이 "Service"로 끝나는
    //   클래스의 모든 메서드, 모든 파라미터
    @Pointcut("execution(* com.ohgiraffers.section01.*Service.*(..))")
    public void logPointCut() {}

    // logPointCut() 조건에 해당하는 메서드 실행 "전"에 동작
    @Before("LoggingAspect.logPointCut()")
    public void logBefore(JoinPoint joinPoint) {

        // 실행 대상 객체 정보
        System.out.println(
            "Before joinPoint.getTarget() : "
            + joinPoint.getTarget()
        );

        // 실행 대상 메서드 시그니처 정보
        System.out.println(
            "Before joinPoint.getSignature() : "
            + joinPoint.getSignature()
        );
    }
}
```

## ✅ 핵심 요약

| 항목                        | 내용                                    |
| ------------------------- | ------------------------------------- |
| AOP                       | 공통 관심사를 핵심 로직에서 분리해 자동 삽입             |
| `@Aspect`                 | 공통 기능을 담는 클래스임을 선언                    |
| `@Pointcut`               | Advice를 적용할 메서드를 선택하는 표현식             |
| `@Before`                 | 대상 메서드 실행 전에 Advice 동작                |
| JoinPoint                 | 현재 실행 지점의 대상 객체·메서드 정보를 담은 객체         |
| Proxy                     | Spring이 `@Aspect` 대상 앞에 자동 생성하는 대리 객체 |
| `@EnableAspectJAutoProxy` | AOP Proxy 자동 생성 활성화 (설정 클래스에 선언)      |

## ⚠️ Advice 종류 비교

| 어노테이션             | 실행 시점                |
| ----------------- | -------------------- |
| `@Before`         | 메서드 실행 전             |
| `@After`          | 메서드 실행 후 (예외 여부 무관)  |
| `@AfterReturning` | 메서드가 정상 반환 후         |
| `@AfterThrowing`  | 메서드에서 예외 발생 후        |
| `@Around`         | 실행 전·후 모두 제어 (가장 강력) |

## ⚠️ `@EnableAspectJAutoProxy` 옵션

```java id="i1r7mz"
// 인터페이스가 없는 클래스(CGLIB 프록시)도 AOP 적용 가능하게 설정
@EnableAspectJAutoProxy(proxyTargetClass = true)
```

```text id="h5d9vu"
proxyTargetClass = false (기본값)
→ JDK 동적 프록시, 인터페이스 필수

proxyTargetClass = true
→ CGLIB 프록시, 인터페이스 없어도 적용 가능
```
