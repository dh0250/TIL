# chap10-api

## 01. Java API (자바 기본 API 활용)

Java에서 제공하는 핵심 클래스들의 사용법과 특징을 이해하여 효율적인 프로그래밍 수행

---

# ✔ 01. Object (객체의 조상)

모든 자바 클래스의 최상위 부모 클래스로, 모든 객체가 공통으로 가져야 할 기본 동작을 정의함

- 동일성 vs 동등성

  `==` 는 주소값을 비교하고, `equals()` 는 논리적 데이터 값을 비교함

- 해시 규약

  `equals()` 가 `true` 라면, 두 객체의 `hashCode()` 값은 반드시 같아야 함


```java
// Book.java에서 equals와 hashCode를 오버라이딩한 예시
@Override
public boolean equals(Object o) {
    if (this == o) return true;
    if (o == null || getClass() != o.getClass()) return false;

    Book book = (Book) o;

    return number == book.number
            && price == book.price
            && Objects.equals(title, book.title);
}

@Override
public int hashCode() {
    return Objects.hash(number, title, price);
}
```

# ✔ 02. String (불변 문자열)

한 번 생성된 문자열은 수정할 수 없으며, 수정 시 새로운 객체가 생성됨

- 주요 활용

  데이터 정제(공백 제거, 자르기), 문자열 비교


```java
String raw = "  [PROD-001] iPhone 15  ";

String trimmed = raw.trim();              // "[PROD-001] iPhone 15"
String code = trimmed.substring(1, 9);    // "PROD-001"

boolean isSame =
        "java".equals("JAVA".toLowerCase()); // true
```

---

# ✔ 03. StringBuilder (가변 문자열)

문자열을 빈번하게 합치거나 수정할 때 메모리 효율을 극대화하기 위해 사용

- 주요 특징

  가변(Mutable) 버퍼를 사용하여 기존 객체 내에서 데이터를 수정함


```java
StringBuilder sb = new StringBuilder("java");

sb.append("mysql");      // "javamysql"
sb.insert(4, " & ");     // "java & mysql"
sb.reverse();            // "lqsym & avaj"

String result = sb.toString();
```

---

# ✔ 04. Wrapper (기본 타입의 객체화)

기본 자료형을 객체로 감싸서 컬렉션에 저장하거나 유용한 유틸리티 메소드(Parsing 등)를 제공받음

- Boxing

  기본 타입 → 래퍼 객체

- Unboxing

  래퍼 객체 → 기본 타입


```java
// Parsing: 문자열을 숫자로 변환
int age = Integer.parseInt("20");
double price = Double.parseDouble("15500.5");

// Autoboxing & Unboxing
Integer num = 10; // 오토박싱
int value = num;  // 오토언박싱

// 주의: 객체 비교는 equals() 사용
Integer n1 = 128;
Integer n2 = 128;

System.out.println(n1.equals(n2)); // true
```

# ✔ 05. Time API (날짜와 시간)

Java 8 이후 표준이 된 날짜/시간 처리 API로, 불변 객체이며 가독성이 높음

- `LocalDate`
- `LocalTime`
- `LocalDateTime`

활용 가능

```java
// 현재 및 특정 날짜 생성
LocalDateTime now = LocalDateTime.now();

LocalDateTime departure =
        LocalDateTime.of(2026, 12, 25, 13, 30);

// 날짜 연산 및 비교
LocalDateTime deadline = departure.minusDays(7);

boolean isLate = now.isAfter(departure);

// 포맷 변환
DateTimeFormatter fmt =
        DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm");

String strDate = departure.format(fmt);
// "2026-12-25 13:30"
```

# 👀 핵심 요약

- `Object`
    - `toString()`, `equals()`, `hashCode()` 는 DTO 작성 시 필수
- `String vs StringBuilder`
    - 단순 조회용 → `String`
    - 반복문 문자열 결합 → `StringBuilder`
- `Wrapper`
    - 문자열 숫자 변환 → `parseXXX()`
    - 객체 값 비교 → `equals()`
- `Time API`
    - 날짜 연산 → `plus()`, `minus()`
    - 날짜 포맷 → `DateTimeFormatter`

# chap11-generic

## 01. Generic (제네릭)

클래스나 메소드에서 사용할 내부 데이터 타입을 컴파일 시점에 지정하는 기술로,

타입 안정성을 높이고 형변환의 번거로움을 제거함

---

# ✔ 주요 개념 및 활용

## 타입 안정성 (Type Safety)

의도하지 않은 타입의 객체가 저장되는 것을 방지하고,

컴파일 타임에 에러를 발견할 수 있음

## 캐스팅 생략

`Object` 타입을 사용할 때처럼 매번 강제 형변환(Downcasting)을 할 필요가 없어 코드가 간결해짐

## 재사용성

하나의 코드로 다양한 데이터 타입을 처리할 수 있는 범용 클래스 설계가 가능함

```java
// 제네릭 클래스 선언
public class GenericBox<T> {

    private T content; // T는 타입 파라미터

    public void setContent(T content) {
        this.content = content;
    }

    public T getContent() {
        return content;
    }
}
```

```java
// 사용 예시
GenericBox<Integer> box = new GenericBox<>();

box.setContent(10);

int value = box.getContent();
// 형변환 없이 바로 사용 가능
```

# 02. Generic Restrictions & Wildcards

(제네릭 제한 & 와일드카드)

제네릭 타입 파라미터에 제한을 두거나,

메소드 매개변수에서 유연한 타입을 허용하기 위한 기술

---

# ✔ 주요 기술 요소

## 제한된 타입 파라미터 (extends)

`extends` 키워드를 사용하여 특정 클래스나

그 자식 클래스만 타입으로 지정할 수 있도록 제한함

```java
// Rabbit 혹은 Rabbit의 후손 클래스만 타입 지정 가능
public class RabbitFarm<T extends Rabbit> {

    private T animal;

    public void setAnimal(T animal) {
        this.animal = animal;
    }
}
```

---

## 와일드카드 (Wildcards)

제네릭 타입을 매개변수로 사용할 때, 타입의 범위를 동적으로 지정함

- `<?>`
    - 모든 타입 허용 (Unbounded Wildcard)
- `<? extends T>`
    - T 또는 T의 자식 타입만 허용 (Upper Bounded)
- `<? super T>`
    - T 또는 T의 부모 타입만 허용 (Lower Bounded)

```java
public class FarmManager {

    // 어떤 토끼 농장이든 관리 가능
    public void manageAnyFarm(RabbitFarm<?> farm) {
        farm.getAnimal().cry();
    }

    // Bunny 혹은 그 후손만 관리 가능
    public void manageBunnyFarm(
            RabbitFarm<? extends Bunny> farm) {

        farm.getAnimal().cry();
    }

    // Bunny 혹은 그 조상만 관리 가능
    public void manageRabbitOrBunnyFarm(
            RabbitFarm<? super Bunny> farm) {

        farm.getAnimal().cry();
    }
}
```

---

# 👀 핵심 요약

- 제네릭의 필요성
    - `Object` 사용 시 발생하는 타입 불안정성과 형변환 비용 해결
- 제한된 제네릭
    - `<T extends 상위타입>` 으로 잘못된 타입 사용 방지
- 와일드카드의 유연성
    - `?` 를 활용해 다양한 제네릭 타입을 유연하게 처리 가능
- 컴파일 시점 검사
    - `ClassCastException` 같은 런타임 오류를 사전에 방지 가능
