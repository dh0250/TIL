## Spring Boot Admin 도메인 초기 설계

### 전체 구조

admin 도메인은 관리자 전용으로, 모든 엔드포인트에 `@PreAuthorize("hasRole('ADMIN')")` 적용

```
admin/
├── controller/   # 6개 컨트롤러
├── service/      # 6개 서비스
├── repository/   # 5개 레포지토리 (interface)
├── projection/   # 4개 프로젝션 (interface)
└── dto/
    ├── request/  # 2개
    └── response/ # 11개 record
```

### 엔드포인트 구성

| 컨트롤러 | 경로 | 역할 |
| --- | --- | --- |
| Dashboard | `/api/v1/admin/dashboard` | 대시보드 통계 |
| Member | `/api/v1/admin/members` | 회원 목록/상태 관리 |
| Inquiries | `/api/v1/admin/inquiries` | 1:1 문의 답변 처리 |
| Product | `/api/v1/admin/products` | 재료 사전 관리 |
| Recipe | `/api/v1/admin/recipes` | 레시피 관리 |
| Statistics | `/api/v1/admin/statistics` | 통계 조회 |

### Response DTO (record)

- **대시보드**: 관리자 수, 전체/비활성 회원 수, 오늘 신규 회원, 레시피 수, 미답변 문의, 평균 점수, 오늘 만료 재료 수
- **회원**: 기본 정보 + 활성/비활성 상태
- **문의**: 목록 + 상세(답변 내용 포함)
- **통계**: 카테고리별 만료 집계, 주간 트렌드, 낭비 재료 Top N (순위 변동 포함)

### 알게 된 것

- `@NotBlank`는 `String`/`CharSequence` 전용 → `Boolean` 필드엔 `@NotNull` 사용
- Spring Data Projection은 반드시 `interface`로 선언해야 동작
- Repository도 `class`가 아닌 `interface`로 선언해야 JpaRepository 상속 가능
- Git은 빈 폴더를 추적하지 않아서 `.gitkeep` 빈 파일로 폴더 구조를 커밋하는 관례가 있음
- GitHub PR 템플릿은 `.github/PULL_REQUEST_TEMPLATE.md` 위치에 있어야 자동 적용됨
