- 별도 Repository 없이 기존 `AdminMemberRepository`, `AdminInquiryRepository` 재사용

## 관리자 문의 관리 API 구현 (PR #118)

### 완성한 기능

- `GET /api/v1/admin/inquiries` 문의 목록 조회 API (#63)
- `GET /api/v1/admin/inquiries/{inquiryId}` 문의 상세 조회 API (#64)
- `POST /api/v1/admin/inquiries/{inquiryId}/answers` 답변 등록 API (#65)
- `PATCH /api/v1/admin/inquiries/{inquiryId}/answers/{answerId}` 답변 수정 API (#66)
- `PATCH /api/v1/admin/inquiries/{inquiryId}/status` 문의 종료 처리 API (#67)
- `DELETE /api/v1/admin/inquiries/{inquiryId}/answers/{answerId}` 답변 삭제 API (#154)

### 주요 구현 내용

- `Inquiry`, `InquiryAnswer` Entity 직접 생성 (Dev4 이탈로 담당)
- `resolveAdminIdOrThrow(email)` - email → memberId 변환 + null 예외 처리
- 답변 등록 시 `inquiry.markAsAnswered()` 자동 호출
- 답변 삭제 시 `inquiry.markAsUnanswered()` 자동 호출 (미답변 상태로 복구)
- `InquiryNotFoundException`, `InquiryAnswerNotFoundException` 추가

### 배운 것

**static vs 인스턴스 메서드**

- `create()` → static (객체 없는 상태에서 새로 생성)
- `update()`, `delete()`, `markAsAnswered()` → 인스턴스 (이미 존재하는 객체 수정)
- 인스턴스 필드(Repository) 사용하면 static 불가

**검증 순서**

```
1. 존재 여부 확인 (orElseThrow)
2. 소유권 검증 (validateUser)
3. 상태 검증 (validateNotAnswered)
```

**3-way Merge 충돌 해결**

- 왼쪽(Left) = 내 코드
- 가운데(Result) = 최종 결과물
- 오른쪽(Right) = dev 최신 코드
- 초록색 = 자동 합침, 분홍색 = 직접 해결 필요
