## 관리자 회원 관리 API 구현 (PR #117)

### 완성한 기능

- `GET /api/v1/admin/members` 회원 목록 조회 API (#58)
- `GET /api/v1/admin/members/search` 회원 검색 API (#59)
- `PATCH /api/v1/admin/members/{memberId}/status` 회원 탈퇴 처리 API (#60)
- `PATCH /api/v1/admin/members/{memberId}/status` 회원 탈퇴 복구 API (#61)
- `PATCH /api/v1/admin/members/{memberId}/role` 회원 권한 변경 API

### 주요 구현 내용

- `AdminMemberRepository` - JPQL `@Query`로 role/status/search 복합 조건 조회
- `AdminMemberService` - `getMembers()`, `updateMemberStatus()`, `updateMemberRole()`
- `AdminMemberResponse` record - `from(Member)` 패턴으로 Entity → DTO 변환
- `MemberNotFoundException` 추가 및 `GlobalExceptionHandler` 등록
- `@PreAuthorize` → SecurityConfig URL 레벨로 통합 관리로 변경

### 배운 것

**더티 체킹 (Dirty Checking)**

- `@Transactional` 안에서 `findById()`로 조회한 Entity는 JPA가 관리
- 필드 변경 시 `save()` 없이도 트랜잭션 종료 시 자동 UPDATE
- INSERT는 `save()` 필요, UPDATE/소프트삭제는 불필요

**레이어별 반환 타입**

```
Repository → Entity
Service    → DTO (Response)
Controller → ResponseEntity<ApiResponse<DTO>>
```

**@RequestBody vs @RequestParam**

- URL에 있는 값 → `@PathVariable`
- Body에 있는 값 → Request DTO로 `@RequestBody`
- GET 쿼리 파라미터 → `@RequestParam`
