# chap04 — MyBatis 동적 SQL (Dynamic Query)

---

## 개념

조건에 따라 SQL의 구조 자체가 달라지는 동적 SQL을 XML 태그로 작성하는 방법

- 정적 SQL은 WHERE 절이 항상 동일 → 조건 없어도 불필요한 구문이 붙음
- MyBatis는 아래 태그로 SQL을 조건부 조립

```
<if>
<choose> / <when> / <otherwise>
<foreach>
<where>
<trim>
<set>
```

- Java 코드에 SQL 분기 로직 없이 XML만 수정
- 관심사 분리 유지

## mybatis-config.xml

```xml
<configuration>

    <properties resource="config/connection-info.properties"/>

    <settings>
        <setting name="logImpl" value="STDOUT_LOGGING"/>
    </settings>

    <typeAliases>
        <typeAlias type="com.ohgiraffers.section01.SearchCriteria"
                   alias="SearchCriteria"/>
        <typeAlias type="com.ohgiraffers.section01.MenuDTO"
                   alias="MenuDTO"/>
    </typeAliases>

    <environments default="dev">
        <environment id="dev">
            <transactionManager type="JDBC"/>
            <dataSource type="POOLED">
                <property name="driver" value="${driver}"/>
                <property name="url" value="${url}"/>
                <property name="username" value="${username}"/>
                <property name="password" value="${password}"/>
            </dataSource>
        </environment>
    </environments>

    <mappers>
        <package name="com.ohgiraffers.section01"/>
    </mappers>

</configuration>
```

### chap03 대비 추가된 설정 3가지

| 태그 | 내용 | 효과 |
|---|---|---|
| `<properties>` | DB 접속 정보를 properties 파일로 분리 | XML에 비밀번호 직접 노출 방지 |
| `<typeAliases>` | 긴 FQCN 대신 별칭 사용 | XML 가독성 향상 |
| `<package>` | Mapper 패키지 자동 스캔 | 파일 추가 시 개별 등록 불필요 |

### 핵심

| 기호 | 역할 |
|---|---|
| `${ }` | properties 값 치환 (설정 레이어) |
| `#{ }` | MyBatis 파라미터 바인딩 (SQL 레이어) |

같은 기호처럼 보이지만 **전혀 다른 레이어에서 처리**된다.

## `<package>` vs `<mapper resource>`

| 방식 | 선언 | 특징 |
|---|---|---|
| `mapper resource` | 파일 개별 등록 | 누락 시 런타임 오류 |
| `package scan` | 패키지 단위 자동 스캔 | 자동 등록 |

### 패키지 스캔 조건

```
java/com/ohgiraffers/section01/
 └─ DynamicSqlMapper.java

resources/com/ohgiraffers/section01/
 └─ DynamicSqlMapper.xml
```

java와 resources 경로 아래 **동일한 패키지 경로** 필요

## SearchCriteria.java — 검색 조건 전달 객체

```java
public class SearchCriteria {

    private String condition;  // 검색 조건 (name / category / menuCode)
    private String value;      // 검색 값

    public SearchCriteria() {}

    public SearchCriteria(String condition, String value) {
        this.condition = condition;
        this.value = value;
    }
}
```

### 핵심

```
SearchCriteria
 ├─ condition  →  "name" / "category" / "menuCode"
 └─ value      →  실제 검색어
```

- `Map<String, String>` 대신 타입 있는 객체 사용
- XML에서 필드명(`condition`, `value`)으로 직접 접근 가능
- `typeAlias` 등록 후 `parameterType="SearchCriteria"` 로 사용

## DynamicSqlMapper.java — Mapper 인터페이스

```java
public interface DynamicSqlMapper {

    // <if> : 가격대별 조회
    List<MenuDTO> selectMenuByPrice(Map<String, Integer> map);

    // <if> : 이름 또는 카테고리명으로 검색
    List<MenuDTO> searchMenu(SearchCriteria searchCriteria);

    // <choose> : 상위 분류별 조회 (식사 / 음료 / 디저트)
    List<MenuDTO> searchMenuBySupCategory(SearchCriteria searchCriteria);

    // <foreach> : 랜덤 코드 목록으로 IN 절 조회
    List<MenuDTO> searchMenuByRandomMenuCode(Map<String, List<Integer>> criteria);

    // <where> : 조건 유무에 따라 전체 또는 단건 조회
    List<MenuDTO> searchMenuByCodeOrSearchAll(SearchCriteria searchCriteria);

    // <trim> : 이름 또는 카테고리 코드로 검색
    List<MenuDTO> searchMenuByNameOrCategory(Map<String, Object> criteria);

    // <set> : 입력된 항목만 UPDATE
    int modifyMenu(Map<String, Object> criteria);
}
```

## 동적 SQL 태그 — `<if>`

### 기본 문법

```xml
<if test="조건식">
    SQL 조각
</if>
```

조건이 참일 때만 해당 SQL 조각을 포함한다.

### OGNL 비교 연산자

| 기호 | OGNL |
|---|---|
| `>` | `gt` |
| `<` | `lt` |
| `>=` | `gte` |
| `<=` | `lte` |

XML에서 `<`, `>` 는 태그 문자로 인식되므로 OGNL 연산자 또는 CDATA 사용

---

### 예제 1 — 가격대별 조건 분기

```xml
<select id="selectMenuByPrice" parameterType="hashmap" resultMap="menuResultMap">
    SELECT A.MENU_CODE, A.MENU_NAME, A.MENU_PRICE,
           A.CATEGORY_CODE, A.ORDERABLE_STATUS
    FROM TBL_MENU A
    WHERE A.ORDERABLE_STATUS = 'Y'

    <if test="price gte 0 and price lte 10000">
        <![CDATA[
        AND A.MENU_PRICE < #{ price }
        ]]>
    </if>

    <if test="price gt 10000 and price lte 20000">
        AND A.MENU_PRICE BETWEEN 10000 AND #{ price }
    </if>

    <if test="price gt 20000 and price lte 30000">
        AND A.MENU_PRICE BETWEEN 20000 AND #{ price }
    </if>

    <if test="price gt 30000 and price lte 40000">
        AND A.MENU_PRICE BETWEEN 30000 AND #{ price }
    </if>

    ORDER BY A.MENU_CODE
</select>
```

### 핵심

`<if>`는 `if() {} if() {} if() {}` 와 동일 — **if-else가 아니다**

조건이 여러 개 참이면 **모두 실행**

---

### 예제 2 — JOIN + WHERE 동적 생성

```xml
<select id="searchMenu" parameterType="SearchCriteria" resultMap="menuResultMap">
    SELECT A.MENU_CODE, A.MENU_NAME, A.MENU_PRICE,
           A.CATEGORY_CODE, A.ORDERABLE_STATUS
    FROM TBL_MENU A

    <if test="condition == 'category'">
        JOIN tbl_category B ON (A.CATEGORY_CODE = B.CATEGORY_CODE)
    </if>

    WHERE A.ORDERABLE_STATUS = 'Y'

    <if test="condition == 'name'">
        AND A.MENU_NAME LIKE CONCAT('%', #{ value }, '%')
    </if>

    <if test="condition == 'category'">
        AND B.CATEGORY_NAME = #{ value }
    </if>

    ORDER BY A.MENU_CODE
</select>
```

### 핵심

`<if>`는 WHERE 절뿐 아니라 `FROM`, `JOIN`, `GROUP BY`, `ORDER BY` **어디든** 사용 가능

## 동적 SQL 태그 — `<choose>` / `<when>` / `<otherwise>`

switch-case 역할 — 처음 참인 `<when>` **하나만** 실행

```xml
<select id="searchMenuBySupCategory" parameterType="SearchCriteria" resultMap="menuResultMap">
    SELECT ...
    FROM TBL_MENU A
    WHERE A.ORDERABLE_STATUS = 'Y'

    <choose>
        <when test="value == '식사'">
            AND A.CATEGORY_CODE IN (4, 5, 6, 7)
        </when>
        <when test="value == '음료'">
            AND A.CATEGORY_CODE IN (8, 9, 10)
        </when>
        <otherwise>
            AND A.CATEGORY_CODE IN (11, 12)
        </otherwise>
    </choose>

    ORDER BY A.MENU_CODE
</select>
```

### `<if>` vs `<choose>` 비교

| 태그 | 대응 Java | 실행 방식 |
|---|---|---|
| `<if>` | `if / if / if` | 참인 조건 **모두** 실행 |
| `<choose>` | `switch-case` | 처음 참인 조건 **하나만** 실행 |

## 동적 SQL 태그 — `<foreach>`

컬렉션을 반복해서 IN 절 자동 생성

```xml
<select id="searchMenuByRandomMenuCode" parameterType="hashmap" resultMap="menuResultMap">
    SELECT ...
    FROM TBL_MENU A
    WHERE A.MENU_CODE IN

    <foreach collection="randomMenuCodeList"
             item="menuCode"
             open="("
             separator=","
             close=")">
        #{ menuCode }
    </foreach>
</select>
```

### 속성 정리

| 속성 | 의미 |
|---|---|
| `collection` | 반복할 컬렉션 (Map의 key 이름) |
| `item` | 반복 시 꺼내올 원소 이름 |
| `open` | 시작 문자 |
| `separator` | 원소 사이 구분자 |
| `close` | 종료 문자 |

### 결과

```sql
WHERE A.MENU_CODE IN (1, 3, 7, 12, 20)
```

### Service에서 List 전달 방식

```java
public void searchMenuByRandomMenuCode(List<Integer> randomMenuCodeList) {

    SqlSession sqlSession = getSqlSession();

    try {
        DynamicSqlMapper mapper = sqlSession.getMapper(DynamicSqlMapper.class);

        Map<String, List<Integer>> criteria = new HashMap<>();
        criteria.put("randomMenuCodeList", randomMenuCodeList);  // key = collection 속성값

        printMenuList(mapper.searchMenuByRandomMenuCode(criteria));
    } finally {
        sqlSession.close();
    }
}
```

### Application에서 랜덤 코드 생성

```java
public static List<Integer> createRandomMenuCode() {

    Set<Integer> set = new HashSet<>();
    while (set.size() < 5) {
        int temp = ((int)(Math.random() * 26)) + 1;
        set.add(temp);
    }

    List<Integer> list = new ArrayList<>(set);
    Collections.sort(list);

    return list;
}
```

## 동적 SQL 태그 — `<where>`

WHERE 자동 생성 + AND/OR 앞부분 자동 제거

```xml
<select id="searchMenuByCodeOrSearchAll" parameterType="SearchCriteria" resultMap="menuResultMap">
    SELECT ...
    FROM TBL_MENU A

    <where>
        A.ORDERABLE_STATUS = 'Y'
        <if test="condition != null and condition eq 'menuCode'">
            AND A.MENU_CODE = #{ value }
        </if>
    </where>
</select>
```

### 기능

| 상황 | 처리 |
|---|---|
| 내부에 내용이 없을 때 | WHERE 자체를 추가하지 않음 |
| 내용이 AND로 시작할 때 | AND 자동 제거 |
| 내용이 OR로 시작할 때 | OR 자동 제거 |

### 생성 결과

```sql
-- condition == null 인 경우
WHERE A.ORDERABLE_STATUS = 'Y'

-- condition == 'menuCode' 인 경우
WHERE A.ORDERABLE_STATUS = 'Y'
AND A.MENU_CODE = ?
```

## 동적 SQL 태그 — `<trim>`

prefix / suffix 문자열 제어 — `<where>`, `<set>`의 일반화 버전

```xml
<select id="searchMenuByNameOrCategory" parameterType="hashmap" resultMap="menuResultMap">
    SELECT ...
    FROM TBL_MENU A
    WHERE A.ORDERABLE_STATUS = 'Y'

    <trim prefix="AND (" suffix=")" prefixOverrides="AND |OR ">
        <if test="nameValue != null">
            A.MENU_NAME LIKE CONCAT('%', #{ nameValue }, '%')
        </if>
        <if test="categoryValue != null">
            AND A.CATEGORY_CODE = #{ categoryValue }
        </if>
    </trim>
</select>
```

### 속성 정리

| 속성 | 의미 |
|---|---|
| `prefix` | 내용이 있으면 앞에 추가 |
| `suffix` | 내용이 있으면 뒤에 추가 |
| `prefixOverrides` | 내용의 맨 앞 해당 문자 자동 제거 |
| `suffixOverrides` | 내용의 맨 뒤 해당 문자 자동 제거 |

## 동적 SQL 태그 — `<set>` (= `<trim>` 응용)

입력된 컬럼만 UPDATE + 마지막 쉼표 자동 제거

```xml
<update id="modifyMenu" parameterType="hashmap">
    UPDATE TBL_MENU

    <!-- 문제 상황: 첫 번째 if가 false면 SET 없이 쉼표가 남아 문법 오류 -->
    <!--
    <if test="name != null and name != ''">
        SET MENU_NAME = #{ name }
    </if>
    <if test="categoryCode != null and categoryCode gt 0">
        , CATEGORY_CODE = #{ categoryCode }
    </if>
    -->

    <!-- 해결: trim으로 SET 앞에 붙이고 맨 앞 쉼표 제거 -->
    <trim prefix="SET" prefixOverrides=",">
        <if test="name != null and name != ''">
            MENU_NAME = #{ name }
        </if>
        <if test="categoryCode != null and categoryCode gt 0">
            , CATEGORY_CODE = #{ categoryCode }
        </if>
        <if test="orderableStatus != null and orderableStatus != ''">
            , ORDERABLE_STATUS = #{ orderableStatus }
        </if>
    </trim>

    WHERE MENU_CODE = #{ code }
</update>
```

### `<set>` 태그 등가 표현

```
<set>   =  <trim prefix="SET"   suffixOverrides=",">
<where> =  <trim prefix="WHERE" prefixOverrides="AND |OR ">
```

## MenuService.java — Service 패턴 정리

### 조회 (트랜잭션 불필요)

```java
public void searchMenuBySupCategory(SearchCriteria searchCriteria) {

    SqlSession sqlSession = getSqlSession();

    try {
        DynamicSqlMapper mapper = sqlSession.getMapper(DynamicSqlMapper.class);
        printMenuList(mapper.searchMenuBySupCategory(searchCriteria));
    } finally {
        sqlSession.close();
    }
}
```

### 수정 (트랜잭션 필요)

```java
public void modifyMenu(Map<String, Object> criteria) {

    SqlSession sqlSession = getSqlSession();

    try {
        DynamicSqlMapper mapper = sqlSession.getMapper(DynamicSqlMapper.class);
        int result = mapper.modifyMenu(criteria);

        if (result > 0) {
            System.out.println("메뉴 정보 변경 성공");
            sqlSession.commit();
        } else {
            System.out.println("메뉴 정보 변경 실패");
            sqlSession.rollback();
        }

    } catch (RuntimeException e) {
        sqlSession.rollback();
        throw e;
    } finally {
        sqlSession.close();
    }
}
```

### 트랜잭션 흐름

```
SqlSession 획득 (autoCommit = false)
     │
     ├── result > 0 → commit()
     ├── result == 0 → rollback()
     └── 예외 발생 → rollback() → throw → finally close()
```

## chap03 vs chap04 설정 비교

| 항목 | chap03 | chap04 |
|---|---|---|
| DB 접속 정보 | XML 직접 작성 | properties 파일 분리 |
| 타입 별칭 | FQCN 사용 | `typeAlias` 등록 |
| Mapper 등록 | `mapper resource` 개별 등록 | `package scan` 자동 등록 |
| 쿼리 실행 | `session.selectList("id")` | `getMapper()` |
| SQL 로그 | 없음 | `STDOUT_LOGGING` |
| 검색 파라미터 | DTO 또는 Map | `SearchCriteria` 또는 Map |

## `#{}` vs `${}`

| 구분 | `#{}` | `${}` |
|---|---|---|
| 용도 | PreparedStatement 바인딩 | 문자열 치환 / properties 주입 |
| SQL Injection | 안전 | 위험 |
| 사용 대상 | 값(Value) | 컬럼명, 테이블명, properties |
| 실무 기본 | 항상 사용 | 최소 사용 |

## 핵심 요약

| 항목 | 내용 |
|---|---|
| `<if>` | 조건이 참일 때만 SQL 포함 |
| OGNL | `gt`, `lt`, `gte`, `lte` 사용 |
| `<choose>` | 처음 참인 `<when>` 하나만 실행 |
| `<foreach>` | IN 절 반복 생성, collection 속성 = Map key 이름 |
| `<where>` | WHERE 자동 추가 + AND/OR 앞부분 자동 제거 |
| `<set>` | SET 자동 추가 + 마지막 쉼표 자동 제거 |
| `<trim>` | `<where>`, `<set>`의 일반화 버전 |
| `typeAlias` | FQCN 단축 |
| `<properties>` | DB 접속 정보 외부화 |
| `<package>` | Mapper 자동 등록 |
| `getMapper()` | 타입 안전한 인터페이스 호출 |

---

> **MyBatis 동적 SQL은 `<if>`, `<choose>`, `<foreach>`, `<where>`, `<set>`, `<trim>` 태그로 조건에 따라 SQL 구조 자체를 동적으로 조립하며, `typeAlias`, `properties`, `package scan`, `getMapper()`를 활용해 유지보수성과 타입 안전성을 높이는 방식이다.**
