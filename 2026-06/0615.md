# chap04 — JPQL (Java Persistence Query Language)

## 개념

엔티티 객체를 대상으로 쿼리하는 객체지향 쿼리 언어

- SQL을 추상화해서 특정 데이터베이스 SQL에 의존하지 않음
- 테이블이 아닌 **엔티티 객체와 필드**를 대상으로 조회
- 최종적으로는 데이터베이스 방언(Dialect)에 맞는 SQL로 변환되어 실행됨

## 주요 특징 및 문법

| 구분 | 설명 | 예시 |
|---|---|---|
| 대상 | 테이블이 아닌 엔티티 객체 대상 | `SELECT m FROM Menu m` |
| 대소문자 | 엔티티와 필드는 대소문자 구분 | `Menu` (엔티티), `menuName` (필드) |
| 별칭(Alias) | 엔티티 별칭 필수 | `Menu AS m` (AS 생략 가능) |
| TypedQuery | 반환 타입이 명확할 때 사용 | `em.createQuery(jpql, Menu.class)` |
| Query | 반환 타입이 불명확할 때 사용 | `em.createQuery(jpql)` |

# section01 — 기본 조회 (Simple JPQL)

## 결과 조회 API

| 메서드 | 설명 | 주의 |
|---|---|---|
| `getResultList()` | 결과가 하나 이상일 때 리스트 반환 | 결과 없으면 빈 리스트 반환 |
| `getSingleResult()` | 결과가 정확히 하나일 때 단일 객체 반환 | 결과 없거나 둘 이상이면 예외 발생 |

## JPQL 연산자 예시

```java
// TypedQuery — 단일 값 조회
String jpql = "SELECT m.menuName FROM Section01Menu AS m WHERE m.menuCode = 8";
String result = entityManager.createQuery(jpql, String.class).getSingleResult();

// DISTINCT — 중복 제거
String jpql = "SELECT DISTINCT m.categoryCode FROM Section01Menu m";

// IN — 목록 내 값 조회
String jpql = "SELECT m FROM Section01Menu m WHERE m.categoryCode IN(11, 12)";

// LIKE — 패턴 매칭 (%: 0개 이상의 문자)
String jpql = "SELECT m FROM Section01Menu m WHERE m.menuName LIKE '%마늘%'";
```

# section02 — 파라미터 바인딩 (Parameter Binding)

## 이름 기준 바인딩 (Named Parameters) — 권장

콜론(`:`) 뒤에 파라미터 이름을 지정하여 바인딩. 위치에 의존하지 않아 유지보수가 용이함

```java
String jpql = "SELECT m FROM Section02Menu m WHERE m.menuName = :menuName";

List<Menu> result = entityManager.createQuery(jpql, Menu.class)
                                 .setParameter("menuName", menuName)
                                 .getResultList();
```

## 위치 기준 바인딩 (Positional Parameters) — 비권장

물음표(`?`) 뒤에 숫자를 지정하여 바인딩 (1부터 시작). 파라미터 순서 변경 시 오류 발생 위험

```java
String jpql = "SELECT m FROM Section02Menu m WHERE m.menuName = ?1";

List<Menu> result = entityManager.createQuery(jpql, Menu.class)
                                 .setParameter(1, menuName)
                                 .getResultList();
```

# section03 — 프로젝션 (Projection)

SELECT 절에 **조회할 대상을 지정**하는 방식

## 1. 엔티티 프로젝션

엔티티 자체를 조회 → **영속성 컨텍스트의 관리 대상**이 됨

```java
String jpql = "SELECT m FROM Section03Menu m";
List<Menu> result = entityManager.createQuery(jpql, Menu.class).getResultList();
```

## 2. 스칼라 타입 프로젝션

특정 필드(컬럼)만 선택 조회 → 영속성 컨텍스트 관리 대상이 **아님**

```java
// 반환 타입이 불명확 → Object[] 로 수동 처리 필요
String jpql = "SELECT c.categoryCode, c.categoryName FROM Section03Category c";
List<Object[]> result = entityManager.createQuery(jpql).getResultList();
```

## 3. new 명령어 프로젝션 — 권장

DTO 클래스 생성자를 직접 호출하여 타입 안정성을 확보

```java
// CategoryInfo(int categoryCode, String categoryName) 생성자 필요
String jpql = "SELECT new com.ohgiraffers.jpql.section03.projection.CategoryInfo"
            + "(c.categoryCode, c.categoryName) FROM Section03Category c";

List<CategoryInfo> result = entityManager.createQuery(jpql, CategoryInfo.class).getResultList();
```

| 프로젝션 종류 | 반환 타입 | 영속성 컨텍스트 관리 |
|---|---|---|
| 엔티티 프로젝션 | `List<Menu>` | O |
| 스칼라 타입 프로젝션 | `List<Object[]>` | X |
| new 명령어 프로젝션 | `List<DTO>` | X |

# section04 — 페이징 (Paging)

JPA가 제공하는 페이징 API로, DB 방언에 상관없이 동일한 코드로 페이징 처리

```java
String jpql = "SELECT m FROM Section04Menu m ORDER BY m.menuCode DESC";

List<Menu> result = entityManager.createQuery(jpql, Menu.class)
                                 .setFirstResult(offset)  // 조회 시작 위치 (0부터)
                                 .setMaxResults(limit)    // 조회할 데이터 개수
                                 .getResultList();
```

| 메서드 | 설명 | 예시 |
|---|---|---|
| `setFirstResult(int)` | 조회를 시작할 위치 (0부터 시작) | `0` → 첫 번째 행부터 |
| `setMaxResults(int)` | 조회할 데이터의 최대 개수 | `5` → 5건 조회 |

# section05 — 그룹 함수 (Group Function)

## 집계 함수

```java
// COUNT — 결과가 없어도 0 반환 (Long 타입)
String jpql = "SELECT COUNT(m.menuCode) FROM Section05Menu m WHERE m.categoryCode = :categoryCode";
long count = entityManager.createQuery(jpql, Long.class)
                          .setParameter("categoryCode", categoryCode)
                          .getSingleResult();

// SUM — 결과가 없으면 null 반환 (주의)
String jpql = "SELECT SUM(m.menuPrice) FROM Section05Menu m WHERE m.categoryCode = :categoryCode";
Long sum = entityManager.createQuery(jpql, Long.class)
                        .setParameter("categoryCode", categoryCode)
                        .getSingleResult();
```

## GROUP BY / HAVING

```java
// 카테고리별 메뉴 가격 합계 중 minPrice 이상인 것만 조회
String jpql = "SELECT m.categoryCode, SUM(m.menuPrice) "
            + "FROM Section05Menu m "
            + "GROUP BY m.categoryCode HAVING SUM(m.menuPrice) >= :minPrice";

List<Object[]> result = entityManager.createQuery(jpql)
                                     .setParameter("minPrice", minPrice)
                                     .getResultList();
```

| 집계 함수 | 설명 | 결과 없을 때 |
|---|---|---|
| `COUNT` | 개수 | `0` 반환 |
| `SUM` | 합계 | `null` 반환 |
| `AVG` | 평균 | `null` 반환 |
| `MAX` / `MIN` | 최대/최소값 | `null` 반환 |

# 핵심 요약

| 항목 | 내용 |
|---|---|
| 객체 중심 | JPQL은 테이블이 아닌 객체 대상 → `SELECT *` 대신 별칭 사용 (`SELECT m`) |
| 타입 안정성 | `TypedQuery` 사용으로 런타임 캐스팅 오류 방지 |
| 보안 및 성능 | 파라미터 바인딩으로 SQL 인젝션 방지 및 쿼리 실행 계획 재사용 |
| 프로젝션 | 여러 필드 조회 시 `new` 명령어로 DTO에 담아 타입 안정성 확보 |
| 페이징 | `setFirstResult` / `setMaxResults` API로 DB 방언에 독립적인 페이징 처리 |
| 그룹 함수 | `COUNT`만 결과 없을 때 `0` 반환, 나머지(`SUM`, `AVG` 등)는 `null` 반환 주의 |
