# 🥕 냉파 마스터 요구사항 요약

## 1. 프로젝트 개요

### 서비스 목적

사용자가 보유 식재료와 유통기한을 관리하고,

- 유통기한 임박 재료 확인
- 보유 재료 기반 레시피 추천
- 날씨 기반 레시피 추천
- 냉파 점수 및 통계 확인

을 통해 식재료 폐기를 줄이는 서비스

### 기술 스택

- Backend : Spring Boot, JPA, MyBatis
- DB : PostgreSQL
- Frontend : Next.js, React
- 추가 기술
    - PostgreSQL pg_trgm (검색)
    - Spring Scheduler / Batch
    - 기상청 API

# 3. 회원 관리

## 회원가입

입력값

- 이메일
- 비밀번호
- 비밀번호 확인

특징

- 이름 입력 없음
- 랜덤 닉네임 자동 생성
- 이메일 중복 불가
- 이메일 형식 검증
- 비밀번호 규칙
    - 8~15자
    - 영문 소문자 포함
    - 특수문자 포함

## 로그인

- 이메일 + 비밀번호 로그인

## 회원 상태

### ACTIVE

정상 회원

### WITHDRAWN

탈퇴 회원

특징

- 직접 탈퇴 불가
- 관리자만 탈퇴 처리 가능
- 탈퇴 회원 로그인 불가
- 데이터 유지
- 동일 이메일 재가입 불가

# 8. 레시피

## 조회

표시 정보

- 제목
- 설명
- 조리시간
- 난이도
- 재료
- 조리 과정

## 추천 로직

### 보유율 계산

보유재료 수 ÷ 필요재료 수 × 100

### 가능 레시피

- 보유율 80% 이상

### 추가 표시

- 부족 재료
- 재료 보유율

## 등록

- 재료
- 조리 과정

조건

- 제목 1~30자
- 재료 1개 이상
- 조리 과정 1단계 이상

## 소셜 기능

- 등록/해제
- 회원별 저장

### 좋아요

- 등록/취소

### 댓글

- 조회

## 권한

- 작성자만 수정/삭제 가능

# 11. 냉파 점수

## 점수 규칙

### 차감

만료 재료 발생

- 1일당 -2점

### 증가

레시피 등록

+3점

### 선택

4일 연속 만료 없음

+5점

## 정책

- 최소 0점
- 서버 계산
- 직접 수정 불가

## 등급

| 점수 | 등급 |
| --- | --- |
| 0~19 | 냉털 새내기 |
| 20~39 | 냉파 수련생 |
| 40~59 | 냉장고 탐험가 |
| 60~79 | 냉파 에이스 |
| 80~89 | 냉파 고인물 |
| 90~100 | 냉파 마스터 |

## 점수 이력

저장 항목

- 점수
- 변동량
- 사유
- 일시
