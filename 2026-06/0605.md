# chap04 — MyBatis 동적 SQL (Dynamic Query)

---

# 개념

### 조건에 따라 SQL의 구조 자체가 달라지는 동적 SQL(Dynamic SQL)을 XML 태그로 작성하는 방법

* 정적 SQL은 WHERE 절이 항상 동일

    * 검색 조건이 없어도 불필요한 조건이 붙음
    * 직접 분기 처리 필요

* MyBatis는 다음 태그로 SQL을 조건부 조립

```xml
<if>
<choose>
<foreach>
<where>
<trim>
<set>
```

* Java 코드에 SQL 분기 로직 없이 XML만 수정
* 관심사 분리 유지

# mybatis-config.xml

## 전체 설정

```xml
<configuration>

    <properties resource="config/connection-info.properties"/>

    <settings>
        <setting name="logImpl" value="STDOUT_LOGGING"/>
    </settings>

    <typeAliases>
        <typeAlias type="com.ohgiraffers.section01.SearchCriteria"
                   alias="SearchCriteria"/>

        <typeAlias type="com.ohgiraffers.section01.MenuDTO"
                   alias="MenuDTO"/>
    </typeAliases>

    <environments default="dev">

        <environment id="dev">

            <transactionManager type="JDBC"/>

            <dataSource type="POOLED">
                <property name="driver" value="${driver}"/>
                <property name="url" value="${url}"/>
                <property name="username" value="${username}"/>
                <property name="password" value="${password}"/>
            </dataSource>

        </environment>

    </environments>

    <mappers>
        <package name="com.ohgiraffers.section01"/>
    </mappers>

</configuration>
```

# 동적 SQL 태그 — `<if>`

## 기본 문법

```xml
<if test="조건식">
    SQL 조각
</if>
```

### 의미

조건이 참일 때만 SQL 포함

---

## OGNL 비교 연산자

| 기호 | OGNL |
| -- | ---- |
| >  | gt   |
| <  | lt   |
| >= | gte  |
| <= | lte  |

---

### 이유

XML에서

```xml
<
>
```

는 태그 문자로 인식됨

---

### 대안

```xml
<![CDATA[
A.MENU_PRICE < #{ price }
]]>
```

CDATA 사용 가능

---

# 예제 1 — 가격대별 조건 분기

```xml
<select id="selectMenuByPrice"
        parameterType="hashmap"
        resultMap="menuResultMap">

    SELECT ...
    FROM TBL_MENU A
    WHERE A.ORDERABLE_STATUS = 'Y'

    <if test="price gte 0 and price lte 10000">
        <![CDATA[
        AND A.MENU_PRICE < #{ price }
        ]]>
    </if>

    <if test="price gt 10000 and price lte 20000">
        AND A.MENU_PRICE BETWEEN 10000 AND #{ price }
    </if>

    <if test="price gt 20000 and price lte 30000">
        AND A.MENU_PRICE BETWEEN 20000 AND #{ price }
    </if>

    <if test="price gt 30000 and price lte 40000">
        AND A.MENU_PRICE BETWEEN 30000 AND #{ price }
    </if>

    ORDER BY A.MENU_CODE

</select>
```

### 핵심

`<if>`는

```java
if (...) {}
if (...) {}
if (...) {}
```

와 동일

---

if-else가 아니다.

조건이 여러 개 참이면 모두 실행

# 예제 2 — JOIN + WHERE 동적 생성

```xml
<select id="searchMenu"
        parameterType="SearchCriteria"
        resultMap="menuResultMap">

    SELECT ...
    FROM TBL_MENU A

    <if test="condition == 'category'">
        JOIN tbl_category B
          ON (A.CATEGORY_CODE = B.CATEGORY_CODE)
    </if>

    WHERE A.ORDERABLE_STATUS = 'Y'

    <if test="condition == 'name'">
        AND A.MENU_NAME
        LIKE CONCAT('%', #{ value }, '%')
    </if>

    <if test="condition == 'category'">
        AND B.CATEGORY_NAME = #{ value }
    </if>

    ORDER BY A.MENU_CODE

</select>
```

### 핵심

`<if>`는 WHERE 절뿐 아니라

```sql
FROM
JOIN
GROUP BY
ORDER BY
```

어디든 사용 가능

# `<choose>` / `<when>` / `<otherwise>`

## switch-case 역할

```xml
<choose>

    <when test="value == '식사'">
        AND A.CATEGORY_CODE IN (4,5,6)
    </when>

    <when test="value == '음료'">
        AND A.CATEGORY_CODE IN (7,8,9)
    </when>

    <otherwise>
        AND A.CATEGORY_CODE NOT IN (4,5,6,7,8,9)
    </otherwise>

</choose>
```

### 핵심

`<if>`

```java
if()
if()
if()
```

여러 개 실행 가능

---

`<choose>`

```java
switch()
```

처음 참인 조건 하나만 실행

# `<foreach>`

## 컬렉션 반복

```xml
<select id="searchMenuByRandomMenuCode">

    SELECT ...
    FROM TBL_MENU A

    WHERE A.MENU_CODE IN

    <foreach
        collection="randomMenuCodeList"
        item="menuCode"
        open="("
        separator=","
        close=")">

        #{ menuCode }

    </foreach>

</select>
```

---

## 속성 정리

| 속성         | 의미      |
| ---------- | ------- |
| collection | 반복할 컬렉션 |
| item       | 반복 원소   |
| open       | 시작 문자   |
| separator  | 구분자     |
| close      | 종료 문자   |

---

### 결과

```sql
WHERE A.MENU_CODE IN (1,3,7)
```

---

# `<where>`

## WHERE 자동 생성

```xml
<where>

    A.ORDERABLE_STATUS = 'Y'

    <if test="menuCode != null">
        AND A.MENU_CODE = #{ menuCode }
    </if>

</where>
```

### 기능

1. 조건이 있을 때만 WHERE 추가
2. 첫 AND 제거
3. 첫 OR 제거

---

### 결과

```sql
WHERE A.ORDERABLE_STATUS='Y'
```

또는

```sql
WHERE A.ORDERABLE_STATUS='Y'
AND A.MENU_CODE=?
```

---

# `<trim>`

## 앞뒤 문자열 제어

```xml
<trim
    prefix="("
    suffix=")"
    prefixOverrides="AND |OR ">

    <if test="nameValue != null">
        A.MENU_NAME LIKE CONCAT('%', #{ nameValue }, '%')
    </if>

    <if test="categoryValue != null">
        OR A.CATEGORY_CODE = #{ categoryValue }
    </if>

</trim>
```

---

## 속성 정리

| 속성              | 의미     |
| --------------- | ------ |
| prefix          | 앞에 추가  |
| suffix          | 뒤에 추가  |
| prefixOverrides | 앞에서 제거 |
| suffixOverrides | 뒤에서 제거 |

---

### 참고

```xml
<where>
```

=

```xml
<trim prefix="WHERE"
      prefixOverrides="AND |OR ">
```

---

```xml
<set>
```

=

```xml
<trim prefix="SET"
      suffixOverrides=",">
```

---

# `<set>`

## UPDATE SET 자동 생성

```xml
<update id="modifyMenu"
        parameterType="hashmap">

    UPDATE TBL_MENU

    <set>

        <if test="name != null">
            menu_name = #{ name },
        </if>

        <if test="categoryCode != null">
            category_code = #{ categoryCode },
        </if>

        <if test="orderableStatus != null">
            orderable_status = #{ orderableStatus },
        </if>

    </set>

    WHERE MENU_CODE = #{ code }

</update>
```

### 기능

* 입력된 컬럼만 UPDATE
* 마지막 쉼표 자동 제거

# MenuService.java

```java
public void selectMenuByPrice(int price) {

    SqlSession sqlSession = getSqlSession();

    try {

        DynamicSqlMapper mapper =
                sqlSession.getMapper(
                        DynamicSqlMapper.class);

        Map<String, Integer> map =
                new HashMap<>();

        map.put("price", price);

        printMenuList(
                mapper.selectMenuByPrice(map)
        );

    } finally {

        sqlSession.close();

    }
}
```

# 동적 SQL 태그 비교

| 태그          | 역할               | 대응 Java     |
| ----------- | ---------------- | ----------- |
| `<if>`      | 조건부 SQL          | if          |
| `<choose>`  | 하나만 선택           | switch-case |
| `<foreach>` | 반복 SQL 생성        | for-each    |
| `<where>`   | WHERE 자동 처리      | -           |
| `<set>`     | SET 자동 처리        | -           |
| `<trim>`    | prefix/suffix 제어 | -           |

---

# chap03 vs chap04 설정 비교

| 항목        | chap03               | chap04         |
| --------- | -------------------- | -------------- |
| DB 접속 정보  | XML 직접 작성            | properties 분리  |
| 타입 별칭     | FQCN 사용              | typeAlias 사용   |
| Mapper 등록 | mapper resource      | package scan   |
| 쿼리 실행     | session.selectList() | getMapper()    |
| SQL 로그    | 없음                   | STDOUT_LOGGING |
| 검색 파라미터   | DTO 또는 Map           | SearchCriteria |

---

# #{} vs ${}

| 구분            | #{}                   | ${}                   |
| ------------- | --------------------- | --------------------- |
| 용도1           | PreparedStatement 바인딩 | 문자열 치환                |
| 용도2           | -                     | properties 주입         |
| SQL Injection | 안전                    | 위험                    |
| 사용 대상         | 값(Value)              | 컬럼명, 테이블명, properties |

---

${url}

# 한 줄 정리

> **MyBatis 동적 SQL은 `<if>`, `<choose>`, `<foreach>`, `<where>`, `<set>`, `<trim>` 태그를 이용해 조건에 따라 SQL 구조 자체를 동적으로 조립하며, `typeAlias`, `properties`, `package scan`, `getMapper()`를 활용해 유지보수성과 타입 안전성을 높이는 방식이다.**
