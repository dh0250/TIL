# chap01 — JPA 영속성 컨텍스트 (Persistence Context)

- JPA는 SQL을 직접 작성하지 않고 객체(Entity)를 통해 DB를 조작
- EntityManagerFactory — 애플리케이션 전체에서 하나만 생성 (싱글톤)
- EntityManager — 트랜잭션 단위로 생성·폐기 (요청마다 새로 생성)
- 영속성 컨텍스트 — 엔티티를 영구 저장하는 환경으로, 1차 캐시와 쓰기 지연 등을 관리

## section01 — EntityManager 생성 및 관리

### 생성 계층 구조

```
Persistence.createEntityManagerFactory("jpatest")
         ↓
EntityManagerFactory (EMF)  ← 앱 전체 하나 (싱글톤), 생성 비용 큼
         ↓ factory.createEntityManager()
EntityManager (EM)          ← 스레드별/트랜잭션별 생성, 1차 캐시 관리
```

| 객체 | 특징 |
| --- | --- |
| EntityManagerFactory | 설정 정보를 읽어 DB 커넥션 풀을 생성함 |
| EntityManager | 내부에 영속성 컨텍스트를 가짐. 스레드 간 공유 금지 (매번 새로 생성) |

## section02 — Entity 매핑 및 CRUD

### @Entity 주요 어노테이션

| 어노테이션 | 역할 |
| --- | --- |
| @Entity | JPA 관리 클래스임을 명시. 기본 생성자 필수 |
| @Table | 매핑할 DB 테이블 이름 (생략 시 클래스명 사용) |
| @Id | PK(기본키) 필드 지정 |
| @Column | 필드 ↔ 컬럼 매핑. name, nullable 등 설정 가능 |

### CRUD 메서드 (조회)

- `em.find(Menu.class, menuCode)` : PK 기반 단건 조회
- 동작 : 1차 캐시 먼저 확인 → 없으면 SQL 실행 → 결과를 1차 캐시에 저장 후 반환

## section03 — Entity Life Cycle (영속성 생명주기)

### 엔티티의 4가지 상태

1. 비영속 (Transient): new로 생성된 객체. 영속성 컨텍스트와 무관함
2. 영속 (Managed): em.persist() 또는 em.find()를 통해 관리되는 상태. 변경 감지 적용
3. 준영속 (Detached): 관리되다가 em.detach(), em.clear(), em.close()로 분리된 상태
4. 삭제 (Removed): em.remove()로 삭제 마킹된 상태

### 주요 전환 메서드

| 메서드 | 설명 |
| --- | --- |
| em.detach(entity) | 특정 엔티티를 관리 대상에서 제외 (변경 감지 안 됨) |
| em.merge(entity) | 준영속 엔티티의 정보를 가져와 새로운 영속 상태 엔티티 반환 |
| em.clear() | 영속성 컨텍스트의 모든 엔티티를 준영속화 (초기화) |
| em.flush() | 영속성 컨텍스트의 변경 내용을 DB에 동기화 (SQL 전송) |

## MyBatis vs JPA 비교

| 항목 | MyBatis | JPA |
| --- | --- | --- |
| SQL 관리 | XML에 직접 SQL 작성 | Java 객체와 어노테이션으로 관리 (SQL 자동 생성) |
| 핵심 객체 | SqlSession | EntityManager |
| 캐싱 | 수동 설정 필요 | 1차 캐시 기본 내장 (영속성 컨텍스트) |
| 변경 감지 | 직접 UPDATE 쿼리 실행 필요 | 엔티티 필드 수정 시 자동 반영 (Dirty Checking) |
| 동일성 보장 | 매번 새 객체 반환 가능 | 동일 EM 내 같은 ID 조회 시 == true 보장 |

## 핵심 요약

| 항목 | 내용 |
| --- | --- |
| EMF & EM | 공장(EMF)은 하나만, 일꾼(EM)은 트랜잭션마다 새로 고용 |
| 영속성 컨텍스트 | EM이 내부적으로 엔티티를 저장하고 관리하는 가상의 DB 공간 |
| 1차 캐시 | 한 트랜잭션 내에서 동일 조회 시 DB 재접근을 방지하는 성능 최적화 |
| 변경 감지 | 영속 상태 객체의 setter 호출만으로 DB UPDATE가 예약됨 |
| 기본 생성자 | JPA가 객체를 동적으로 생성하기 위해 protected 이상의 기본 생성자가 반드시 필요 |

# chap02 — 엔티티 매핑 (Entity Mapping)

## 개념

객체(Java)와 관계형 데이터베이스(DB)의 테이블을 어떻게 연결할 것인지 정의하는 과정

- JPA의 핵심은 객체와 테이블을 정확히 매핑하는 것
- 매핑 어노테이션을 통해 DB 스키마 자동 생성 및 SQL 생성을 제어함
- 데이터 타입, 제약 조건, 기본키 생성 전략 등을 설정

---

## 프로젝트 구조

```

└── src/main/java/com/ohgiraffers/mapping/section01/entity/
    ├── Member.java           ← 엔티티 클래스 (매핑 정보 포함)
    ├── MemberRole.java       ← 권한 정보를 담은 Enum 클래스
    ├── MemberRegistDTO.java  ← 데이터 전송 객체
    ├── MemberRepository.java ← 데이터 접근을 위한 Repository
    └── MemberService.java    ← 비즈니스 로직 처리
```

## 주요 매핑 어노테이션

| 어노테이션 | 설명 | 주요 속성 |
| --- | --- | --- |
| `@Entity` | 클래스를 테이블과 매핑할 엔티티로 지정 | name: JPQL에서 사용할 엔티티 이름 (ex: entityMember) |
| `@Table` | 엔티티와 매핑할 실제 DB 테이블 명시 | name: 테이블명 (ex: tbl_member) |
| `@Id` | 테이블의 기본키(PK) 매핑 | 필드에 선언하여 PK 임을 명시 |
| `@GeneratedValue` | 기본키 생성 전략 설정 | IDENTITY(DB 위임), SEQUENCE, AUTO |
| `@Column` | 필드를 컬럼에 매핑 | name, nullable, unique, length, columnDefinition |
| `@Enumerated` | 자바의 enum 타입을 매핑 | EnumType.STRING(문자열 저장), EnumType.ORDINAL |
| `@Transient` | 특정 필드를 컬럼 매핑에서 제외 | DB에 저장하거나 조회하지 않을 데이터에 사용 |

---

## 상세 매핑 설정 (Member.java 예시)

### 1. 기본키 매핑

- GenerationType.IDENTITY: MySQL의 auto_increment와 같이 DB가 키 생성을 담당하도록 설정

### 2. 컬럼 제약 조건

- unique = true: 해당 컬럼에 유니크 제약 조건 추가
- nullable = false: NOT NULL 제약 조건 설정
- columnDefinition: varchar(10) 또는 default 'Y' 처럼 특정 DB SQL 문구를 직접 삽입 가능

### 3. Enum 매핑

- @Enumerated(EnumType.STRING) 사용: 권한명(ROLE_MEMBER 등) 그대로 저장되어 데이터 가독성과 안전성이 높음

## 핵심 로직 동작 요약

```java
// 1. 엔티티 객체 생성 (기본키는 DB에 위임하므로 생성 시 제외)
Member member = new Member(
    "user01", "pass01", "홍길동",
    "010-1234-5678", "서울시", LocalDateTime.now(),
    MemberRole.ROLE_MEMBER, "Y"
);

// 2. 비즈니스 로직 (MemberService.java)
// DTO를 통해 전달받은 데이터를 엔티티 객체로 생성 후 저장
@Transactional
public void registMember(MemberRegistDTO newMember) {
    Member member = new Member(
        newMember.getMemberId(),
        newMember.getMemberPwd(),
        // ... 생략
    );
    memberRepository.save(member); // 영속성 컨텍스트에 저장 및 DB 반영
}
```

## 핵심 요약

| 항목 | 내용 |
| --- | --- |
| @Entity 주의점 | 기본 생성자(protected Member() {})가 반드시 필요함 |
| @Column 속성 | length 속성으로 문자열 길이를 제한하거나 nullable로 필수값을 제어함 |
| @Transient 활용 | DB와 무관하게 자바 객체에서만 사용하고 싶은 필드(ex: 임시 계산값)에 사용 |
| IDENTITY 전략 | DB에 저장이 되어야만 PK 값을 알 수 있으므로, 영속화 시 즉시 INSERT SQL이 실행됨 |
