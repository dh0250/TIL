# chap05 — Spring Boot + MyBatis 통합

## 개념

MyBatis를 Spring Boot와 통합하여 REST API 서버를 구성하는 방법

- chap04까지는 `SqlSessionFactory`를 직접 생성하고 `SqlSession`을 열고 닫는 코드가 필요했음
- Spring Boot가 DataSource, SqlSessionFactory, 트랜잭션을 자동으로 관리
- `@Mapper` + `@MapperScan`으로 Mapper 빈 자동 등록
- `@Transactional`로 commit/rollback 수동 코드 제거
- `@RestController`로 JSON API 응답

## menuMapper.xml 주요 포인트

### resultMap — 컬럼명 ↔ 필드명 매핑

```xml
<resultMap id="menuResultMap" type="com.ohgiraffers.springmybatis.menu.model.dto.MenuDTO">
    <id property="code" column="menu_code"/>
    <result property="name" column="menu_name"/>
    <result property="price" column="menu_price"/>
    <result property="categoryCode" column="category_code"/>
    <result property="orderableStatus" column="orderable_status"/>
</resultMap>
```

### INSERT — useGeneratedKeys

```xml
<insert id="registMenu"
    parameterType="com.ohgiraffers.springmybatis.menu.model.dto.MenuDTO"
    useGeneratedKeys="true"
    keyProperty="code">
    insert into tbl_menu (menu_name, menu_price, category_code, orderable_status)
    values ( #{ name }, #{ price }, #{ categoryCode }, #{ orderableStatus } )
</insert>
```

```
INSERT 실행
   ↓
DB가 AUTO_INCREMENT로 menu_code 자동 생성
   ↓
useGeneratedKeys="true" → 생성된 키를 Java로 돌려줌
   ↓
keyProperty="code" → MenuDTO.code 필드에 자동 세팅
```

### Soft Delete

```xml
<!-- <delete> 가 아닌 <update> 사용 — 실제 행을 지우지 않음 -->
<update id="deleteMenu" parameterType="_int">
    update tbl_menu
       set orderable_status = 'N'
     where menu_code = #{ menuCode }
</update>
```

## MenuService.java — @Transactional

```java
@Service
public class MenuService {

    private final MenuMapper menuMapper;

    public MenuService(MenuMapper menuMapper) {   // 생성자 주입
        this.menuMapper = menuMapper;
    }

    public List<MenuDTO> findAllMenu() {          // 조회 → @Transactional 불필요
        return menuMapper.findAllMenu();
    }

    @Transactional
    public MenuDTO registNewMenu(MenuDTO newMenu) {
        menuMapper.registMenu(newMenu);           // insert 후 newMenu.code 자동 세팅됨
        return newMenu;
    }

    @Transactional
    public boolean modifyMenu(MenuDTO menu) {
        return menuMapper.modifyMenu(menu) > 0;
    }

    @Transactional
    public boolean deleteMenu(int menuCode) {
        return menuMapper.deleteMenu(menuCode) > 0;
    }
}
```

| 항목 | chap04 | chap05 |
| --- | --- | --- |
| 트랜잭션 시작 | `SqlSession` 생성 | 자동 (Spring AOP) |
| 성공 시 | `sqlSession.commit()` | 자동 commit |
| 실패 시 | `sqlSession.rollback()` | 예외 발생 시 자동 rollback |

| 코드 | try-catch-finally 블록 직접 작성 | `@Transactional` 어노테이션 하나 |

# chap01 — JPA 영속성 컨텍스트 (Persistence Context)

## 개념

JPA의 핵심인 `EntityManager`와 영속성 컨텍스트가 무엇인지, 어떻게 생성·사용하는지 이해하는 단원

- JPA는 SQL을 직접 작성하지 않고 **객체(Entity)** 를 통해 DB를 조작
- `EntityManagerFactory` — 애플리케이션 전체에서 하나만 생성 (싱글톤)
- `EntityManager` — 트랜잭션 단위로 생성·폐기 (요청마다 새로 생성)
- `persistence.xml` — JPA 설정 파일 (DB 접속 정보 + 엔티티 클래스 등록)
- `em.find()` — PK 기반 단건 조회 (SQL `SELECT` 자동 생성)

## EntityManagerFactory vs EntityManager

```
Persistence.createEntityManagerFactory("jpatest")
         ↓
  EntityManagerFactory  ← 애플리케이션 전체에서 하나 (싱글톤)
         ↓  factory.createEntityManager()
  EntityManager         ← 트랜잭션 단위로 생성·폐기 (매번 새 인스턴스)
         ↓
  영속성 컨텍스트 (Persistence Context)  ← EM이 내부적으로 관리하는 1차 캐시
```

| 객체 | 생명주기 | 비유 |
| --- | --- | --- |
| `EntityManagerFactory` | 앱 시작 ~ 종료 (싱글톤) | DB 커넥션 풀 |
| `EntityManager` | 트랜잭션 단위 | DB 커넥션 하나 |

## section02 — Menu.java (@Entity)

```java
@Entity(name = "Section02Menu")   // JPQL에서 사용할 엔티티 이름
@Table(name = "tbl_menu")         // 실제 DB 테이블명
public class Menu {

    @Id
    @Column(name = "menu_code")   // 컬럼명 ↔ 필드명 매핑
    private int menuCode;

    @Column(name = "menu_name")
    private String menuName;

    // ...

    protected Menu() {}   // JPA 기본 생성자 필수
}
```

| 어노테이션 | 역할 |
| --- | --- |
| `@Entity` | JPA가 관리할 클래스로 지정 |
| `@Table(name)` | 매핑할 DB 테이블 이름 지정 |
| `@Id` | PK 필드 지정 |
| `@Column(name)` | 필드 ↔ 컬럼명 매핑 (이름이 다를 때 필수) |
| `protected Menu()` | JPA 프록시 생성에 필요한 기본 생성자 (필수) |

## section02 — EntityManagerCRUD.java

```java
public Menu findMenuByMenuCode(int menuCode) {
    entityManager = EntityManagerGenerator.getInstance();
    return entityManager.find(Menu.class, menuCode);
    // find(엔티티타입, PK) → SELECT * FROM tbl_menu WHERE menu_code = ? 자동 생성
}
```

### em.find() 동작 흐름

```
em.find(Menu.class, 2)
```

| 항목 | MyBatis | JPA (chap01) |

| `EntityManager` | 트랜잭션마다 새로 생성·폐기 — 스레드 공유 금지 |
| `persistence.xml` | `META-INF/` 위치 필수, `persistence-unit name`이 팩토리 생성 인수와 일치해야 함 |
| `@Entity` | JPA가 관리할 클래스 선언, 기본 생성자(`protected` 이상) 필수 |
| `@Id` | PK 필드 지정 필수 |
| `@Column(name)` | 필드명과 컬럼명이 다를 때 매핑 |
| `em.find()` | PK로 단건 조회, 1차 캐시(영속성 컨텍스트) 먼저 확인 후 DB 조회 |
| 영속성 컨텍스트 | EntityManager가 내부적으로 관리하는 1차 캐시 — 같은 EM 내에서 동일 PK 조회 시 SQL 재실행 없음 |

> **chap01은 JPA의 시작점으로, SQL 없이 `@Entity` + `em.find()` 만으로 DB 조회가 가능하며, `EntityManagerFactory`(싱글톤) vs `EntityManager`(트랜잭션 단위) 의 생명주기 차이를 이해하는 것이 핵심이다.**
